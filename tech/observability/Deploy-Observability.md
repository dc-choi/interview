# 배포 가시성 — 배포를 관측 신호로 통합하기

장애 대응의 첫 질문은 대부분 "최근에 뭐 배포했어?"다. 배포 플랫폼이 여러 개면 이 질문에 답하려고 ArgoCD, 배포 서버, Git을 순회해야 하고, 그 시간이 곧 장애 대응 지연이 된다. 해법은 **모든 플랫폼이 공유하는 신호(APM 트레이스의 배포 태그)로 배포 이벤트를 통합 탐지**하고, 장애가 나면 배포 이력이 장애 스레드로 찾아오게 만드는 것.

## 문제 — 플랫폼마다 다른 배포, 다른 알림

- EKS: GitOps(ArgoCD) 매니페스트 머지로 배포
- ECS: 태스크 정의 교체로 롤아웃
- EC2/ASG: 배포 서버에서 rsync로 수십 대 배포

각 플랫폼이 **다른 채널, 다른 포맷**으로 알림을 내면 배포 이력이 한곳에 모이지 않는 관측 공백이 생긴다. 배포 자체는 되는데 "무엇이 언제 어디에 나갔는가"를 아무도 한눈에 못 본다.

## 핵심 아이디어 — APM 스팬 태그를 공통 신호로

플랫폼은 제각각이어도 **모든 서비스가 APM 트레이스를 생성한다면** 그것이 플랫폼 중립적 신호다.

- 전 서비스에 트레이서를 심고 Unified Tagging으로 `version`, `git.commit.sha`를 스팬에 태깅
- `service + env + version + git_sha` 조합이 처음 관측되는 순간 = 새 배포
- 배포 파이프라인 3종을 각각 후킹하는 대신, **이미 깔린 모니터링 토대 위에 배포 관측을 얹는** 구조

### 트레이드오프

- **트래픽 전제**: 트레이스가 생겨야 탐지된다 — 트래픽 없는 서비스는 잡히지 않음
- **태깅 규율 필수**: 태그 누락 = 탐지 실패. 전 서비스 공통 태깅 규칙이 선행 조건
- **폴링 지연**: 주기 조회(pull) 방식은 배포 시점과 탐지 시점 사이 지연이 있다 — 파이프라인이 직접 이벤트를 쏘는 push 방식(빌드 시작 → 완료 → 배포 시작 → 완료 타임라인)이 다음 단계

## 아키텍처

EventBridge(1분 스케줄) → Lambda가 APM 스팬 집계 조회 → 신규 조합 탐지 + DynamoDB로 dedup → GitHub API로 커밋 내용, 커밋터 보강 → **단일 Slack 채널에 통일된 카드 포맷**(서비스, 버전, 플랫폼, 커밋, 바로가기) 발송.

## 함정 사례 — 같은 언어, 다른 실행 모델

웹(FPM)은 요청마다 스팬이 생겨 태깅이 자연스럽지만, **CLI 배치는 프로세스 시작 시 root 스팬이 즉시 생성**되어 이후 태그를 소급할 수 없다 → 배치만 탐지에서 누락. 해결은 CLI에서 이미 생성된 root 스팬에 직접 태그를 기록하는 것. 같은 런타임이라도 실행 모델(요청 단위 vs 프로세스 단위)에 따라 계측 방식이 달라진다는 일반 교훈.

## 장애 스레드로 배포를 끌어오기

통합 채널만으로는 사람이 여전히 찾아가야 한다. 다음 단계는 **긴급 알림이 발생하면 직전 30분의 배포 이력을 같은 스레드에 자동 첨부** — 장애 인지와 동시에 용의자 목록이 도착한다.

- 사람이 배포 이력을 찾는 구조 → 배포 이력이 장애에 찾아오는 구조로 역전
- 장애 대응의 출발선 자체가 당겨진다 (원인 후보 확인이 대응의 첫 단계이므로)

## 사례 (아임웹, 2026)

prod 서비스 247개(ECS, EC2, EKS 전체)를 통합, 장애 스레드 배포 이력 첨부 중앙값 13초, 운영 기간 중 24건의 원인 규명에 기여 (사람 21건, AI 3건). 향후 계획으로 push 방식 전환과 LLM 기반 배포 위험 등급(P1~P4) 자동 산정, 위험 배포 시 모니터링 포인트 자동 제시.

## 체크포인트

- 장애 대응 시 "최근 배포"를 확인하는 데 몇 개의 시스템을 순회하는가
- 멀티 플랫폼 배포 통합의 두 접근 비교 — 파이프라인별 후킹(push, 정확하지만 N개 통합 필요) vs 공통 관측 신호(pull, 한 번에 되지만 트래픽과 태깅 전제)
- 배포 탐지의 식별자 설계 (service + env + version + git SHA)
- 요청 단위 런타임과 프로세스 단위 배치의 계측 차이
- 알림 통합에서 멈추지 않고 장애 컨텍스트에 자동 결합했는가

# 메모리 개요와 할당 방식

## 메모리의 종류

| 종류 | 특징 |
|------|------|
| 레지스터 | CPU 내부에 존재. 가장 빠름. 휘발성. 32bit/64bit 구분이 레지스터 크기 |
| 캐시 | 레지스터와 메인 메모리의 속도 차이를 줄임. L1 → L2 → L3 순으로 확인 |
| 메인 메모리(RAM) | 운영체제와 프로세스가 올라가는 곳. 휘발성 |
| 보조저장장치(ROM) | 비휘발성. 가격 저렴 |

### 메모리 계층 구조와 접근 속도
- 레지스터: CPU의 1 사이클
- 캐시: 수 사이클 ~ 수십 사이클
- 메인 메모리: 수백 사이클
- 보조기억장치: **수백만 사이클**

## 메모리와 주소

운영체제는 메모리를 1바이트 크기로 구역을 나누고 숫자를 매김. 이것이 주소.

### 32bit vs 64bit
- 32bit: 레지스터, ALU, 버스가 32bit → 메모리 최대 4GB
- 64bit: 레지스터, ALU, 버스가 64bit → 메모리 2^64까지 지원

### 물리 주소 공간 vs 논리 주소 공간
- **물리 주소 공간**: 메모리를 연결하면 0x00번지부터 시작하는 실제 주소
- **논리 주소 공간**: 사용자 관점에서 바라본 주소
- 사용자는 물리 주소를 몰라도 논리 주소로 물리 주소에 접근 가능

### 경계 레지스터
- CPU 내에 있는 레지스터로 운영체제 영역과 사용자 영역을 하드웨어적으로 분리
- 메모리 관리자가 사용자 프로세스가 경계 레지스터 값을 벗어났는지 검사하고 벗어나면 프로세스를 종료

### 절대주소와 상대주소
- **상대주소(논리 주소)**: 컴파일러가 0번지에서 시작한다고 가정하고 만든 주소
- **절대주소(물리 주소)**: 메모리 관리자가 바라본 실제 주소
- **재배치 레지스터**: 상대주소 + 재배치 레지스터 값 = 절대주소
- 프로세스가 메모리 위치를 이동해도 재배치 레지스터 값만 변경하면 됨

## 메모리 할당 방식

### 메모리 오버레이
- 프로세스 메모리 공간이 부족할 때 당장 필요한 부분만 메모리에 올리고 나머지는 하드디스크의 swap 영역에 저장
- 사용자는 메모리가 더 큰 것으로 느끼지만 swap 과정 때문에 실제보다 느리게 동작

### 가변 분할 방식
- 프로세스 크기에 따라 메모리를 나눔 (연속 메모리 할당)
- 장점: 내부 단편화가 일어나지 않음
- 단점: **외부 단편화** 발생

### 고정 분할 방식
- 프로세스 크기에 상관없이 고정된 크기로 메모리를 나눔 (비연속 메모리 할당)
- 장점: 구현이 간단하고 오버헤드가 적음
- 단점: **내부 단편화** 발생 (작은 프로세스도 큰 영역에 할당)

### 외부 단편화
- 가변 분할에서 서로 다른 크기의 프로세스가 종료/생성되며 사용할 수 없는 빈 공간이 분산
- 해결: **조각모음** (현재 실행 프로세스를 일시중지하고 메모리 공간 이동 → 오버헤드 발생)

### 내부 단편화
- 고정 분할에서 프로세스가 고정 크기보다 작을 때 남는 공간이 낭비
- 완전한 해결은 불가. 고정값 조정으로 최소화

### 버디 시스템
- 2의 승수로 메모리를 분할하여 할당
- 프로세스 크기보다 작은 값을 만날 때까지 반복 분할하고 그보다 큰 구역에 할당
- 내부 단편화가 발생하지만 기존보다 적음
- 프로세스 종료 후 근접 공간을 합치기 쉬움 (조각모음보다 훨씬 간단)
- 가변 분할의 유연성 + 외부 단편화 방지의 장점

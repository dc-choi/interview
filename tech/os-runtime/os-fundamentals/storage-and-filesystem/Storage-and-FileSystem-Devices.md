# 저장 장치와 주변장치

## 기억장치계층

| 계층 | 구성요소 | 특성 |
|------|---------|------|
| CPU내부 | 레지스터, L1/L2/L3캐시 | 가장 빠름, 용량 작음 |
| 1차메모리 | RAM | 휘발성, 전원 꺼지면 소멸 |
| 2차메모리 | SSD, HDD | 비휘발성, 영구 저장 |

- 위로 갈수록 빠르고 비싸고 용량이 작음
- 아래로 갈수록 느리고 싸고 용량이 큼

### 캐시메모리
- CPU와 RAM 사이의 속도 차이를 극복하기 위해 존재
- 자주 접근할 것으로 예측되는 데이터를 미리 RAM에서 캐시로 적재
- L1(가장 빠름, 가장 작음) → L2 → L3(가장 느림, 가장 큼)

### 각기억장치관리방법
| 기억장치 | 관리방식 |
|---------|---------|
| 레지스터 | 고유 이름 (EAX, EBX 등) |
| RAM | 일련번호 = 메모리 주소 |
| HDD/SSD | 트랙 번호 + 섹터 번호 |

## HDD(하드디스크)

### 구조
- 자기 디스크 위를 헤드가 회전하며 읽기/쓰기
- 데이터는 **트랙**(원형 경로)과 **섹터**(트랙의 구간)로 구분
- **스핀들**: 플래터를 회전시키는 막대
- **플래터**: 자기화된 원판. 표면의 자극이 N이면 0, S이면 1로 인식. 보통 2개 이상
- **디스크 암**: read/write 헤드가 고정. 헤드는 항상 같이 움직임
- **실린더**: 여러 플래터의 같은 트랙의 집합
- **섹터**: 하드디스크의 가장 작은 단위

### 배드섹터
- 특정 섹터에 약 10만번 OverWrite하면 물리적으로 사용 불가
- 이를 배드섹터라고 하며 복구 불가

### 조각모음
- 파일이 여러 섹터에 흩어져 저장되면(파편화) 헤드 이동 시간 증가 → 성능 저하
- 조각 모음 = 파편화된 데이터를 연속된 영역으로 재배치
- 요즘 OS는 자동으로 처리

## SSD

- HDD의 자기 디스크 → 반도체 칩으로 대체
- 트랙/섹터 개념은 동일하게 논리적으로 관리
- 조각 모음 불필요 (칩으로 랜덤 접근하므로 물리적 이동 없음)
- 칩의 I/O 성능에 따라 속도 결정

### SSD 특성
- 성능이 좋고 소음이 없음. 자석에 데이터가 손상되지 않고 충격에 강함
- 단점: 특정 영역에 데이터를 사용했으면 덮어쓰기가 불가능
- 플래시 메모리는 지우는 횟수가 정해져 있어 같은 지점을 계속 지우고 쓰면 사용 불가

### TRIM(보강)
- SSD에서 삭제된 블록을 OS가 미리 알려주는 명령
- TRIM 없이는 새 데이터를 쓸 때 기존 데이터를 먼저 지워야 해서 쓰기 성능 저하
- 현대 OS는 TRIM을 자동으로 실행

## 주변장치

### 종류

| 구분 | 예시 | 전송 단위 |
|------|------|----------|
| 캐릭터 디바이스 | 마우스, 키보드, 사운드카드 | 글자 단위 (적은 양) |
| 블록 디바이스 | 하드디스크, SSD, 그래픽카드 | 블록 단위 (많은 양) |

### 버스 구조
- 각 하드웨어는 address, data, control 세 가지 버스로 연결
- **시스템 버스**: 고속 CPU, 메모리가 사용
- **입출력 버스**: 주변장치가 사용 (고속/저속으로 다시 분리하여 병목 해결)
- 그래픽카드는 대용량 데이터를 다루므로 시스템 버스에 직접 연결

### 입출력 제어기와 DMA
- CPU가 IO 명령을 만나면 입출력 제어기에 작업을 맡기고 다른 작업 수행
- 입출력 제어기가 메모리에 접근하려면 원래 CPU가 필요했으나 **DMA(Direct Memory Access)** 제어기로 해결
- DMA를 통해 CPU 도움 없이 데이터를 직접 메모리에 저장/가져올 수 있음
- CPU와 DMA가 메모리 충돌하지 않도록 영역을 나누는 것이 **Memory Mapped I/O**

### 마우스/키보드의 동작
1. 디바이스 컨트롤러가 입력을 감지
2. CPU에 인터럽트를 보냄
3. 디바이스 드라이버가 데이터를 읽어감
4. 드라이버가 운영체제에 이벤트 전달
5. 운영체제가 애플리케이션에 이벤트 전달
6. 애플리케이션이 이벤트 처리

# 채용 인터뷰

## 프론트엔드 개발자 채용 면접 질문

### 기술 기초

**JavaScript**
- 이벤트 루프 동작 방식
- ESM vs CJS 차이
- this 바인딩 규칙

**TypeScript**
- 유틸리티 타입 활용
- 타입 추론 vs 명시적 타입 선언 기준
- 타입 단언 (as, !)의 적절한 사용
- zod를 이용한 런타임 유효성 검사
- type vs interface 차이와 선택 기준

### Git / 버전 관리
- Git 브랜치 전략 (Git Flow, Trunk-based 등)
- 커밋 메시지 컨벤션 (Conventional Commits 등)

### 배포 전략
- 블루/그린, 카나리, 롤링 업데이트의 차이와 선택 기준
- AWS & 인프라 경험도 함께 확인

### 기술부채 개선 경험
- 현재 도메인과 UI가 강결합된 구조에서 어떻게 개선할 것인지
- 기존 레거시와 새로운 코드가 혼재된 상황에서 기능 구현과 개선을 병행하는 방법
- 수치화 → 데이터 기반 의사결정 경험

### 협업 & 커뮤니케이션

**API 명세 관련**
- 백엔드 API 명세가 완전하지 않을 때 어떻게 커뮤니케이션하나요?
- API 명세서 문서화를 요구하거나 직접 정리해본 경험
- Swagger, Postman, Stoplight 같은 도구 사용 경험
- 협업 시 가장 효과적이었던 소통 방식

**코드 리뷰**
- 코드 리뷰를 어떻게 진행하는지 (코드 중복 줄이기 관점)
- 리뷰에서 가장 중요하게 보는 기준 (성능, 일관성 등)
- conflict나 의견 충돌이 생겼을 때 조율 방법
- 코드 리뷰 도입 후 팀 퀄리티가 개선됐던 경험

### AWS & 인프라 (보안 중심)

**S3 + CloudFront 정적 배포**
- S3 정적 웹사이트 배포 시 퍼블릭 액세스 차단과 보안 관리 방법
- S3 버킷 정책 구성 경험
- Presigned URL 활용 경험
- CloudFront 연동 시 Origin Access Control (OAC) 설정 경험

**인증**
- AWS Cognito나 다른 인증 시스템 사용 경험
- OAuth2/JWT 토큰 기반 인증 처리 구성 방법
- 토큰 만료 처리, silent refresh 방식
- Auth 관련 에러 디버깅 및 사용자 경험 개선 사례

**보안**
- 자격증명 유출이나 권한 관리 문제 경험
- IAM 권한 최소화 원칙 적용 경험
- 환경변수에 민감 정보가 노출되지 않도록 관리하는 방법
- 프론트엔드에서 민감한 정보가 노출되지 않도록 하기 위한 고려사항

## 채용 전략 — 신입 vs 경력직 ROI

### 경력직 채용의 현실

- **원하는 기술 스택 + 팀 문화 적합 + 좋은 태도**를 모두 갖춘 시니어는 시장에 거의 없음
- 세 가지 중 어느 하나를 포기하고 채용하면 이후 팀 성장이 어려워지는 경우가 많음 (특히 태도·문화 적합성 포기 시)
- "지금 당장" 문제를 푸는 데에는 유리하나, 장기적으로 팀 문화 형성·학습 곡선 부담

### 신입 채용의 장기 가치

- **태도·커뮤니케이션·야망·두뇌**는 경력과 거의 무관
- 좋은 태도의 신입이 1~2년 후 **경력직 수준으로 성장** 하는 사례 다수
- 팀 문화 형성에 **긍정적 방향** — 성장 마인드를 팀 전체로 전파
- 장기 ROI로 보면 시니어 채용보다 나은 경우가 많음

### 태도 > 기술 우선순위

경력 무관하게 평가 순위:
1. **좋은 태도** — 성장 욕구, 겸손, 피드백 수용
2. **커뮤니케이션** — 협업·정보 공유 능력
3. **기술 경험** — 학습 가능한 영역

기술은 3~6개월에 새로 배울 수 있지만 태도·커뮤니케이션은 바꾸기 어렵다.

### 신입 온보딩 철학

- **3개월 내 온전한 업무** 수행 기대 가능
- **1년 후 중요 프로젝트** 담당 가능한 수준까지
- 친절한 문서화·페어링으로 러닝 커브 단축
- 이직 시 아쉬움보다 **남은 사람에 집중** — 성장한 신입이 떠나도 절대적으로 남은 사람이 더 많다

### 신입 채용 시 확인 포인트

- 학습 의욕이 있는가 (개인 프로젝트·글쓰기·스터디 흔적)
- 피드백을 **방어적으로** 받지 않는가
- 본인의 부족함을 솔직히 드러내는가
- 팀 상호작용을 즐기는가
- 3년 후에 이 사람이 팀을 떠난다면 "감사한 동료"로 기억될 사람인가

### 채용 결정에서 경계할 것

- **"즉시 성과" 압박**으로 태도 맞지 않는 경력자 채용
- **기술 스택만** 보고 커뮤니케이션·태도 간과
- **학력·대기업 경험** 편향
- **주니어 투자 회피** — 팀이 경력자만으로 채워지면 학습 문화 고사

# 포용 디자인 · Inclusive Design 원칙

"평균 사용자"를 위한 제품은 **누구도 완전히 만족시키지 못한다**. 신체·문화·상황적으로 소외되는 사람이 반드시 존재하고, 그 수는 항상 엔지니어의 예상보다 크다. 포용 디자인은 **소외되는 극단부터 설계하면 나머지 모두가 이득을 본다**는 원리로 접근한다. exclusion은 innovation의 재료.

## 핵심 명제

- **평균에 맞추면 아무도 완전히 맞추지 못한다** — 평균을 표상하는 인간은 존재하지 않음
- **소외(exclusion)는 혁신(innovation)의 실마리** — 극단부터 설계하면 전체가 덕을 본다
- **장애는 영구·일시·상황의 3축** — 누구나 일시적으로 장애 사용자가 된다
- **색·청각·특정 언어 같은 단일 채널 의존 금지**
- **문화 현지화의 기준은 "make sense in that culture"**
- **동료 관찰·피드백이 가장 큰 정보원**

## 평균의 함정

### 문제

- 제품을 평균 사용자에 맞추면 **좌우 양 끝이 소외**
- "평균" 자체가 구성된 통계일 뿐 실제 그런 사용자는 없음
- 평균이 만족하는 지점은 의외로 **아무도 완벽히 만족하지 않는 지점**

### 해법: Adjustable

- 치수·색·입력 방식·폰트 크기 등 **모든 요소를 조정 가능**하게 설계
- Fix시키지 말고 **개인화**
- 좌측 극단(가장 어려움을 겪는 사용자)을 만족시키면 **우측까지 자동 만족**되는 경우가 흔함

### 대표 사례

- **자동차 충돌테스트 마네킹** — 오랫동안 남성용만 존재. 엔지니어가 남성이었기 때문. 결과적으로 여성 탑승자 안전성 저평가
- **검색어 자동완성** — 장애인을 위해 만들어졌지만 범용으로 쓰임
- **유튜브 자동 자막** — 청각장애인 개발자가 만들었고 전 세계 언어 학습자가 수혜

## Exclusion → Innovation 모델

### 관점 전환

- 소외 사용자를 위한 설계는 **비용**이 아니라 **차세대 혁신의 씨앗**
- 제약이 강한 환경에서 만든 해법이 **더 넓게 일반화**
- "접근성 = 자선"이 아니라 **"접근성 = 시장 확대"**

### 생성 루프

1. **관찰** — 소외되는 사용자군 식별
2. **제약 수용** — 그들의 환경을 설계 제약으로 받아들임
3. **해법 생성** — 특수 상황을 해결하는 도구
4. **일반화** — 만들어진 도구가 일반 사용자에게도 유용
5. **시장 확장** — 소외군 + 일반군 = 더 큰 시장

## Bias의 악순환

편향은 한 지점에서 시작해 시스템 전체를 오염시킨다.

```
Dataset Bias → Confirmation Bias → Interaction Bias
    ↑                                    ↓
Association Bias ← Automation Bias ←
```

- **Dataset Bias** — 훈련 데이터의 편중
- **Confirmation Bias** — 편향에 맞는 결과만 해석
- **Interaction Bias** — 편향된 시스템 사용이 새 편향 데이터 생성
- **Automation Bias** — 자동화된 결정을 과신
- **Association Bias** — 연관 규칙이 편향을 강화

한번 시작된 편향은 루프에서 증폭. 원천 데이터 다양성이 무너지면 복구 어려움. 실무 영향: AI 얼굴 인식 인종별 정확도 편차, 추천 알고리즘의 그룹 편중, 음성 인식의 액센트 편향. **대응**: 데이터 수집 다양성·정기 bias 감사·다양한 피드백 루프.

## Persona 재정의: 영구·일시·상황

전통적 Persona는 "장애 사용자 vs 정상 사용자"로 이분화. 현실은 3축이다.

| 유형 | 예시 | 설계 함의 |
|---|---|---|
| **영구적** | 팔이 없는 사용자 | 한 손 조작 가능해야 |
| **일시적** | 팔 부상으로 깁스 중 | 회복 기간 동안 한 손 |
| **상황적** | 한 손에 아기·장바구니·커피를 들고 있음 | 누구나 이 상황에 놓임 |

→ **누구나 언제든 장애 사용자가 된다**. "정상 사용자"는 허상.

### 설계 효과 (한 극단 설계 → 모두 혜택)

- 한 손 조작: 영구 장애 + 부상 + 아기 안은 부모
- 자막: 청각장애 + 소음 환경 + 외국어 학습자
- 큰 글자: 시각장애 + 노안 + 저조도

## Mental Model: Shut-In / Shut-Out

소외를 만드는 사고 패턴. 설계 단계마다 체크.

1. **What we made** — 정말 그게 필요한가
2. **Why we make** — 누구의 문제를 푸는가
3. **How it's made** — 만드는 방식이 누구를 배제하는가
4. **Who makes it** — 만드는 팀의 다양성
5. **Who uses it** — 쓰는 사람과 안 쓰는(못 쓰는) 사람

만드는 팀이 단일하면 **자연스럽게** 쓰는 사람도 단일해진다. 다양성은 **조직 구성부터** 시작된다.

## Color와 Shape: 다중 채널 설계

**색만으로 의미 전달 금지** (Never rely on color alone). 색맹·적록색약(남성 약 8%)·문화적 색 의미 차이·흑백 인쇄·저조도 등으로 단일 채널 실패.

**대체·보완**: 색 + 아이콘·패턴·텍스트 라벨·위치·모양. 오류를 빨강으로만 표시하지 말고 ⚠️ + "오류" 병행.

**최소 요소로 최대 기능**: "이 정보를 빼도 이해되는가"를 계속 질문. 이미지 디스크립션을 붙여도 청각장애인이 그 이미지 자체를 이해 못 할 수 있음 → **빼는 게 답**일 수 있다. 좋은 디자인은 더하기가 아니라 **빼기**에서 드러난다.

## Cultural Localization

"번역"이 아니라 "그 문화에서도 make sense 하는가" 수준까지.

### Pixar 인사이드 아웃 사례

- 원본: 아이들이 싫어하는 채소 = 브로콜리
- 일본판: 브로콜리 → 피망(일본 아이들이 싫어하는 채소)
- 같은 감정·같은 서사, **현지의 매개체로 교체**

### 소프트웨어 적용

날짜·주소 포맷, 이름 순서(성-이름 vs 이름-성), 화폐·세금·법적 고지, 성별·종교·정치 기본값, 색 의미(흰색=순결/상), RTL 언어 레이아웃(아랍어·히브리어).

**위험**: "그 문화 잘 안다" → 거의 항상 오판. 평균에 맞춰 만든 제품이 다른 나라에서 성공할 것이라는 가정 = 평균의 함정 재발. 현지 사용자 피드백 없는 번역은 실패.

## Easy Language

사용자 수준에 맞춘 언어·일러스트.

### 원칙: Who are you working for?

- 대상 사용자의 언어를 파악
- 쉬운 단어·직관적 그림
- 전문 용어·약어는 **풀어 쓰거나** 설명 링크

### 서비스 설계 레버

- **Layout Resizing** — 다양한 디바이스·폼 팩터
- **Color** — 충분한 대비, 다중 채널
- **Touch Target Size** — 손가락이 닿을 만큼
- **Labels** — 아이콘 옆에 이름
- **Content Descriptions** — 시각장애인을 위한 대체 텍스트

### 중고등학생·노인층 대상

- 그들이 쓰는 언어를 알면 역으로 **복잡한 뒷단도 견딜 만한 프론트엔드**가 나옴
- 설명이 간단하면 내부 구조가 복잡해도 사용자가 성공

## 릴리즈 노트의 사소함

- 오타·번역 실수는 **제품 전체의 품질 의심**으로 확대
- "소프트웨어에서 중요하지 않은 건 없다"
- 모든 접점(에러 메시지·버튼 문구·안내 툴팁)을 **세밀하게** 챙김

## 동료 관찰: 가장 큰 정보원

- **영감은 아마추어의 것** — 프로는 평소대로 일하며 데이터에서 영감을 추출
- 동료의 실수·하소연·습관·피드백에 **의미 있는 패턴**
- 초보자의 열정은 실수로 이어지기 쉽고, 동료가 그 실수를 사전에 잡아준다
- 팀·회사 선택에서 [[Company-Selection-Criteria|가장 중요한 기준이 동료]]인 이유

## 체크리스트 · 현실적 한계

설계 리뷰 질문: 극단적 상황에서 작동하는가 / 단일 채널 의존이 있는가 / Fix를 Adjustable로 바꿀 수 있는가 / 소외 그룹 관찰·피드백 / 다른 문화에서 make sense / 팀 구성 다양성 / 소외 해소를 혁신 재료로 보는가.

"완벽한 제품"은 꿈 — 맹점 인지·보완 스탠스가 의미. 핵심 기능이 작동하면 **사회적 공명**으로 맹점 점진 보완. 소외 해소 → 시장 확대 → 재투자의 **선순환**이 지속가능한 혁신의 경로.

### 흔한 오해

- "접근성 = 소수를 위한 부가 기능" → 실제는 **시장 확대 레버**
- "평균 잘 만들면 배포 가능" → 평균의 함정 반복
- "이 문화 잘 안다" → 현지 검증 없이는 거의 실패
- "색 대비만 맞추면 끝" → 다중 채널·인지·운동·언어 전방위
- "영감이 오면 만든다" → 영감은 평소 관찰의 누적

## 면접 체크포인트

- "평균의 함정"을 한 문장으로 설명
- 영구·일시·상황적 장애의 3축과 예시
- Exclusion → Innovation 루프의 실제 사례 3가지
- 색 의존의 위험과 다중 채널 설계
- Localization vs Translation의 차이
- 제품 다양성이 팀 다양성에서 나오는 이유

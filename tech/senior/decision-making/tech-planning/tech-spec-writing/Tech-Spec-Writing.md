# 테크스펙 (Tech Spec) 작성

테크스펙은 **문서가 아니라 "토론의 장"**. 애자일 선언을 오해해 "문서는 비효율"로 치부하는 조직이 많지만, 현대적 테크스펙은 구현 전에 **값싸게 피드백을 받는 협업 도구**. 애자일의 "작동하는 소프트웨어가 포괄적 문서보다 중요"는 **일방적, 고정된 문서**를 비판한 것이지, 대화를 배제한 게 아님.

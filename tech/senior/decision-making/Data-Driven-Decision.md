# 데이터 기반 의사결정

감(感)이 아닌 데이터를 근거로 제품과 기술 방향을 결정하는 역량이다. 시니어 엔지니어는 "왜 이걸 하는가?"에 숫자로 답할 수 있어야 한다.

## 핵심 원칙

1. **측정 정의가 먼저다** — 가설, 분모, 코호트와 이벤트 연결이 없으면 숫자를 만들지 않고 미측정으로 둔다
2. **숫자에 맥락을 붙인다** — "WAU 30곳"이 아니라 "WAU 30곳, 전주 대비 +1, 성남동 이탈(-1) + 호평동 전환(+1)"
3. **행동으로 이어진다** — 데이터 수집 자체가 목적이 아님. 데이터 → 인사이트 → 액션

## 전환 퍼널 분석

사용자가 서비스를 처음 접한 순간부터 핵심 가치를 경험하기까지의 단계별 전환율을 추적한다.

school-manage의 과거 퍼널은 가입 사용자와 단체 이벤트를 서로 다른 분모로 나눠 전환율처럼 읽은 문제가 있었다. 2026-07-15 스냅숏에서도 `first_visit` 8건은 관측됐지만 `first_group_created`, `first_student_registered`, `first_attendance_recorded`는 관측되지 않아 Activation 전환율은 **미측정**이다.

**의사결정:**
- 가입 사용자와 단체 생성 이후의 `first_*` 이벤트를 같은 코호트 식별자로 연결
- 계측 완성 전에는 단계별 전환율과 병목 원인을 단정하지 않음
- 랜딩 이탈처럼 별도 정의가 있는 지표도 측정 기간, 분모와 변경 전후 조건을 함께 기록

## 코호트 분석

같은 시기에 가입한 사용자 그룹의 행동 패턴을 비교한다.

과거 0%, 15%, 36% 코호트 수치는 현재 코호트 정의와 이벤트 연결로 재현되지 않아 제품 개선 증거로 사용하지 않는다. 같은 가입 시점, 같은 활성화 정의와 관찰 기간을 갖춘 뒤에야 코호트 차이를 제품 변화나 유입 구성과 연결할 수 있다.

## 사용자 세그먼트 분석

모든 사용자를 동일하게 보지 않고, 행동 패턴에 따라 분류한다.

현재 GA4 맞춤 차원은 실제 단체와 계정 값을 제공하지 않아 단체별 WAU, WVO와 과거 S유형 수치를 재산출할 수 없다. 세그먼트 전략은 조직 식별 계측을 고친 뒤 최소 표본, 관찰 기간과 행동 정의를 명시해 다시 검증한다.

## GA4 행동 데이터 활용

제품 내 사용자 행동을 이벤트 단위로 추적한다.

- 2026-07-09부터 2026-07-15까지 `liturgical_card_viewed` 203건, 72 AU와 `patron_feast_card_viewed` 185건, 67 AU가 관찰됨
- 이 값은 가톨릭 맥락 조회의 **행동 신호**이며 단체별 주간 재방문이나 인과를 증명하지 않음

**의사결정:** 조회 경험을 핵심 가치 가설로 유지하되, 조직 귀속 WVO와 반복 방문을 연결하기 전에는 킬러 피처로 확정하지 않는다.

## 설득 도구로서의 데이터: 이해관계 충돌의 중재

데이터의 용도는 내 의사결정만이 아니다 — **부서 간 이해가 충돌할 때 추측을 객관 지표로 바꿔 합의를 만드는 중재 도구**다. 한 기능을 두고 영업은 특정 부서 특혜라 반대하고 마케팅은 다른 운영안을 주장하는 상황에서, 사전에 수집해 둔 행동 데이터(클릭수)가 논쟁을 끝낸 사례가 전형적이다.

- 분석용 데이터는 답을 찾고, 설득용 데이터는 **합의를 만든다** — 같은 수치라도 쓰임이 다르다.
- 충돌이 벌어진 뒤 수집한 데이터는 진영 논리로 읽히기 쉽다 — 평소에 깔아 둔 측정이 설득력의 원천. 계측을 미리 심는 것이 정치적 자산이기도 하다.
- 설득에 실패했을 때의 선택지는 포기, 타협, 협상 세 가지인데, 협상의 토대가 곧 논리와 데이터다 — 우기지 말고 모르면 배우고, 근거로 설득한다.

## 이탈 감지와 자동 대응

데이터 기반으로 이탈 위험을 조기 감지하고 자동으로 개입한다.

- 14일이나 30일 같은 고정 미활동 기준만으로 이탈을 확정하지 않음. 과거에는 56일 뒤 복귀 사례도 있었음
- 입력 공백, 운영 중단과 영구 이탈을 분리하고 복귀 분포를 다시 산출한 뒤 개입 기준을 정함
- 시스템 알림, 사람 DM과 구체적 사용 안내를 서로 다른 개입으로 측정

## 면접 포인트

Q. 데이터 기반 의사결정 경험이 있는가?
- 서로 다른 분모를 전환율로 읽은 오류를 발견하고 Activation을 미측정으로 되돌림
- 조회 이벤트를 가치 가설의 신호로 사용하되 단체 재방문과 인과가 확인되기 전에는 킬러 피처로 확정하지 않음
- 같은 코호트 식별자와 `first_*` 이벤트를 연결하는 계측을 다음 의사결정의 선행 조건으로 둠

Q. 어떤 지표를 추적했는가?
- 운영 기준: 2026-07-15 MAO 34곳, 목표 50곳. MAO 50은 운영지표이며 90일 사업 검증의 진입 조건은 아님
- Activation: 가입→단체→학생→출석을 같은 코호트로 연결한 뒤 측정 예정
- 행동 신호: 조회 이벤트 AU와 건수. 조직 귀속 전에는 단체별 WAU/WVO로 해석하지 않음

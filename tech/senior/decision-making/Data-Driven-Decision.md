# 데이터 기반 의사결정

감(感)이 아닌 데이터를 근거로 제품과 기술 방향을 결정하는 역량이다. 시니어 엔지니어는 "왜 이걸 하는가?"에 숫자로 답할 수 있어야 한다.

## 핵심 원칙

1. **측정할 수 없으면 개선할 수 없다** — 가설을 세우고, 지표를 정의하고, 결과를 측정한다
2. **숫자에 맥락을 붙인다** — "WAU 30곳"이 아니라 "WAU 30곳, 전주 대비 +1, 성남동 이탈(-1) + 호평동 전환(+1)"
3. **행동으로 이어진다** — 데이터 수집 자체가 목적이 아님. 데이터 → 인사이트 → 액션

## 전환 퍼널 분석

사용자가 서비스를 처음 접한 순간부터 핵심 가치를 경험하기까지의 단계별 전환율을 추적한다.

**퍼널 단계 예시:**
모임 가입 → 그룹 생성(88%) → 학생 등록(71%) → 출석 기록(84%) → WAU 전환(74%)

**병목 식별:**
- 학생 등록 단계에서 29%가 이탈 → 입력 부담이 원인
- 출석 미기록 16% → 온보딩 넛지 배너로 전환 유도

**의사결정:**
- 이탈률 81.8% → 랜딩 페이지 구현 → 12.5%로 69pp 개선
- 데이터가 "어디서 사용자를 잃는가"를 보여주고, 그 지점에 집중 투자

## 코호트 분석

같은 시기에 가입한 사용자 그룹의 행동 패턴을 비교한다.

**발견 예시:**
- 2월 중순 가입자: WAU 전환율 0% (서비스 미성숙기)
- 2월 하순: 15%
- 3월: 36% (서비스 성숙 + 타겟 사용자 집중 유입)

**인사이트:** 시간이 지남에 따라 제품이 개선되고 있음을 코호트 단위로 검증. 초기 이탈이 제품 문제인지 타이밍 문제인지 구분 가능.

## 사용자 세그먼트 분석

모든 사용자를 동일하게 보지 않고, 행동 패턴에 따라 분류한다.

**세그먼트 예시:**
- S유형(즉시 안착): 평균 학생 64명, 출석 81건, WAU 100%
- 나머지: 평균 학생 2.8명, 출석 1.5건, WAU 3%

**의사결정:** S유형의 공통 특성(팀 사용, 다수 계정)을 파악하고, 다른 세그먼트도 이 패턴으로 유도하는 전략 수립

## GA4 행동 데이터 활용

제품 내 사용자 행동을 이벤트 단위로 추적한다.

- 가장 반복 방문하는 페이지: `/students` (26명)
- 가장 오래 머무는 페이지: `/students/new` (3분10초)
- 킬러 피처 발견: 축일/전례 카드 (대시보드 진입자 21명 중 20명 사용, 인당 8.5회)

**의사결정:** "출석 기록"보다 "정보 조회(축일)"가 실제 재방문 동기임을 발견 → 제품 방향 조정

## 이탈 감지와 자동 대응

데이터 기반으로 이탈 위험을 조기 감지하고 자동으로 개입한다.

- 14일 미활동 시 이탈 위험으로 분류 → 자동 이메일 알림
- 3주 이상 활성 후 1주 미기록 → 재활성화 이메일
- GA4 이벤트 26개로 사용자 행동 자동 추적

## 면접 포인트

Q. 데이터 기반 의사결정 경험이 있는가?
- 전환 퍼널 분석으로 병목 식별 → 랜딩 페이지 구현으로 이탈률 81.8% → 12.5% 개선
- GA4 행동 데이터로 킬러 피처(축일 카드) 발견 → 제품 방향 조정
- 코호트 분석으로 제품 개선 효과를 시간축으로 검증
- 사용자 세그먼트 분석으로 S유형(즉시 안착) 패턴 식별 → 다른 세그먼트 전환 전략 수립

Q. 어떤 지표를 추적했는가?
- North Star: 월간 활성 단체 중 주간 출석 기록 지속 비율
- 퍼널별 전환율 (모임→그룹→학생→출석→WAU)
- 코호트별 리텐션, 세그먼트별 행동 차이

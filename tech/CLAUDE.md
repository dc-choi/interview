# Technical Knowledge Domain Rules

이 파일은 `tech/` 아래의 기술 지식 문서, 카테고리 인덱스와 기술 메모에 적용한다. 메모 입력을 문서화할 때는 `memo` 스킬도 함께 사용한다.

`tech/senior/`에서 조직 문화, 의사결정, 피드백, 회고와 OKR을 다룰 때는 `fit/CLAUDE.md`도 읽는다.

## 현재 실무 기술 축

- 현재 주력 ORM은 TypeORM이다. ORM 학습, 기술 문서와 면접 답변에서는 `@nestjs/typeorm`, QueryRunner 트랜잭션과 마이그레이션 CLI를 우선하고 Prisma는 과거 실무 사례로 활용한다.

## 기술 문서 최신성

- AWS, OpenAI, Anthropic, Node.js, TypeScript, Kubernetes, 데이터베이스 제품과 보안 표준처럼 변동성이 큰 기술 문서는 공식 1차 출처를 확인한 날짜를 frontmatter의 `verified_at: YYYY-MM-DD`로 기록한다.
- `verified_at`은 본문과 출처를 실제로 대조한 경우에만 갱신한다. 자동 스캔이나 파일 이동만으로 검증한 것처럼 날짜를 변경하지 않는다.
- 가격, quota, 지원 정책, 시험 범위, 제품 버전과 API 동작은 공식 출처와 검증 기준일 없이 현재 사실처럼 단정하지 않는다.
- `항상`, `무조건`, `제한 없음`, `지원 종료`, `무료`, `동일`과 `완전` 같은 절대 표현은 공식 규격이나 정책이 전체 범위를 보장하고 출처를 함께 제시할 때만 쓴다. 그렇지 않으면 버전, 리전, 에디션, 시점과 예외 조건을 명시한다.
- 사실 관계를 수정했다면 관련 문서의 상충 표현과 출처를 함께 확인하고 `verified_at`을 갱신한다.

## 카테고리 인덱스

- `tech/` 카테고리 최상위 인덱스는 루트의 leaf 폴더 분할 규칙에서 제외한다.
- 기술 메모를 추가하거나 이동할 때 대상 카테고리 인덱스와 README 진행률을 함께 확인한다.

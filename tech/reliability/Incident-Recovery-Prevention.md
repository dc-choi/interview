# 장애복구와재발방지

> 출처: [[Toss-Runners-High-Seminar|토스 러너스하이 세미나]] 기술 운영 평가

## 장애복구설계

### 스스로에게물어볼질문
1. 재시작이 필요한가, 수동 작업이 요구되는가?
2. 일부 기능만 멈추는가, 서버 전체가 중단되는가?
3. 내가 설계한 요소가 예기치 못한 상황을 만나도 충분히 회복할 수 있는가?

### 복구수준
| 수준 | 설명 | 목표 |
|------|------|------|
| 수동 복구 | 사람이 개입하여 수동으로 복구 | 최소화해야 함 |
| 자동 재시작 | 프로세스/컨테이너 자동 재시작 | 기본 수준 |
| 부분 장애 격리 | 일부 기능만 멈추고 나머지는 정상 운영 | 권장 수준 |
| 자동 복구 | 시스템이 스스로 상태를 감지하고 복구 | 이상적 수준 |

## 재발방지(RootCause분석)

### 스스로에게물어볼질문
1. 현재 대처가 문제를 회피하는 것인가? **Root Cause**는 무엇인가?
2. 동일한 장애가 다시 발생하지 않을 수 있는가?
3. **사람에 의존하는가, 시스템에 의존하는가?**

### 참고사례
- SLASH24 - eBPF를 활용한 Root Cause 분석
- Pay Cluster: 사람 의존 → 시스템 의존으로 전환

### 재발방지프로세스
```
장애 발생 → 임시 대응 → Root Cause 분석 → 재발 방지 대책 수립 → 시스템에 반영 → 검증
```

## 운영자동화와지속적개선

- 성능 최적화/운영 자동화를 위해 지속적으로 노력하는가?
- **사업의 확장은 최전선에서 일어남**
- 지나간 일에 파묻히면 앞으로 나아갈 동력을 잃게 됨
- 과거 장애 대응에만 매몰되지 않고, 미래를 위한 자동화에 투자

## 장애 등급 분류

| 등급 | 정의 | 대응 SLA |
|------|------|---------|
| **P1** | 서비스 전체 중단 | 즉시 대응, 30분 내 1차 조치 |
| **P2** | 주요 기능 장애 (결제, 로그인 등) | 1시간 내 대응 |
| **P3** | 일부 사용자 영향 | 당일 내 대응 |
| **P4** | 성능 저하, 경미한 이슈 | 다음 스프린트에 처리 |

## 장애 대응 프로세스

```
1. 감지 (Grafana Alerting → Slack 자동 알림)
2. 영향 범위 파악 (TraceId로 요청 단위 로그+메트릭 연계 조회)
3. 판단: 롤백 vs 핫픽스
   - 데이터 정합성 문제 → 즉시 롤백
   - 단순 로직 버그 → 핫픽스
4. 근본 원인 분석 (Loki 로그 + Prometheus 메트릭 교차 분석)
5. 재발 방지 (알림 임계값 조정 + 테스트 케이스 추가 + 포스트모템)
```

## 포스트모템 (Blameless Postmortem)

### 작성 항목
1. **타임라인**: 발생 → 감지 → 대응 → 복구 시각
2. **근본 원인**: 왜 발생했는가 (5 Whys 기법)
3. **영향 범위**: 몇 명의 사용자, 어떤 기능, 얼마나 오래
4. **재발 방지 액션 아이템**: 담당자 + 기한 명시
5. **교훈**: 잘한 점 + 개선할 점

### 핵심 원칙
- **비난 없는 문화**: 사람이 아니라 시스템/프로세스의 문제를 찾는 것
- 포스트모템은 문서화하여 팀 전체에 공유 → 조직의 학습 자산

## 장애 사전 예방

| 방법 | 설명 |
|------|------|
| SLO 기반 알림 | 문제가 커지기 전에 감지 (Error rate, Slow SQL 등) |
| 카나리 배포 | 일부 트래픽만 새 버전으로 → 문제 시 빠른 롤백 |
| 로드 테스트 | k6, Artillery 등으로 병목 사전 발견 |
| 코드 리뷰 + 테스트 게이트 | PR 단계에서 품질 확보 |
| Circuit Breaker | 외부 의존성 장애 시 연쇄 장애 차단 |

## 면접포인트
- "장애 경험?" → 감지 → 대응 → Root Cause → 재발 방지까지 전체 사이클로 답변
- "사람 의존 vs 시스템 의존?" → 수동 대응을 자동화로 전환한 경험
- "운영 자동화 경험?" → 반복 작업을 시스템화하여 본업에 집중할 수 있도록 개선
- "장애 등급을 어떻게 나누나?" → P1~P4 등급별 대응 SLA 차등
- "포스트모템에 뭘 쓰나?" → 타임라인, 근본 원인, 영향 범위, 액션 아이템. 비난 없는 문화가 핵심

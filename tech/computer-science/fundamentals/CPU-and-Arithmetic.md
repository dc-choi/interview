# CPU와연산(논리회로,사칙연산)

## 논리게이트

CPU는 전자계산기이며, 모든 사칙연산은 논리 게이트의 조합으로 구현된다.

| 게이트 | 규칙 | 용도 |
|--------|------|------|
| AND | 둘 다 1이면 1 | 자리올림 판단 |
| OR | 둘 중 하나라도 1이면 1 | 조건 합성 |
| XOR | 서로 다르면 1 | 덧셈 (자리올림 제외) |
| NOT | 0→1, 1→0 | 반전, 뺄셈(보수) |

## 사칙연산

### 덧셈
- **XOR**: 자리올림 없는 합 계산
- **AND**: 자리올림 발생 여부 판단
- 이 둘을 조합한 것이 반가산기(Half Adder), 자리올림 입력까지 처리하면 전가산기(Full Adder)

### 뺄셈(2의보수)
1. NOT 연산으로 비트 반전
2. 반전된 수에 1을 더함
3. 원래 수와 덧셈 후 자리올림은 버림
- 이 방식으로 뺄셈을 덧셈 회로로 처리할 수 있음

### 곱셈(비트시프트)
- 왼쪽 시프트 1칸 = ×2, 2칸 = ×4
- 오른쪽 빈자리는 0으로 패딩
- 홀수 곱셈: 시프트 후 1을 더하는 방식

### 나눗셈(비트시프트)
- 오른쪽 시프트 1칸 = ÷2
- 왼쪽 빈자리는 0으로 패딩
- **0으로 나누기**: 몫을 구하는 과정이 무한 반복 → 프로세서가 멈출 수 있음

## CPU연산과정

```
메모리(RAM) → [주소로 접근] → 데이터 → CPU 레지스터에 저장
→ ALU(산술논리장치)에서 연산 → 결과를 메모리에 저장
```

1. 메모리에는 데이터에 접근할 수 있는 주소가 있음
2. 해당 주소에서 값을 가져와 CPU 레지스터에 저장
3. ALU에서 연산 수행
4. 연산 결과를 다시 메모리에 저장

### 32비트운영체제의한계
- 32비트 = 2^32 = 약 42억개의 주소 = **최대 4GB RAM**
- 메모리 용량이 커지면서 32비트 운영체제가 사라지는 이유
- 64비트 = 2^64 = 이론상 16EB까지 주소 지정 가능

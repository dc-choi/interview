# 디지털기초(진법,인코딩,이미지)

## 비트와진법

### 1비트
- 전기 스위치 1개. 전기가 흐르면 1, 아니면 0
- 4비트 = 16가지 경우의 수 (2^4)
- 외우기: 16, 8, 4, 2, 1

### 16진수
- 16진수 한 글자 = 4비트
- 10 이상: A(10), B(11), C(12), D(13), E(14), F(15)
- 예: `F4` = `1111 0100` (8비트)
- 거의 대부분 16진수 표기를 사용함

### 16진수사용사례
- **색상 표현**: RGB 각 8비트 = 256단계 (0~255), `#FF0000` = 빨강
- **하드웨어 주소**: MAC 주소 등
- **메모리 값**: 메모리 주소, 덤프

## 단위체계

| 단위 | 크기 |
|------|------|
| 1비트 | 표현의 최소 단위 |
| 1바이트 | 8비트, 기억 공간의 최소 단위, 영문자 1글자 |
| 1KB | 1,024바이트 |
| 1MB | 1,024KB |
| 1GB | 1,024MB |
| 1TB | 1,024GB |
| 1PB | 1,024TB |
| 1EB | 1,024PB |
| 1ZB | 1,024EB |
| 1YB | 1,024ZB |

- 2^10 = 1,024
- 한글은 2바이트 필요 (영문은 1바이트)

## 문자인코딩

### ASCII
- 미국 정부가 표준화한 코드 체계
- 7비트로 128개 문자 표현 (영문, 숫자, 특수문자)
- 회사마다 코드 체계가 달랐던 문제를 해결

### Unicode/UTF-8(보강)
- ASCII만으로는 전 세계 문자를 표현할 수 없음
- **Unicode**: 전 세계 모든 문자에 고유 번호를 부여한 표준
- **UTF-8**: Unicode를 가변 길이(1~4바이트)로 인코딩하는 방식
  - ASCII 호환 (영문 1바이트)
  - 한글 3바이트
  - 현재 웹의 사실상 표준

### Binary
- 숫자와 글자를 구별하지 않고 정보를 말할 때 Binary라고 함

## 이미지표현

### Pixel
- 모니터 화면의 점 하나 = 화소(Pixel)
- 여러 점을 모아 사진을 구성

### 색상깊이
| 비트 | 색상 수 | 용도 |
|------|---------|------|
| 8비트 | 256색 | GIF |
| 16비트 | 65,536색 | High Color |
| 24비트 | 16,777,216색 | RGB (R8 + G8 + B8) |
| 32비트 | + 투명도 | RGBA |

- 빛의 3원색(Red, Green, Blue)에 착안한 RGB

### 이미지형식
- **비트맵(BMP)**: 화소를 일일이 나열 → 용량이 큼
- **JPG**: 손실 압축, 사진에 적합
- **PNG**: 무손실 압축, 투명도 지원
- **GIF**: 256색, 애니메이션 지원

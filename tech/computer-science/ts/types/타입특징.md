# 타입 특징

TypeScript는 JS의 동적 타입 시스템 위에 **정적 타입 시스템**을 얹어 컴파일 단계에서 타입 오류를 미리 잡아낸다. 타입 시스템은 값이 "어떤 집합에 속하는지"를 검사하고, 개발자에게 의미 있는 컴파일 에러와 IDE 자동완성을 제공한다.

## 정적 타입 시스템의 가치
1. **런타임 오류 사전 차단** — 컴파일 단계에서 잘못된 호출·잘못된 속성 접근을 걸러낸다.
2. **코드 문서화** — 함수 시그니처가 곧 의도의 문서다. 팀원 간 협업 시 의도를 명시적으로 전달한다.
3. **IDE 경험** — 자동완성, 리팩토링, 타입 기반 이동 등이 타입 정보에 기반해 동작한다.

## 타입 계층과 집합론 관점
모든 타입은 "값들의 집합"으로 해석할 수 있다. 이 관점으로 보면 타입 계층은 **전체집합(`any`) → 부분집합들 → 공집합(`never`)** 구조를 이룬다. 기본적으로 **업캐스팅(상위 집합으로 대입)은 허용되고 다운캐스팅(하위 집합으로 대입)은 금지**되는데, 이는 부분집합의 값이 상위집합에도 속한다는 성질을 그대로 반영한 것이다.

자세한 타입 계층은 프로젝트에 포함된 `타입계층도.png` 이미지로 시각화되어 있다.

## 원시 타입 (Primitive)
값 하나로 표현되는 기본 타입이다.
- **`number`** — 정수·실수 모두 포함. NaN과 Infinity도 number.
- **`string`** — 문자열.
- **`boolean`** — true / false.
- **`null`** — 명시적 "없음".
- **`undefined`** — 선언만 되고 값이 할당되지 않은 상태.
- **리터럴 타입** — `"left" | "right"`처럼 값 자체를 타입으로 사용한다. Union과 결합해 상태(state)나 이벤트 종류를 표현할 때 강력하다.

코드 예시는 `types/primitive/` 디렉토리의 `number.mts`, `string.mts`, `boolean.mts`, `literal.mts`, `null.mts`, `undefined.mts` 참고.

## 복합 타입 (Object / Array / Tuple)
- **Object** — TS는 **구조적 타이핑(structural typing)** 을 따른다. "이름"이 같지 않아도 **필드 모양이 같으면 같은 타입으로 간주**된다. 선택적 속성(`?`)으로 옵셔널 필드를 표현할 수 있다.
- **Array** — 가변 길이, 원소 타입은 동일하다. `T[]` 또는 `Array<T>` 문법을 쓴다.
- **Tuple** — 고정 길이 배열로, 각 인덱스마다 타입이 다를 수 있다. React의 `useState` 반환값처럼 `[값, 세터]` 형태를 표현할 때 유용하다.

코드 예시: `types/object/object.mts`, `types/array/array.mts`, `types/type-index/typeIndex.mts`

## 특별한 타입 (any / unknown / void / never)
이 네 타입은 "값의 집합" 관점에서 극단에 있는 특별한 타입들이다.

- **`any`** — 전체집합. 타입 검사를 사실상 포기한다. 점진적 마이그레이션 중이거나 외부 동적 데이터를 임시로 받을 때만 사용해야 한다. 남용하면 TS를 쓰는 이유가 사라진다.
- **`unknown`** — "모든 값을 받을 수 있지만, 사용하려면 타입을 좁혀야 한다"는 **안전한 any**. API 응답이나 `JSON.parse` 결과처럼 타입을 모르는 값에 사용한다.
- **`void`** — 반환값이 없는 함수의 반환 타입.
- **`never`** — 공집합. "절대 반환하지 않는 함수"(에러를 throw하거나 무한 루프에 빠지는 함수)의 반환 타입이고, Union에서 다른 타입이 전부 소거된 뒤 남는 타입이기도 하다.

**`any`와 `unknown`의 차이는 의도의 표현에 있다.** `any`는 "검사 생략을 선언"하는 것이고, `unknown`은 "나중에 검사할 것을 약속"하는 것이다. 라이브러리 API 경계에서는 가능한 한 `unknown`을 쓰는 게 안전하다.

코드 예시: `types/any-unknown/any.mts`, `types/any-unknown/unknown.mts`, `types/void-never/void.mts`, `types/void-never/never.mts`

## 집합 연산 타입 (Union / Intersection)
집합론의 합집합·교집합을 타입 시스템으로 가져온 것이다.

- **Union (`|`)** — "A 또는 B 타입". `number | string`은 number **또는** string 값을 허용한다. 사용 시점에는 타입 좁히기로 어느 쪽인지 확정한 뒤에만 해당 타입의 메서드를 쓸 수 있다.
- **Intersection (`&`)** — "A이면서 동시에 B인 타입". 주로 객체 타입을 합쳐 확장할 때 사용한다. 원시 타입끼리 교집합을 내면 대부분 `never`가 된다(예: `string & number` = `never`).

## 타입 좁히기 (Type Narrowing)
Union 타입 값을 특정 타입으로 "좁혀서" 안전하게 사용하는 기법이다. 컴파일러는 조건문 블록 내부에서 값이 어떤 타입으로 좁혀졌는지 추적한다. 덕분에 Union 타입을 받아도 블록 안에서는 구체 타입처럼 사용할 수 있다.

주요 좁히기 수단:
- **`typeof`** — 원시 타입 구분 (`typeof x === "string"`)
- **`instanceof`** — 클래스 인스턴스 구분
- **`in` 연산자** — 객체 속성 존재 여부로 구분
- **사용자 정의 타입 가드(`x is T`)** — 복잡한 조건을 타입 술어(type predicate) 함수로 추출

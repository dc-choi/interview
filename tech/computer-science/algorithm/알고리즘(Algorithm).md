# Algorithm

알고리즘이란 어떤 문제를 해결하기 위한 확실한 방법이다. 자료구조에 따라 영향을 받는다.

---

## 시간복잡도

특정 알고리즘이 어떤 문제를 해결하는 데 걸리는 시간을 나타낸다.
사용자마다 컴퓨터 성능이 다르기 때문에 직접 시간을 재는 것이 아니라, 코드의 오래 걸리는 부분(주로 반복문)을 찾아 실행 시간을 예측한다.

### Big O 표기법
최악의 경우를 기준으로 표기한다.

| 표기 | 이름 | 설명 |
|---|---|---|
| O(1) | 상수 시간 | 입력 크기와 무관 |
| O(log n) | 로그 시간 | 이진 탐색 등 |
| O(n) | 선형 시간 | 단순 반복 |
| O(n log n) | 선형 로그 시간 | 병합/퀵 정렬 |
| O(n^2) | 이차 시간 | 이중 반복문 |
| O(2^n) | 지수 시간 | 피보나치(단순 재귀) 등 |
| O(n!) | 팩토리얼 시간 | 순열 등 |

Big O는 가장 성능에 영향을 끼치는 부분만 계산한다.
- `n^2 + 100` -> O(n^2)
- `3n^2 + 100n + 100` -> O(n^2)

## 재귀

어떠한 것을 정의할 때 자기 자신을 참조하는 것. 재귀적으로 정의된 함수를 재귀함수라고 한다.

### 기저 조건
재귀는 무조건 탈출(기저) 조건이 있어야 한다. 없으면 콜스택 메모리가 가득 차서 종료된다.

### 콜스택
함수가 호출되면서 올라가는 메모리 영역. 스택의 특성(FILO)을 그대로 가진다.

### 재귀 패턴
1. **단순 반복 실행**: 반복문 대비 이점 없음. 오히려 콜스택 공간을 더 차지해 성능 문제 발생
2. **하향식 계산**: 하위 문제를 이용해 상위 문제를 해결 (팩토리얼 등)
   - for문을 사용하는 상향식 계산과 반대
   - 보통 하향식 계산을 위해 재귀를 사용

### 문제 해결 팁
반복문으로 해결할 수 있는 것을 재귀로 하면 성능상 좋지 않지만, 복잡한 문제(하노이 탑 등)를 해결하는 데 유용하다.

## 정렬

### 버블 정렬
가장 쉽게 생각할 수 있는 정렬. 구현이 쉽지만 성능이 좋지 않다. O(n^2)
옆 자리 숫자와 비교하면서 자리를 바꾸는 것이 버블이 일어나는 것 같아서 버블 정렬이라 부른다.

### 선택 정렬
구현이 쉽지만 성능이 좋지 않다. O(n^2)

### 삽입 정렬
구현이 쉽지만 성능이 좋지 않다. O(n^2)

### 병합 정렬
분할정복(Divide and Conquer) 알고리즘. O(n log n)

해결하기 힘든 문제를 한 번에 해결하려 하지 말고, 해결하기 쉬울 정도로 문제를 쪼갠 다음 하나씩 해결하는 방식이다.

하나의 배열을 최소 단위까지 나눈 다음 각각 병합하면서 정렬한다.

- 장점: 성능이 좋다
- 단점: 이해와 구현이 어려움

### 퀵 정렬
분할정복 알고리즘. 평균 O(n log n), 최악 O(n^2)

최악은 잘 발생하지 않으며, 병합정렬보다 적은 비교와 적은 메모리를 사용하기 때문에 실제로 더 빠르다.

## 동적 프로그래밍 (DP)

### 메모이제이션 (Memoization)
재귀적 기법의 계산 결과를 저장해서 여러 번 계산하지 않는 기법이다.
적용하면 O(n) 성능을 가지게 된다.
속도를 위해 메모리 공간을 사용하는 트레이드오프가 있다 (캐시와 동일한 개념).

### 타뷸레이션 (Tabulation)
메모이제이션과 달리 하향식이 아닌 **상향식 접근**으로 계산에 필요한 모든 값을 테이블에 미리 저장한다.

### 선택 기준
- 분할정복 시 재귀가 더 직관적이라면 -> **메모이제이션**
- 상향식 접근이 직관적이라면 -> **타뷸레이션**

## P-NP

### 결정 문제
어떤 문제가 주어졌을 때 yes 또는 no로 대답할 수 있는 문제.
예: "10은 2로 나누었을 때 딱 떨어지는가?"

### 최적화 문제
어떤 상황이 주어지고 거기서 최적의 해를 찾는 문제.
예: "출발지에서 도착지로 가는 가장 짧은 거리를 구하시오."

대부분의 최적화 문제는 결정 문제로 바꿀 수 있다.

## 필수 알고리즘 체크리스트

### 기본 알고리즘
| 알고리즘 | 시간복잡도 | 비고 |
|---|---|---|
| 버블/선택/삽입 정렬 | O(n^2) | 기본 정렬 |
| 병합 정렬 (Merge Sort) | O(n log n) | 분할정복, 안정 정렬 |
| 퀵 정렬 (Quick Sort) | O(n log n) | 분할정복, 실전에서 가장 빠름 |

### 탐색 알고리즘
| 알고리즘 | 시간복잡도 | 비고 |
|---|---|---|
| 이진 탐색 (Binary Search) | O(log n) | 정렬된 배열 필수 |
| BFS | O(V+E) | 큐 사용, 최단 경로 |
| DFS | O(V+E) | 스택/재귀 사용 |

### 자주 쓰이는 패턴 & 알고리즘
| 알고리즘 | 비고 |
|---|---|
| 투 포인터 (Two Pointer) | 정렬된 배열에서 효율적 탐색 |
| 슬라이딩 윈도우 (Sliding Window) | 연속 부분 배열/문자열 |
| 그리디 (Greedy) | 매 순간 최적의 선택 |
| DP (Dynamic Programming) | 메모이제이션/타뷸레이션 |
| 다익스트라 (Dijkstra) | 최단 경로 |
| 플로이드-워셜 (Floyd-Warshall) | 모든 쌍 최단 경로 |

## 실제 백엔드 개발에서 활용

| 사례 | 활용 자료구조 & 알고리즘 |
|---|---|
| API Rate Limit | Trie, Sliding Window |
| 추천 시스템 | 그래프 탐색, BFS/DFS |
| 로그 검색 최적화 | 해시 테이블, 이진 탐색 |
| 데이터 캐싱 (LRU Cache) | HashMap + LinkedList |
| 배치 작업 최적화 | 우선순위 큐, 그리디 알고리즘 |
| DB 성능 최적화 | B+Tree, 인덱스 최적화 |
| 로드 밸런싱 | 해싱, 라운드 로빈 알고리즘 |
| 배송 경로 최적화 | 다익스트라, 플로이드-워셜 |

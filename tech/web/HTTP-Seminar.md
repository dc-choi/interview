# HTTP세미나

> 우아한형제들 김정환님 "퇴근길 밋업" 발표 정리

## HTTP가필요한이유

- 웹에서 가장 중요한 것은 브라우저
- 브라우저는 항상 HTML을 가지고 있지 않으므로 서버와 통신이 필요
- 서버와 브라우저 사이의 거리는 알 수 없음 → 이 사이에서 통신하기 위한 규약이 HTTP

## HTTP버전별진화

| 버전 | 연도 | 주요변화 |
|------|------|---------|
| 0.9 | 1993 | 텍스트(문자)만 주고받기 위해 설계 |
| 1.0 | 1996 | Method(GET, POST), Header, Body 도입. 캐싱 헤더 추가 |
| 1.1 | 1997 | PUT, DELETE 추가. Cache-Control 헤더 추가 |
| 2.0 | 2015 | 메시지를 스트림으로 전송 → 성능 향상 (멀티플렉싱) |
| 3.0 | 2022 | UDP 기반(QUIC) → 추가 성능 향상 |

- 2.0/3.0은 성능 향상이 목적이며, 몰라도 애플리케이션 구현에는 문제없음
- 3.0은 아직 버그 이슈로 사용을 제한하는 회사도 있음

## 핵심기본요소

- **URL**: 리소스의 위치
- **Header**: 메타데이터 (캐싱, 인증, 콘텐츠 타입 등)
- **Body**: 실제 데이터
- **Status Code**: 응답 상태

## 관련용어

| 용어 | 설명 |
|------|------|
| HTML | 텍스트 파일을 구조화한 문서 |
| HTTP | HTML 문서를 전송하기 위한 규약 |
| httpd | HTTP를 준수하며 웹 문서를 제공하는 프로세스 |
| WWW | World Wide Web |

## 현업에서의HTTP지식

### 알아두면도움되는것
- **CORS**: 원인과 해결 방법을 모르면 실무에서 문제가 됨
- **리다이렉트**: 동작 원리 이해
- **캐싱**: Cache-Control, ETag 등 캐시 전략

### 학습방법
- 직접 서버와 클라이언트를 구현해보는 것이 효과적 (Node.js 등)
- 프론트엔드는 깊게 몰라도 되지만, 백엔드는 기본 이해 필수

## QnA발췌

**네트워킹 지식이 현업에서 도움이 되었나?**
→ 다른 회사에서 어떻게 하는지 알면 지식의 폭과 자신의 위치를 파악할 수 있음

**다른 직군과 협업하려면?**
→ 비개발자에게 기술 용어는 통하지 않음. 공통된 언어를 찾아야 함

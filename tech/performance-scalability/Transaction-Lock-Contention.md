# 트랜잭션경합과Lock문제

> 출처: [[Elegant-OOP-Design|우아한 객체지향]]

## 문제:객체참조가만드는트랜잭션확대

객체 참조(연관관계)는 모든 것을 연결시켜 하나의 트랜잭션으로 묶이게 만든다.

### 예시:배달완료
배달 완료 처리 시 가게·주문·배달이 객체 참조로 연결되어 있으면:
- 3개 도메인이 **하나의 트랜잭션**으로 묶임
- 각 도메인의 **변경 빈도가 다름** (주문은 자주, 가게 정보는 드물게)
- 변경 빈도가 낮은 도메인까지 Lock이 걸림 → **Lock 경합으로 성능 저하**

### 추가문제
- **N+1 문제**: 어디까지 조회할 것인가? (Lazy Loading의 함정)
- **결합도**: 참조로 연결된 모든 객체를 따라가서 기능을 구현하게 됨

## 해결방법

### 1.Aggregate경계분리
- 함께 변경되는 객체만 하나의 트랜잭션으로 묶기
- 경계 안: 객체 참조 / 경계 밖: **ID + Repository**
- 트랜잭션 범위가 줄어들어 Lock 경합 감소

### 2.도메인이벤트
- Aggregate 간 통신을 이벤트로 분리
- 주문 완료 → 이벤트 발행 → 가게·배달이 각자 처리
- 각 도메인이 **독립적인 트랜잭션**으로 동작

### 3.시스템분리
- 도메인별로 패키지를 잘게 쪼개면 시스템 단위 분리 가능
- 시스템 간에는 **메시지 브로커**를 통한 이벤트 기반 통신

## 면접포인트
- "트랜잭션 범위를 어떻게 결정하나?" → Aggregate 경계 = 트랜잭션 경계
- "Lock 경합을 어떻게 해결하나?" → 경계 분리, 이벤트 기반 비동기 처리
- "MSA에서 트랜잭션 관리?" → Saga 패턴, 이벤트 기반 최종 일관성

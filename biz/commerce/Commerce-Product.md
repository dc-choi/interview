# 커머스 상품(Product) 도메인

## 상품의 정의

상거래를 목적으로 파는 물건, 매매를 목적으로 한 재화. 커머스 시스템에서 상품은 단일 엔티티가 아니라 가격, 재고, 물류, 전시가 만나는 접점이다.

## 상품 정보의 4분류

| 분류 | 포함 정보 | 주 사용처 |
|---|---|---|
| 기본 정보 | 상품명, 브랜드명, 품번, 판매자 정보, 카테고리, 옵션(컬러, 사이즈, 재질) | 상품 식별, 관리 |
| 주문을 위한 정보 | 가격 정책과 할인, 재고(수량, 품절 여부), 배송 정책 | 주문, 결제 흐름 |
| 탐색을 위한 정보 | 상품 이미지와 상세정보, 후기와 리뷰, 상품 문의, 연결 상품(추천) | 검색, 전시, 전환 |
| 표시 의무 정보 | 상품 고시 정보, 배송과 교환, 반품 안내, 기타 혜택 안내 | 법적 요건, CS |

같은 상품이라도 정보의 성격(변경 주기, 책임 주체, 법적 요건)이 달라 시스템에서도 분리 관리되는 경우가 많다.

## 상품상세 페이지: 복잡성의 구조

대다수 고객이 검색과 가격 비교를 거쳐 상품상세로 바로 진입하므로, 서비스의 실질적 첫인상은 메인이 아니라 **상품상세 페이지**다. 한국 커머스의 상세 페이지가 복잡해 보이는 것은 미적 취향의 문제가 아니라 **마케팅 요구와 법적 규제가 누적된 결과**이고, 기획의 미션은 둘을 다 수용하면서 트렌디하되 지나치게 낯설지 않은 사용성을 지키는 것이다.

| 영역 | 들어가는 것 | 복잡해지는 이유 |
|---|---|---|
| 가격 | 현재가, 종전가, 할인가 다중 노출 + 결제수단별 할인(제휴카드 청구할인) + 행사 쿠폰 + 배송비 | 이중가 표시 규제([[Commerce-Pricing\|가격 도메인]]) + 배송비에 민감한 이용자 + 혜택 총량을 보여줘야 전환되는 시장 |
| 법적 고지 | 상품정보 제공고시(공정위 품목별 40여 항목), 교환반품 안내 | 고객 이용률은 낮지만 분쟁 대비 의무 표기 — 회피 불가 영역 |
| 상품평 | 텍스트 리뷰, 별점 | 식품 등 효능 과장 우려 카테고리는 텍스트 표현이 제한돼 별점이 주된 신뢰 전달 수단 |
| 연결고리(추천) | 동일 브랜드와 카테고리 인기상품, 함께 구매한 상품, 이전 조회 상품 | 클릭수와 매출 전환 효율이 높아 마케팅이 포기하지 않는 영역 |

백엔드 관점: 상세 페이지는 단일 화면이지만 가격, 재고, 리뷰, 추천, 고시 정보 등 변경 주기가 전혀 다른 도메인의 조합이다 — 고시 정보처럼 정적인 데이터와 가격, 재고처럼 실시간성이 필요한 데이터의 캐싱 전략을 분리하고, 화면 조립은 API 애그리게이션 계층의 문제가 된다.

## 카테고리의 두 가지 유형

| 유형 | 기준 | 특징 | 예시 |
|---|---|---|---|
| 품목 카테고리 | 상품의 종류와 용도 | 상위 → 하위 계층 구조, 직관적이고 체계적, 변경이 드묾 | 남성용 의류 > 상의 > 긴소매 |
| 전시 카테고리 | 고객 편의, 머천다이징 | 시즌, 트렌드, 키워드 기반으로 수시 변경 | 여름옷, 봄 신상 |

품목 카테고리는 안정적인 분류 체계이고 전시 카테고리는 마케팅 도구다. 변경 주기가 전혀 다르므로 시스템 설계에서도 둘을 분리하고, 상품 하나가 여러 전시 카테고리에 동시에 걸리도록(N:M) 모델링하는 것이 표준이다.

### 카테고리 분류 기준

상품 종류와 용도, 성별과 연령과 취향, 브랜드와 제조사, 가격대.

### 기획전: 전시의 단위 매장

특정 테마를 기준으로 상품을 모아놓은 매장 페이지. 상단 배너와 타이틀로 테마를 표현하고 하단에 소타이틀 단위로 상품을 그룹핑한다. 전시 카테고리가 상시 운영되는 분류라면 기획전은 기간 한정의 단위 매장이다.

| 구분 | 기획전 | 검색결과, 카테고리 |
|---|---|---|
| 규모 | 테마 단위의 작은 덩어리 | 광범위 |
| 고객 심리 | 호기심 기반 탐색 | 필요성 기반 탐색 |
| 판매자 의도 | 강하게 반영 (배치, 그룹핑을 사람이 결정) | 약함 (랭킹, 정렬 로직이 결정) |

구성 공식은 높은 할인율의 **주연 상품**으로 유입을 만들고 판매자의 **조연 상품**을 함께 배치해 객단가와 광고 효과를 키우는 것. 플랫폼 입장에서는 운영 비용을 절감하면서 판매자 광고비까지 확보하는 수익 구조다. 백엔드 관점에서 기획전은 전시 도메인의 대표 엔티티다 — 상품과의 N:M 매핑, 노출 기간, 배너와 그룹 구조를 가지며, 변경 주기가 빠른 전시 영역이 상품 원본과 분리된 도메인으로 설계되는 이유가 여기 있다.

### 키워드, 태그 정책: 통제 vs 자유

분류 체계를 작성자나 판매자가 직접 붙이는 키워드로 보강할 때의 트레이드오프:

| 정책 | 장점 | 한계 |
|---|---|---|
| 통제 어휘 (규정된 키워드 풀에서 N개 선택) | 키워드 기반 큐레이션, 분류 자동화가 쉽고 스팸성 다량 태그를 차단 | 정확히 표현할 키워드가 풀에 없으면 작성자는 표현을 못 하고 운영팀은 그 니즈를 볼 수 없음 |
| 자유 태그 (폭소노미) | 표현력과 세밀한 분류, 사용자 니즈가 태그 데이터로 드러남 | 동의어 난립, 스팸, 자동 분류 어려움 |

절충은 통제 어휘를 기본으로 두고 키워드 제안 채널(VoC 수집)이나 보조 자유 태그를 더하는 하이브리드. 선택 개수 제한(3개 등)은 어뷰징 방지 장치다.

## 상품평과 Q&A

타인의 말은 구매 결정의 마지막 관문이다 — 가격 비교를 끝내고 들어온 고객이 결제 직전에 확인하는 것이 상품평과 Q&A이고, 이 시점의 신뢰가 전환을 가른다. 둘은 작성 시점과 자격으로 구분된다: **상품평은 구매 후**(구매자 확인 필수), **Q&A는 구매 전**(누구나 작성).

설계 포인트 세 가지:

1. **구매자 확인** — 구매확정 단계를 거친 고객만 작성하게 해 신뢰를 확보한다. 최소 글자 수 기준으로 프리미엄 리뷰와 일반 리뷰를 구분하기도 한다.
2. **좋은 리뷰 선별** — 판매자 수기 선정(신뢰 낮음)에서 고객 추천(도움이 돼요)으로 진화했다. 추천 수가 리뷰 정렬 기준이 된다.
3. **작성 부담 완화** — 상품군별 사전 정의 태그를 클릭만으로 선택하게 하면, 작성 마찰을 줄이면서 처음부터 구조화된 통계 데이터가 쌓인다.

백엔드 관점: 리뷰 작성 자격은 주문 상태(구매확정)와의 검증 문제이고, 태그형 리뷰는 자유 텍스트와 달리 구조화 데이터라 집계 파이프라인이 단순해진다. 추천 기반 정렬은 어뷰징(자기 추천, 계정 동원) 방지 설계가 따라온다. 고객이 외부 SNS 대신 자사 플랫폼에 쓰게 만드는 동기 문제는 [[Community-Strategy|커뮤니티 전략]]의 UGC 동기 참조.

## 상품과 물류의 연결

상품의 물성(크기, 무게, 부피, 보관 방법, 안전 규정)은 물류 비용과 처리 방식을 결정한다. 직매입, PB상품, 물류 기반 커머스에서는 판매자가 등록한 **스토어 상품**과 물류, 카탈로그 기준의 **표준상품**을 구분해 이해해야 한다 (오픈마켓의 동일 상품 매칭, 가격 비교가 이 표준상품 개념 위에서 동작한다).

### 오프라인 상품 데이터의 한계: 바코드와 SKU

마트는 고객이 무엇을 샀는지는 알아도 정확히 어떤 단위로 샀는지 모른다 — 바코드 체계가 원인이다.

| 구분 | 온라인 (SKU 체계) | 오프라인 (바코드 체계) |
|---|---|---|
| 가격 변경 | 상품 데이터에서 가격만 수정 | 바코드 재발행 (식별자가 가격표를 겸함) |
| 판매 단위 | 명확한 품목, 가격, 사이즈 = 1개 코드 | 소분 단위(반근, 한근)마다 별도 바코드, 떨이는 기존 바코드 위에 덧붙임 |
| 입력 정합성 | 코드 = 상품 식별자 | 가격만 같으면 다른 상품 바코드로 스캔하는 관행 (부위 바꿔치기, 번들 1회 스캔) |

판매 데이터와 실재고가 어긋나는 구조라, 오프라인 마트의 온라인 장보기 진출에서 결품이 늘어나는 근본 원인이 된다. 해결은 두 갈래인데 모두 비싸다 — 신선식품 전처리(사전 소분으로 판매 단위 표준화, 인건비)와 온오프 통합 재고 시스템(레거시 POS 교체). 처음부터 데이터 기반으로 설계한 온라인 출신 사업자가 장보기 시장에서 유리한 구조적 이유다.

백엔드 관점: 식별자가 식별 외 역할(가격표)을 겸하면 데이터 품질이 무너진다 — 식별자에 의미를 싣지 말라는 설계 원칙의 반면교사. 그리고 재고 정합성은 입력 시점(스캔 관행)의 정확성을 넘어설 수 없다 — 시스템이 아무리 좋아도 garbage in, garbage out.

### 물류의 경제학: 규모의 경제와 풀필먼트

물류창고를 늘리는 것 자체는 규모의 경제가 아니다 — **규모의 경제는 물동량(거래량)이 커져 단위 비용이 떨어지는 것**이고, 거래량 성장 없이 인프라만 늘리면 고정비 부담만 커진다. 신선식품 중심이면 함정이 더 깊다: 직매입 상품의 폐기 리스크, 콜드체인의 권역 한정 운영 때문에 거래량이 정체된 상태의 창고 추가는 성장 문제를 물류 투자로 풀려는 원인과 수단의 혼동이 된다.

물류가 비용 요소를 넘어 수익원이 되는 경로가 풀필먼트다:

| 구분 | 물류창고 | 풀필먼트 센터 |
|---|---|---|
| 역할 | 자사 상품 보관과 출고 | 제3자(셀러) 물류 대행 — 보관, 포장, 배송 |
| 수익 | 없음 (비용 센터) | 건당 이용료 — 서비스형 수익 모델 |
| 규모의 경제 | 자사 거래량에 종속 | 외부 물동량을 받아 단위 비용을 함께 낮춤 |

자체 배송망의 거대한 고정비를 외부 셀러의 물동량으로 채우는 구조(FBA형)가 물류 내재화의 손익을 뒤집는 열쇠다 — 물류 내재화(경험 통제) → 구조적 적자 → 풀필먼트 개방(수익화)의 순서로 진행돼 온 흐름 ([[Commerce-Korea-History-2014-2017|한국 이커머스 역사 2014-2017]]).

## 상품 배송 서비스

고객의 배송 선택 기준은 세 가지 — 언제 출고되고 배송되는가, 배송비는 얼마인가, 안전하게 오는가(필수 전제).

### 출고 방식

- 일반 출고, 지정일(예약일) 출고, 순차 출고, 주문 후 제작

### 배송 수단

| 수단 | 용도 |
|---|---|
| 택배 | 일반적인 배송 |
| 화물 | 대형 물품 |
| 설치 배송 | 대형 가구, 가전 |
| 냉장, 냉동 | 온도 유지 필요 상품 |
| 이륜차 | 단거리 신속 배송 |
| 새벽 배송 | 신선식품 |
| 시간대 지정 | 특정 시간대 수령 |
| 해외 배송 | 국제 배송 |

### 배송비 정책

무료배송, 조건부 무료배송(N원 이상 구매 시), 고정 배송비, 구간별 배송비.

### 배송 정책을 구성하는 정보

- 출고지, 반품지 정보
- 배송비 정책
- 출고 관련: 주문 마감 시간, 휴무 설정, 출고와 발송 정책 구분
- 배송 관련: 배송 수단, 배송사(운송사), 배송 권역

## 면접 체크포인트

- 상품상세의 복잡성은 규제와 마케팅의 산물 — 단순화 제안을 할 때는 어느 요소가 의무(고시, 이중가)이고 어느 요소가 선택(추천, 행사)인지 구분하는 것이 먼저다.
- 품목 카테고리와 전시 카테고리를 왜 분리 설계하는가 — 변경 주기와 책임 주체(MD vs 카탈로그 관리)가 다르다.
- 옵션과 재고를 어떻게 모델링하는가 — Unit-Option-Stock 구조는 [[Ecommerce-Shopping-Mall-ERD|이커머스 쇼핑몰 ERD]] 참조.
- 상품 정보 변경이 기존 주문에 영향을 주면 안 되는 이유 — 주문 시점 스냅샷 패턴.
- 오픈마켓에서 동일 상품을 어떻게 묶는가 — 스토어 상품 vs 표준상품(카탈로그) 매칭.
- 온오프 통합(옴니채널) 재고 시스템이 어려운 이유 — 기술이 아니라 오프라인 입력 품질(바코드 관행)과 판매 단위 비표준이 병목. 재고 뻥튀기([[Commerce-Korea-Market|한국 이커머스 시장의 특수성]])가 입점 구조의 정합성 문제라면 이쪽은 입력 품질의 정합성 문제.
- 규모의 경제를 말할 때는 무엇의 단위 비용이 어떤 물량 축으로 떨어지는지 명시한다 — 인프라 확장과 규모의 경제는 다른 것. 물류 투자의 손익 전환점은 외부 물동량 수용(풀필먼트 개방) 여부다.

# D2C와 브랜드 데이터 전략

플랫폼(이커머스) 종속이 심해질 때 브랜드사(제조, 공급사)가 고객 접점과 데이터 주도권을 되찾는 전략. 입점 구조에서 브랜드가 자기 고객을 볼 수 없는 문제([[Commerce-Member|커머스 회원 도메인]]의 입점 데이터 비대칭)가 출발점이다.

## 플랫폼화가 브랜드에 만드는 4가지 위협

| 위협 | 내용 |
|---|---|
| 노출 기회 상실 | 검색은 브랜드를 여러 번 노출하지만, AI 추천(채팅, 음성 인터페이스)은 단 하나의 최적 상품을 전제 — 1등이 아니면 비교 대상조차 되지 못함 |
| 신뢰 대상의 이동 | 브랜드는 장사꾼, 데이터 기반 추천은 전문가로 인식 — 신뢰가 브랜드에서 추천 시스템으로 이동. 가성비 판단이 어려운 카테고리(가전)에서만 브랜드 신뢰가 우위 |
| 유통 파트너의 교체 | 기술 플랫폼(포털, 통신)이 유통에 진입 — 브랜드 배려 없이 고객 중심으로만 구성되고, 물건 빼기식 협상력이 무력화 |
| 마케팅 검증 불가 | 판매 데이터를 플랫폼이 독점해 캠페인 효과 측정이 불가 — 플랫폼 광고에 의존하게 되고 수수료와 광고비를 이중 부담 |

## 플랫폼의 약점과 브랜드의 대응

| 플랫폼의 약점 | 브랜드의 대응 |
|---|---|
| 추천의 대세가 없다 — 모두 초기 시도 단계 | 자체 추천과 챗봇 구축 — 클라우드 AI, 메신저 채널로 진입 장벽이 낮아짐 |
| 상품을 잘 모른다 — AI 3요소(컴퓨팅 파워, 알고리즘, 레이블 데이터) 중 레이블 데이터만 미해결 | 레이블화된 상품 데이터 선점 + 비기능적 정보(어울림, 사용법)의 전문가 서비스 |
| 표준화된 상품만 다룬다 | 커스터마이징, 규격화 불가능한 상품 (주문형 개인화 생산) |
| 배송 후에는 배제된다 | 판매 후 사용 데이터는 브랜드만 수집 가능 — IoT 탑재로 사용 패턴 기반 재구매 추천 |
| 플랫폼은 콘텐츠가 좌우한다 | 구매로 이어지는 전문 콘텐츠(트레이닝 가이드, 컨설팅)로 유입과 락인 |

기능적 상품 정보는 가격 비교 크롤링으로 누구나 모으지만, 이 옷이 후드티에 어울리는가 같은 **비기능적 정보는 브랜드만 답할 수 있다** — AI 추천 품질이 레이블 데이터에 묶여 있는 한 이것이 브랜드의 데이터 해자다.

## 실행 과제: 데이터 3축

1. **상품 데이터를 만들자** — 레이블화된 상품 정보 축적. 판단 가능한 데이터가 많은 상품이 추천의 우선순위를 차지한다.
2. **주문 데이터를 되찾자** — 자사몰 등 직접 판매 채널(D2C)로 구매 데이터 확보.
3. **사용 데이터로 대화하자** — 배송 후 IoT와 앱 기반 사용 데이터로 고객 관계 유지. 이커머스가 구조적으로 못 들어오는 영역.

상품, 주문, 사용의 3축에서 플랫폼이 가져간 것(주문)과 구조적으로 못 가져가는 것(비기능 상품 정보, 사용 데이터)을 구분하는 프레임이다.

## 면접 체크포인트

- 플랫폼 종속 리스크 질문에 데이터 3축(상품, 주문, 사용)으로 분해해 답하면 구조적이다 — 플랫폼 측의 수집 전략([[Payment-Service|결제 서비스]]의 데이터 시야)과 거울상.
- IoT와 자사 앱의 가치는 기능이 아니라 판매 후 데이터 접점 — 이커머스가 배제되는 유일한 구간이라는 관점.
- 커스터마이징 상품은 취향 대응이 아니라 추천, 비교가 불가능한 상품을 만드는 탈플랫폼 전략이기도 하다.

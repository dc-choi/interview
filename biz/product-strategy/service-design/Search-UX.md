# 검색 UX 설계

검색에는 두 모드가 있다 — 특정 대상을 찾는 **목적형 검색**과, 무엇을 찾을지 자체가 불명확한 **탐색형 검색**(트렌드, 영감, 요즘 인기 있는 것). SNS와 커머스의 검색은 탐색형 비중이 높아서, 검색창만 던져 놓으면 사용자에게 검색어 선택의 부담이 전가된다.

## 사용자 여정으로 본 불편 지점

검색 여정 4단계마다 전형적인 불편이 있다:

1. **검색어 입력** — 어떤 키워드로 검색해야 할지 힌트가 없다 (검색어 부담).
2. **결과 보기** — 광고성 콘텐츠가 상위를 점유해 신뢰가 깎이고, 정렬 옵션이 없다.
3. **좁히기** — 비슷한 태그가 난립해 무엇을 골라야 할지 또 부담.
4. **상세 진입** — 찾은 계정, 카테고리 안에서 다시 찾을 수단(스코프 검색)이 없다.

## 설계 요소

| 요소 | 푸는 문제 | 패턴 |
|---|---|---|
| 입력 보조 | 검색어 부담 | 추천 검색어, 인기 검색어 리스트, 자동완성 |
| 정렬 | 관련도 단일 정렬의 한계 | 추천순 기본 + 저장순, 최신순 옵션 |
| 필터 | 결과 과다 | 도메인 특화 속성 필터 (패션이면 소재, 무늬, 디테일 단위) |
| 신뢰도 | 광고성 콘텐츠의 결과 점유 | 광고 표시, 광고 제외 필터 |
| 유사 묶음 | 비슷한 태그 난립 | 키워드가 포함된 태그들의 통합 표시 |
| 스코프 검색 | 전역 검색만 존재 | 계정 내, 카테고리 내 검색 |

## 백엔드 관점

- **정렬 옵션 추가는 UI 작업이 아니라 인덱스 설계 문제다** — 저장 많은 순 정렬은 집계 값의 비정규화 또는 별도 인덱스가 필요하고, 갱신 빈도와 정합성 전략까지 따라온다.
- 추천, 인기 검색어는 **검색 로그 집계 파이프라인**의 산출물이다 — 기능 한 줄 뒤에 수집, 집계, 어뷰징 필터링이 있다.
- 스코프 검색은 텍스트 검색과 필터 조건의 결합 — 검색 엔진 쿼리 설계와 인덱스 매핑이 전제.
- 광고 제외 필터는 콘텐츠에 광고 여부 시그널(분류 데이터)이 있어야 가능하다 — 필터는 데이터 모델의 함수.

## 면접 체크포인트

- 검색 개선 질문에는 여정 4단계(입력 → 결과 → 좁히기 → 스코프)로 분해해 답한다.
- 기획의 정렬, 필터 요구를 받았을 때 인덱스와 집계 비용을 함께 말할 수 있어야 한다 — 기능 난이도가 아니라 데이터 준비 난이도가 본체인 경우가 많다.
- 탐색형 검색 비중이 높은 서비스에서 검색어 부담을 줄이는 장치(추천, 인기, 태그 통합)는 검색 전환율 지표와 직결된다.

# 사용자 가이드 제공 패턴

사용자에게 서비스를 이렇게 이용하라는 메시지를 전달하는 콘텐츠와 그 배치 방식. 목적은 사용자의 행동을 기획 의도대로 유도하는 것이고, 기준은 하나다 — **적재적소에 배치되어 실제로 도움이 되는가.** 이상적 구성은 항상 접근 가능한 기본 가이드(도움말)와 사용자의 현재 과업에 맞춘 맞춤형 가이드의 병행.

## 가이드가 필요한 3가지 상황

| 상황 | 목적 | 예 |
|---|---|---|
| 활용 극대화 | 신기능, 숨은 기능의 사용 유도 | 새 에디터 기능 홍보 |
| 사용자 보호 | 위험 상황 사전 차단 | 거래 사기 예방 수칙, 여행 중 긴급 연락처 |
| 악용 방지 | 부정 사용 사례 예방 | 광고성 콘텐츠 금지 수칙 고지 |

## 5가지 제공 방식

| 방식 | 특징 | 유의점 |
|---|---|---|
| 채팅 기반 안내 | 과업(가입, 글 등록) 완료 시점에 트리거되는 메시지 + FAQ 연결 버튼 | 시점이 생명 — 과업 직후가 골든타임 |
| 이벤트형 | 리워드(포인트, 경품)로 기능 사용을 유도 | 단기 효과 — 이벤트 종료 후 활용률 회귀 |
| 배너, 버튼 | 고정 위치의 도움말 진입점 | 사용자가 위치를 학습하게 됨. 복잡한 협업툴에선 필수 |
| 목록형 콘텐츠 | 가이드를 실제 콘텐츠와 같은 UI로 목록에 삽입 | 이질감 없이 노출되지만 콘텐츠로 오인될 수 있음 |
| 툴팁, 레이어 | 가볍고 빠른 인라인 안내 | 툴팁은 신기능 학습용, 레이어는 필수 정보(작성 수칙, 긴급번호) 강조용 |

## 설계 포인트

- **과업 트리거형이 가장 강하다** — 사용자가 막 그 일을 끝낸(또는 시작한) 맥락에서 도착하는 안내가 읽힌다. 시스템 관점에서는 사용자 행동 이벤트에 메시지 발송을 거는 트리거 설계.
- **리워드 이벤트는 인지용, 습관은 못 만든다** — 기능 인지를 빠르게 끌어올리는 데는 좋지만 지속 사용은 기능 자체의 가치가 만든다 ([[Gamification|게이미피케이션]]의 보상 설계와 같은 한계).
- **보호와 악용 방지 가이드는 노출 강제력이 필요** — 선택적 도움말이 아니라 레이어로 과업 흐름 안에 끼워 넣는다.

## 면접 체크포인트

- 온보딩 활성화 질문에 과업 트리거형 가이드(가입 직후, 첫 등록 직후 메시지)를 액션으로 제안할 수 있다 — 활성화 분기점 지표와 묶으면 강하다.
- 백엔드 관점: 과업 트리거 안내는 행동 이벤트 기반 메시징 시스템(이벤트 발행 → 조건 매칭 → 발송)으로 구현된다.
- 악용 방지 가이드는 어뷰징 정책의 첫 방어선이다 — 고지 없이 제재하면 CS 분쟁이 된다.

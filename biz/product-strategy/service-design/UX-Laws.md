# UX 심리학 10가지 법칙 (Laws of UX)

인지 심리학 연구를 UI 설계 원칙으로 정리한 법칙 모음. 이론 암기로는 실무에서 떠오르지 않는다 — 구체적 서비스 사례에 대입해 체득해야 설계할 때 무의식적인 감으로 작동한다.

## 10가지 법칙

| 법칙 | 정의 | 적용 예 |
|---|---|---|
| 제이콥의 법칙 | 사용자는 다른 서비스에서 익힌 UI 패턴을 기대한다 | 상품 상세를 일반 커머스와 같은 레이아웃으로 — 학습 비용 제거 |
| 피츠의 법칙 | 타겟 도달 시간은 거리와 크기의 함수 | 참여 기능(댓글, 작성)을 엄지가 닿는 중하단에 배치 |
| 힉의 법칙 | 선택지가 많을수록 결정 시간이 급증 | 열린 질문 대신 사전 정의된 선택지 제공 (거래 후기) |
| 밀러의 법칙 | 작업 기억은 7±2 청크 — 그루핑으로 부하 감소 | 판매내역, 구매내역, 관심목록 탭 분류 |
| 포스텔의 법칙 | 받는 입력엔 관대하게, 내보내는 출력은 엄격하게 | 가격 입력을 숫자 키패드로 제한해 쓰레기 입력 원천 차단 |
| 피크엔드 법칙 | 경험 평가는 절정과 마지막 순간이 좌우 | 작성 완료 시 뱃지 보상, 거래 후 상호 후기로 끝맛 설계 |
| 심미적 사용성 효과 | 아름다운 디자인은 더 쓰기 쉽다고 인지된다 | 오류 페이지에 캐릭터를 넣어 부정 경험을 완화 |
| 폰 레스토프 효과 | 비슷한 것들 사이 다른 하나가 기억된다 | 핵심 액션에만 포인트 컬러 — [[Visual-Hierarchy\|시각적 위계]] |
| 테슬러의 법칙 | 시스템의 복잡성 총량은 줄일 수 없다 — 누가 감당할지만 정할 수 있다 | 동네 범위를 직접 고르게 하지 않고 슬라이더로 — 복잡성을 시스템이 흡수 |
| 도허티 임계 | 0.4초 이내 반응이어야 사용자 생산성과 몰입 유지 | 스켈레톤 로더로 진행 중임을 먼저 시각화해 체감 속도 확보 |

## 백엔드 관점의 연결

- **도허티 임계(0.4초)는 API 응답 시간 버짓의 근거다** — p95 레이턴시 목표를 정할 때 사용자 몰입 임계가 출발점이 된다. 응답이 그보다 늦으면 스켈레톤, 낙관적 업데이트 같은 체감 속도 설계가 프론트와 백엔드의 공동 과제가 된다.
- **포스텔의 법칙의 원전은 TCP 견고성 원칙** — 받을 땐 관대하게, 보낼 땐 엄격하게. API 설계에서도 같은 이름으로 쓰인다 (관대한 파싱, 엄격한 응답 스키마). UX에서는 입력 단계 제약으로 잘못된 데이터를 원천 차단해 시스템 견고성을 지키는 형태로 적용된다 — 입력 제약은 곧 데이터 무결성과 검색 필터의 정상 동작 보장.
- **테슬러의 법칙은 복잡성 보존 법칙** — 사용자에게 단순하게 보이려면 그 복잡성을 서버와 시스템이 흡수해야 한다. [[Service-Design-Principles|GOV.UK 단순성 원칙]](복잡성은 시스템 뒤로)과 같은 말.

## 면접 체크포인트

- 성능 목표 질문에 도허티 임계를 근거로 — 왜 응답 시간 목표가 그 수치인지 사용자 몰입 임계로 설명할 수 있다.
- 입력 검증 설계를 포스텔의 법칙으로 — 프론트 입력 제약(1차)과 서버 검증(최종 방어선)의 역할 분담.
- 기능 단순화 요구에 테슬러의 법칙으로 — 복잡성은 사라지지 않고 이동할 뿐이므로, 누가(사용자 vs 시스템) 감당할지의 의사결정임을 짚는다.

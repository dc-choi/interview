# 비즈니스 모델 & 수익 구조

## Business Model Canvas (BMC)

Alexander Osterwalder가 2005년에 제안한 비즈니스 모델 설계 도구. 9개 블록으로 구성된다.

| 블록 | 질문 |
|---|---|
| **고객 세그먼트** | 누구를 위한 가치를 만드는가? |
| **가치 제안** | 어떤 문제를 해결하는가? |
| **채널** | 고객에게 어떻게 도달하는가? |
| **고객 관계** | 고객과 어떤 관계를 맺는가? |
| **수익원** | 어떻게 돈을 버는가? |
| **핵심 자원** | 가치 제안에 필요한 자산은? |
| **핵심 활동** | 가치를 전달하기 위해 무엇을 하는가? |
| **핵심 파트너** | 누구와 협력하는가? |
| **비용 구조** | 비용은 어디서 발생하는가? |

## 수익 모델 유형

비즈니스 모델 = 서비스 모델(무엇을 제공하는가) + 수익 모델(어떻게 돈을 받는가)의 합이다. 검색 서비스를 제공하며 광고로 버는 식으로 둘은 분리된다.

| 모델 | 설명 | 예시 |
|---|---|---|
| 구독 (Subscription) | 정기 과금 | Netflix, SaaS |
| 프리미엄 (Freemium) | 무과금 핵심 + 유료 부가 | Slack, Spotify |
| 마켓플레이스 | 거래 중개 수수료 | 배달의민족, 에어비앤비 |
| 광고 | 사용자 주의를 광고주에게 판매 | 구글, 인스타그램 |
| 거래 수수료 | 거래당 과금 | Stripe, 토스페이먼츠 |

### 3분류 관점과 세부 유형

위 유형들은 크게 **판매**(사용자에게 직접 받기), **광고**(사용자의 주의를 상품화해 광고주에게 받기), **중개**(거래 양쪽을 연결하고 수수료 받기)의 3분류로 묶인다.

판매 모델의 세부 트리:

- **전면 유료화** — 일회성 결제(콘텐츠 단건 구매), 정기 구독(월정액 무제한), 종량제(쓴 만큼 과금)
- **부분 유료화** — 아이템 모델(게임 아이템, 웹툰 쿠키, 후원 별풍선 같은 가상 재화), Freemium(기본 무료 + 프리미엄 유료)

광고 모델의 과금은 **CPM**(노출당)과 **CPC**(클릭당 — 실제 행동이 측정돼 광고주가 효율을 확인 가능)로 나뉜다. 중개 모델은 시장을 독과점하면 수수료 인상으로 이어지는 부작용이 따른다 — 플랫폼 규제 논의의 단골 배경 (대표 사례: [[In-App-Purchase|인앱결제]]).

실무는 **하이브리드**가 일반적이다 — 광고를 보면 가상 재화를 지급(광고 + 아이템)하거나, 구독하면 광고를 제거(구독 + 광고)하는 식으로 한 서비스에 여러 모델을 조합한다.

수익원을 평가할 때는 **공급 제약 여부**도 본다 — 공급자의 시간에 묶인 매출(공연, 직접 출연, 컨설팅)은 확장에 상한이 있고, 간접 참여형 매출(MD, 라이선싱, 멤버십, 콘텐츠 판매)은 시간 제약 없이 확장된다. 성장 전략은 후자의 비중을 키우는 방향으로 설계된다.

**수익 모델은 카테고리와 타겟 특성의 함수다.** 저단가 대량 거래는 수수료 부과가 어려워 트래픽이 커도 광고 의존이 되기 쉽고, 고단가 신뢰 민감 거래(명품, 한정판)는 검수와 안전결제에 지불 의향이 있어 수수료 모델의 선순환이 성립한다. MAU, 체류시간 같은 트래픽 지표가 좋아도 수익 모델이 정립되지 않으면 적자가 누적된다 — 상세는 [[Category-Expansion|카테고리 확장과 카니발리제이션]].

### 수익 모델 선택과 PMF

수익 모델 선택의 기준이 수익 극대화가 아니라 **PMF 우선**일 수 있다. 수수료를 받으면 당장 수익이 나지만 거래의 매력이 깎인다 — 수수료 대신 광고를 골라 무료 거래의 매력을 지키고 시장 우위를 먼저 점하면, 흑자 미달성이어도 성장 가능성으로 기업가치는 높게 평가된다 ([[PMF-Funnel|PMF 검증]]). 단, 의도적 선택과 수익모델 미정립을 구분해야 한다 — 전자는 수익화 옵션(수수료 전환, 광고 고도화)을 가진 채 미루는 것이고, 후자는 옵션 자체가 없는 것이다.

### 플랫폼의 성립 조건

중개 수수료 모델을 가졌다고 플랫폼인 것은 아니다. 공급자와 사용자를 잇는 장터가 진짜 양면 플랫폼으로 성립하는 조건 세 가지:

1. **양면의 입장 전환** — 참여자가 공급자와 소비자를 쉽게 오갈 수 있어야 한다 (개인 간 중고거래, 승차와 숙박 공유). 판매자가 대부분 사업자인 오픈마켓은 구조상 중개여도 고객 인식은 B2C 쇼핑몰이다.
2. **출발부터 플랫폼 정체성** — 참여자가 처음부터 플랫폼으로 인식해야 한다. B2C 쇼핑몰로 자리 잡은 서비스의 사후 플랫폼 전환은 인식을 바꾸지 못해 실패하기 쉽다.
3. **가치 교환이 이용 목적** — 거래가 서비스의 본질 목적이어야 하고(등가 교환이 아닌 후원, 도네이션도 가치 교환), 안전과 편의 같은 플랫폼 고유의 특장점이 분명해야 한다. 특장점 없는 중개는 커뮤니티 장터로 전락한다.

쇼핑몰의 플랫폼화가 어려운 근본 이유 — 누군가 물건을 팔려고 작정한 공간에서 사람들은 자신을 드러내지 않는다. 내 정보와 활동이 판매에 활용된다는 인식 때문에 자발적 상호작용이 일어나지 않는다 ([[Community-Strategy|커뮤니티 전략]]의 UGC 동기 문제와 같은 축). 가능한 경로는 입장 전환이 성립하는 중고 장터형이거나, 상품 구매와 연결되는 별도의 가치 교환 축을 처음부터 새로 짜는 것이다.

### 구독 모델 상세

구독 경제의 4가지 형태:

| 형태 | 대상 | 예시 |
|---|---|---|
| 정기 배송 | 주기적 구매 상품 | 꽃, 샐러드, 생필품 |
| 무제한 이용 | 디지털 콘텐츠 | 동영상, 음원 스트리밍 |
| 렌탈 | 고가 내구재 | 정수기, 비데 |
| 클라우드 | 소프트웨어, 인프라 | AWS, Adobe |

구독의 본질은 고객을 팬으로 만든 뒤 LTV를 키우는 것 — **방향은 팬 → 구독이다.** 팬이 없는 상품들을 모아 놓고 구독부터 제안하는 역방향은 단순 큐레이션에 그쳐 실패한다. 신규 구독자를 모집하는 구독 경제와 기존 팬을 적립, 혜택으로 유지하는 멤버십 모델([[Commerce-Member|커머스 회원 도메인]]의 락인)은 접근 방향이 다르며, 트래픽이 아무리 커도 팬 단계를 건너뛸 수는 없다.

## Unit Economics

비즈니스의 단위당 수익성을 측정하는 핵심 지표이다.

- **CAC (Customer Acquisition Cost):** 고객 1명 획득 비용
- **LTV (Lifetime Value):** 고객 1명이 생애 동안 가져다주는 수익
- **LTV/CAC 비율:** 3:1 이상이면 건강한 비즈니스 (벤치마크)
- **Payback Period:** CAC를 회수하는 데 걸리는 기간 (12개월 이내 권장)

## 현장 적용: school-manage

- **가치 제안:** "매주 일요일, 이거 하나 열면 다 되는 모임 운영 도구"
- **수익 모델:** Freemium (무과금 핵심 → 광고 → 유료 구독 3단계)
- **로드맵 연결:** 기능이 추가되는 단계와 유료 전환 시점을 일치시킴
- **비용 구조:** 월 ~40만원 (서버+AI), 1인 개발로 인건비 최소화

## 면접 포인트

- 로드맵 단계와 가격 구간을 일치시켜 기능 가치 = 과금 이유

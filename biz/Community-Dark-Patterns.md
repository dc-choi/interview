# 커뮤니티·프로덕트 다크 패턴

**"진입은 쉽게, 이탈은 어렵게"** 라는 설계가 넘어서면 안 되는 윤리 경계. 커뮤니티 앱·SNS·게임·구독 서비스가 리텐션을 높이려다 빠지기 쉬운 함정. 건강한 리텐션 설계와 조작적 리텐션 설계를 구분하는 프레임.

## 핵심 명제

리텐션은 좋은 지표다. 하지만 **사용자가 가치를 느껴서 남는 것**과 **빠져나가기 어려워 남는 것**은 다르다. 장기적으로 후자는 법적 규제(dark pattern 규제)·여론 역풍·브랜드 붕괴로 돌아온다.

## 4가지 다크 패턴 메커니즘

종교 컬트 조직에서 가장 강하게 관찰되지만, 커뮤니티·SNS·MMO·서브스크립션 서비스에 그대로 이식된다.

### 1. 몰입형 온보딩 (Immersive Onboarding)

사용자의 **일상 리듬을 재구성**한다. 단순 튜토리얼이 아니라 생활 시간을 설계.
- 매일 출석 보상, 타이머 기반 이벤트, 연속 접속 streak
- 알림으로 시간축 장악 (아침·저녁·주말 시간대 확보)
- 초기 며칠간 앱 사용 시간이 비정상적으로 길게 설계됨

**건강 vs 조작**: 학습형 온보딩(가치를 이해시키는 과정) ≠ 시간 장악형 온보딩(탈출 타이밍을 없애는 과정).

### 2. 정서적 네트워크 (Emotional Network)

1:1 관계를 인위적으로 설계해 **감정 기반 의존성**을 만든다.
- 전담 매칭(마스터·선생님·매니저)으로 책임감 유도
- 이탈 시 "누군가를 실망시키는" 죄책감 발생
- 익명·거래 관계가 아니라 의도된 친밀성

**건강 vs 조작**: 자발적 커뮤니티 친밀감(사용자가 통제 가능) ≠ 설계된 친밀감(이탈 비용 부과 목적).

### 3. 정체성 기반 기여 (Identity-Based Contribution)

사용자가 **자기 정체성을 제품 안에서 형성**하게 한다.
- 등급·직책·타이틀이 실제 삶의 정체성을 대체
- "회사 모임원"·"레벨 500"·"인플루언서"가 자아의 핵심
- 이탈 = 정체성 상실 → 이탈을 막는 장벽으로 작동

**건강 vs 조작**: 제품이 **도구**인가 **정체성**인가. 도구라면 이탈은 단순 계약 종료. 정체성이면 이탈이 실존 위기.

### 4. 심리적 잠금장치 (Psychological Lock-in)

물리적·기술적 강제가 없어도 관계·시간·감정 자산으로 이탈을 어렵게.
- 투자한 시간 (sunk cost bias 자극)
- 축적된 디지털 자산 (레벨·팔로워·컨텐츠)
- 사회적 관계 (친구·매칭된 사람들)
- 이탈 절차의 의도적 복잡화

이 영역이 가장 뚜렷한 **dark pattern**. EU DSA·FTC·한국 공정위가 규제 대상으로 삼는 영역.

## 건강한 리텐션 vs 다크 패턴 — 판별 체크리스트

| 기준 | 건강한 리텐션 | 다크 패턴 |
|---|---|---|
| 사용자가 **왜 남는가** 질문에 답할 수 있는가 | "○○한 가치가 있어서" | "그만두기 번거로워서" |
| **이탈 절차**가 가입만큼 쉬운가 | 1–2 클릭 | 여러 페이지·확인·전화 상담 |
| 가치 전달 이전에 **의존성부터 만드나** | 가치 → 습관 | 습관 → 의존 → 가치 주장 |
| **이탈 후 만족도**는? | 무관하거나 긍정 | 후회·상실감·죄책감 |
| **이탈 이유**를 정직하게 묻는가 | NPS·exit survey 활용 | 묻지 않거나 이탈 방해 |
| 경쟁사로 이동 시 **데이터 이동 가능**한가 | export·API 지원 | 수동 스크래핑만 가능 |
| **알림 빈도**는 가치 정보인가 체류 유도인가 | 실제 이벤트 기반 | 시간축 장악용 |

## 프로덕트 지표로 감지하기

- **활성도 분포의 이상치**: 상위 1% 사용자의 일일 체류 시간이 비정상적 (SNS·게임 중독 위험)
- **이탈 시도 후 복귀율 vs 이탈 후 후기**: 복귀율 높은데 후기가 부정적이면 의존이 다크 패턴
- **churn 설문의 솔직도**: "왜 떠나는가"에 "잘 모르겠다"가 많으면 사용자가 스스로 이탈 이유를 언어화 못 하는 상태 → 심리적 잠금
- **브랜드 안티 커뮤니티 규모**: 탈퇴자 모임·후기가 비정상적으로 활발하면 조직적 문제 신호
- **미성년자·취약계층 비율**: 높을수록 윤리 경계가 엄격해짐

## 장기 비용 — 왜 단기 리텐션에 매달리면 안 되는가

| 비용 | 설명 |
|---|---|
| **규제 리스크** | EU DSA·GDPR·FTC·공정위의 dark pattern 제재 확대 |
| **브랜드 붕괴** | 한 번의 언론 보도·사회적 이슈로 장기 신뢰 손실 |
| **직원 윤리적 번아웃** | "우리 제품이 해롭다"는 자각이 퇴사로 이어짐 |
| **법적 소송** | 미국의 "unfair design" 소송, 한국의 표시광고법 |
| **장기 LTV 하락** | 강제된 리텐션은 구전 마케팅을 역으로 소모 |

단기 리텐션 지표에 최적화하면 장기 지속가능성이 붕괴되는 **로컬 최적 함정**.

## 반대 사례 — 건강한 리텐션 설계

- **Basecamp**: 언제든 데이터 export, 이탈 절차 2클릭, 이메일로 사용률 낮으면 "해지하시겠어요?" 먼저 제안
- **Mailchimp**: 비활성 사용자에 자동으로 플랜 다운그레이드 제안
- **Spotify**: 추천 품질로 재진입 유도, 이탈 절차 짧음 (subscription 해지 가능)
- 반면 예: **삭제 절차가 지원팀 문의만 가능한 서비스**, **탈퇴 후에도 계정 복구 유도 메일 반복 발송하는 서비스** 등

## 윤리 경계 — 언제 선을 넘는가

리텐션 설계와 다크 패턴의 경계는 4가지 질문으로 판별:

1. **사용자가 알면 동의할까** — 알리지 못한다면 조작 신호
2. **경쟁사로 옮길 자유가 있는가** — 없다면 lock-in 다크 패턴
3. **이탈이 당사자에게 이익인데 막는가** — 막는다면 윤리 선 밖
4. **취약 계층에도 같은 전략을 쓰는가** — 쓴다면 사회적 책임 방기

## 자주 헷갈리는 포인트

- **리텐션 자체는 다크 패턴이 아님** — 가치를 느껴 남는 사용자는 환영. 문제는 *왜* 남는가
- **게이미피케이션도 선악 스펙트럼** — 학습 동기로 쓰면 좋지만 체류 유도만 목적이면 다크
- **"고객이 좋아하니까"는 면죄부 아님** — 도박·SNS 중독도 당사자는 "좋아"함. 설계자 책임 별개
- **FOMO 활용 vs 조작 구분** — 이벤트 알림(이탈해도 OK) vs 이탈 방해 알림(이탈을 막으려는 전략)
- **규제는 지표가 아닌 구조를 본다** — "사용자가 자발적으로 선택했다"가 변명이 안 되는 시대

## 면접 체크포인트

- **리텐션과 다크 패턴의 경계**를 설명할 수 있는가
- 4가지 다크 패턴 메커니즘과 건강한 대응 설계
- **이탈 설계(churn UX)** 가 브랜드에 주는 장기 가치
- 다크 패턴이 **단기 지표는 올리지만 장기 LTV를 해치는** 이유
- 규제 동향(EU DSA·FTC·공정위)이 프로덕트에 주는 제약
- 프로덕트 윤리를 평가할 **4가지 판별 질문**

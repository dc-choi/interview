# 엔터테인먼트도메인특강(카카오엔터)

## 백엔드개발자의역할

### 데이터관리
- 클라이언트 요청에 맞는 응답을 줄 수 있도록 데이터를 어떻게 잘 저장하고 가져올지 관리

### 클린코드&아키텍처
- 남들이 보기에 무리 없는 코드, 표준을 따르는 아키텍처를 고민해야 함

### 비즈니스관점
- 개발자도 비즈니스 관점에서 생각해야 함
- 비즈니스 요구에 맞는 아키텍처를 선택하는 것이 핵심

### 테스팅
- 사전 과제를 주는 회사 중 테스트 코드 없으면 탈락시키는 곳이 있음
- 테스트 코드 작성을 습관화할 것
- 프레임워크 내부 동작을 확인해보면 좋음

### 기술업데이트
- 라이브러리 릴리즈 노트를 확인하는 습관
- 시대에 맞는 패러다임 변화를 따라가야 함

## 면접에서깊게물어보는것들

### 프로그래밍언어
- 문법/사용법은 기본
- **왜** 사용하는지, 사용 시 **문제점**은 무엇인지
- 사용 중인 버전에서 개선된 부분을 알고 있는지
- GC 종류와 동작 원리
- JVM 메모리 영역, 객체 생명주기

### 데이터베이스
- RDB: 데이터 무결성이 중요한 영역
- NoSQL: 데이터가 수시로 바뀌는 경우
- 프로젝트 특성에 따라 DB를 선택한 이유를 설명할 수 있어야 함

### API&네트워크
- HTTP 표준, 요청/응답 구조에 대한 이해
- 로드밸런서, 서비스 라우터

### 보안
- 기본적인 보안 수칙을 지킬 수 있어야 함
- 서비스 접근 권한, 네트워크 망 설정에 대한 이해

### 클라우드
- AWS가 주류. 보수적인 금융권도 퍼블릭 클라우드 사용 중
- AWS만 알아도 충분

## 백엔드인프라전체그림

| 계층 | 구성요소 |
|------|---------|
| 네트워크 | 로드밸런서, 서비스라우터 |
| 보안 | 정책매니징, 인증제공자 |
| 컴퓨팅 | API게이트웨이, 애플리케이션, 컨테이너&오케스트레이션 |

| 모니터링 | CloudWatch, Datadog |

## DevOps
- 개발과 운영 사이클을 빠르게 하기 위한 **문화적 관점**에서 생겨난 것
- 개발자와 운영자의 관점을 동시에 볼 수 있도록 환경을 세팅하는 역할
- AWS를 잘 아는 백엔드 개발자라면 DevOps도 가능
- 계속해서 자신의 영역을 넓히는 것이 중요

## AI활용
- GPT-4, Copilot으로 대체 가능한 작업을 빠르게 전환하는 추세
- 테스트 코드 작성, 로그 분석 등에 활용
- AI를 잘 활용하는 것이 경쟁력

## 취업조언

### 면접
- 면접 = 같이 일할 사람을 찾는 과정
- **능력**: "왜?"라는 질문에 깊게 파고들 수 있는지
- **태도**: "같이 일하고 싶다"는 마인드로 접근
- **기업 문화 파악법**: 개발자 커뮤니티에서 질문, 면접 리뷰 검색, 채용 공고 우대사항 확인

### 준비순서
1. 코딩 테스트 (최소한의 역량 검증)
2. CS (자료구조, 알고리즘, 운영체제, 네트워크)
3. 포트폴리오/프로젝트 (내가 설명할 수 있는 것만 채울 것)

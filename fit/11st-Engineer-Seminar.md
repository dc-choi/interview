# 11번가엔지니어세미나

## 면접관점

### 이직사유
- 개발 외적인 업무가 많아 성장이 어려운 환경이었다는 점을 솔직하게 전달
- 자체 서비스를 해보고 싶다는 성장 동기를 명확히
- 꼬리질문 대비: "왜 그 환경에서 개선을 시도하지 않았는가?", "자체 서비스란 구체적으로 무엇인가?"

### 면접팁
- **애매하게 아는 것은 말하지 않는다.** 잘못된 답변으로 이어질 수 있음
- 모르는 것은 모른다고 하는 것이 낫다

## 기술적인사이트

### Redis활용사례
- 선착순 쿠폰 시스템에서 동시성 제어를 위한 분산 락 관리에 Redis 사용

### 테스트전략
- 서비스 레이어는 반드시 테스트
- 컨트롤러 → 통합테스트
- 서비스 → 단위테스트
- Mock은 최소화 (실제 동작을 검증해야 의미 있음)

### 코드리뷰Pn룰
- 코멘트에 P1~P5 태그를 붙여 중요도를 표현
  - **P1**: 반드시 수정 (버그, 보안 등)
  - **P2~P3**: 수정 권장 (설계, 성능)
  - **P4~P5**: 제안/선택사항 (스타일, 개선 아이디어)
- 리뷰어의 의도가 명확해져 불필요한 커뮤니케이션 비용을 줄임

### 리팩토링
- 시간이 날 때 필요성이 있다면 진행
- 별도 일정을 잡기보다 여유가 있을 때 점진적으로

### 엔지니어링문화
- 서비스 단위로 개발하더라도, 다른 팀의 프로젝트 구성이나 코드리뷰 내용을 열람 가능
- 조직 전체의 기술 수준을 공유하는 투명한 문화

## 성장방법

### 학습순서
- Java → 알고리즘 → 프로젝트 순서로 단계적 성장

### 꾸준함의비결
- 스스로 재미를 찾는 것
- 하는 만큼 얻어간다는 마인드
- 스터디를 통한 동료와의 학습

### 신기술학습법
- 공식문서 → 디버깅 → 책 → 강의 순서
- 가능하면 원서로 읽는 것을 권장

### 추천도서
- 내 코드가 그렇게 이상한가요?
- 이펙티브 자바
- 모던 자바 인 액션
- 이펙티브 소프트웨어 테스팅
- 프로그래머의 길, 멘토에게 묻다

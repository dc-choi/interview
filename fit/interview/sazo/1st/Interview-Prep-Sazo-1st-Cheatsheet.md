> **06-16(화) 17:30~18:10 화상 40분**, 회의 링크는 캘린더 확인, 담당 채용담당자(HR 가능성)
> 절차: 서류합격(6/12) → **1차 핏(현재)** → 이후 미공지(역질문으로 확인). 톤: 가벼운 핏 자리지만 **가벼움 = 평가 면제 아님**.

## 0. 면접장 들어가기 전 3원칙 (에고이즘 교훈)

1. **역질문은 마지막에 몰기** — 면접관이 마이크 넘길 때까지 답변에 집중 (초반 역질문 = 제지당함).
2. **모든 답변 끝줄 = 사줘의 구매대행 파이프라인/성장 구간으로 연결.**
3. **갭은 인정 → 전이 → 흡수 3단, 우기지 말 것.** 답변 30~45초 기본, 길어도 90초.

## 1. 오프닝 (이건 거의 그대로)

**자기소개 첫 문장**: "안녕하세요, 4년차 백엔드 개발자 [이름]입니다."
→ 트라이포드 NestJS VMI(IoT 재고 Lock 정합성 + EventBridge/SQS 발주 자동화 + 복합인덱스 99.3% + Grafana/Prometheus/Loki) → 시솔 국제배송(통관 환율 배치 + 택배사 15곳 추적 스크래핑) → **"빠르게보다 운영에서 안 깨지는 구조, 실패 전제 재시도/DLQ/알림 설계 중시."**

**자기소개 끝줄(앵커)**: 사줘가 임의 일본몰 URL 파싱 → 환율/관세 사전 확정 → 구매/검수/국제배송 직접 책임지는 파이프라인 = 제가 국제배송, 재고정합, 이벤트 자동화로 풀어온 문제와 본질이 같다.

## 2. FIT 답변 키워드 (입으로 나오는지 체크)

| 질문 | 트리거 키워드 (끝줄은 항상 사줘 연결) |
|---|---|
| **왜 사줘 (3축)** | ① 도메인 정확히 겹침(시솔 국제배송=환율/통관/추적, 사줘는 더 큰 규모 + 구매대행이라 파이프라인 직접 소유) ② 문제 명확+성과 측정(파서 안정화, 1점물 race, 거래액 반년 189배 스케일) ③ AI를 팀 워크플로우로(Claude Code 파이프라인 재설계) |
| **이직 사유** | 트라이포드 단독 책임 0→1/자동화/PoC → 다음은 **깊이**. B2C 빠른 피드백 사이클 + 동료와 아키텍처 합 맞추기. 사줘 = 그 두 조건 직격. (외부귀책 금지) |
| **잦은 이직 꼬리** | 각 자리 한 축씩(이썸 도메인 → 시솔 운영품질 → 트라이포드 0→1/자동화). 이번은 **최소 2~3년 깊게.** |
| **임팩트 있던 일** | 발주 자동화(정성 먼저) → 수치는 꼬리: 수기발주 1시간→자동, 재고관리 4시간→10분, MSK 574$ vs SQS 0~18$. 끝줄: "구매대행도 단계 전이가 이벤트로 흐르고 실패가 일상 → 같은 설계 사고." |
| **AI 도구** | 코드 생성 아닌 **파이프라인 재설계**(계층형 CLAUDE.md, MCP, Subagent, Hook). 설계/의사결정은 직접. (AI-First 조직 = 직격) |
| **협업/비개발 소통** | 의도 먼저 묻고 재확인, 상대 언어로. |
| **사이드 프로젝트** | 출석부 SaaS — **모임 104개, 본당 82곳, 학생 2,864명, 출석 ~1.7만 건, MAO 38곳** (06-09 기준). 본업 가드 한 줄. |
| **워라밸/강도** | 의미 있으면 투자 OK + 지속가능성은 역질문으로 마이크 넘김. (부정단어 먼저 꺼내지 말 것) |
| **5년 후** | Staff 트랙, 권한 아닌 영향력. |

## 3. 기술 갭 3종 (1차에도 "Kafka 안 써봤는데?" 가능 — 인정→전이→흡수)

- **Kafka**: 클러스터 운영 X. but MSK 574$ vs SQS 0~18$ 비교로 **안 쓴 게 의사결정**. at-least-once 중복/멱등컨슈머/DLQ/키순서 = SQS에서 이미 운영. "visibility timeout→오프셋 커밋, MessageGroupId→파티션 키 차이라 학습은 개념 아닌 도구 숙련." **사줘 규모(189배, 백프레셔, 다단계 소비)면 Kafka가 정답 — 그래서 지원.**
- **Elasticsearch**: 운영 X. but **왜 RDB가 한계인지** 실무 체감(LIKE 선행 와일드카드 풀스캔 → 역색인). B-Tree 내부 아는 상태라 자료구조만 다름. 입사 후 mapping/analyzer + MySQL 동기화(CDC/배치)부터 파악. (kuromoji+nori 다국어 = 사줘 직격, 개념 언급 가치)
- **MSA**: 물리 분리 X. but 모놀리스에서 **경계 긋기**(공유테이블 금지, 이벤트/포트 통신, 클린아키 5계층) + 발주 EventBridge/SQS 분리. "이 규칙 지킨 사람은 물리 분리 한 걸음만 남음."
- **스크래핑**: 시솔 택배사 15곳, API 없는 곳 스크래핑 + 어댑터 패턴 + Sentry 변경 감지. 본질 = "**200 주면서 조용히 틀림** → 모니터링이 설계의 절반." (정직: HTTP 레벨이었고 Puppeteer 헤드리스는 미사용). 끝줄: 사줘 파서 파이프라인에 그대로 심화 적용.
- **PG/TypeORM**: 최적화 방법론 DB 중립(EXPLAIN, 카디널리티 복합인덱스, 커버링). PG 차이(BUFFERS, BRIN/GIN, VACUUM bloat) 정리됨. **Go/gRPC 미경험 — 솔직**, 개념만(HTTP/2, Protobuf, 내부통신).

## 4. 도메인 멘트 — 2개만 입에 붙이기

1. **(깨짐 감지)** "스크래핑은 만드는 것보다 깨짐을 빨리 아는 게 본질. 사줘 파서도 결국 **소스별 성공률 모니터링 싸움**이라 봅니다."
2. **(환율 시점)** "환율 배치를 30분 간격 재시도 4회로 운영해봐서, **표시/결제스냅샷/관세청 고시 환율이 다 다른 시점 문제**라는 걸 압니다. 가격 엔진은 환율 스냅샷을 주문에 어떻게 고정하느냐가 핵심."

> 여유 있으면 추가: 1점물 race(IoT 동시 재고 정합성 전이), 150$ 합산과세(합포장이 고객 세금 바꿈 → WMS 출고 + 세금엔진 결합), 양방향(직구+역직구) 언급 = 조사 깊이 시그널.

## 6. 역질문 (마지막에 몰기, 5개 중 시간 맞춰)

1. **다음 단계 형식** — "이후 전형은? 기술 면접 형식(과제/라이브코딩/이력서 딥다이브)을 알 수 있을까요?" (절차 미공지라 정보가치 최대)
2. **팀 구성** — "백엔드는 몇 분이고, 테크리드 두 분과 어떤 구조로 일하나요?"
3. **AI 우선순위** — "백엔드 개발 사이클에서 AI는 어디까지 적용돼 있나요?"
4. **스택 확인** — "우대에 Go/gRPC, PG/TypeORM이 있던데 NestJS와 Go 역할 분담은?" (JD 정독 시그널)
5. (시간 남으면) **병목** — "189배 성장 구간에서 백엔드가 가장 아픈 병목 하나는?"
> 아껴둘 것(다음 단계): 6개월 핵심 과제, MSA 분리 수준, 앱 평점 로드맵(운영 안정화로 포장).

## 7. 지뢰밭 (절대 밟지 말 것)

- ⚠️ **입사 시점** — 윤회 최종합격 수락(6/25 온보딩, **6/29 출근**). 입사시점 질문에 **확답 회피, "협의 가능" 선까지만.** 처우는 처우 단계에서 시장가 기준.
- ⚠️ **일본어** — 못함, 솔직. "일본 사이트 파싱은 언어보다 **구조 문제** + AI 번역 활용." 영어 문서는 가능.
- ⚠️ **외부 귀책 금지** — 회사 사정/비전 없음 X. 회사 평가 톤(안정적이라서) X. 자기 귀책 + 학습 + 현재 적용.
- ⚠️ **부정 단어**(워라밸, 연봉) 본인이 먼저 꺼내지 않기.
- ⚠️ **Kafka/ES 우기지 말 것** — 다음 단계에서 들통.
- ⚠️ 따옴표 강조, 과한 대구 없이 사람처럼 말하기.

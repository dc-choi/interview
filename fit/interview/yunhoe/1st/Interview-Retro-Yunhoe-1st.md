## 진행 흐름 (실제 받은 질문)

### 1. 도메인 이해 검증 (기획담당, 팀장)
- **Q**: DPP 기본 이해? 어떻게 트레이싱? → QR, 순환경제, 생산-유통-재가공 ESG로 이해, 물류 도메인 경험과 연결
- **Q**: 물류는 RFID 아닌가? → 현장에선 QR/바코드 썼다 (CBT→WMS/OMS 확장 프로젝트), 업체마다 표준화 안 됨
- **Q**: GS1, GTIN 아나? → **모른다 인정** (바코드 규격인 건 인지). 팀장이 글로벌 표준, 왜 중요한지 설명
- 회사측 설명: **티어 구조**(티어4 원료→티어1 봉제에서 DPP 부착), 데이터 캐리어(QR/NFC), DPP 레지스트리 7/19 출시, OSS 테스트 완료, EU ESPR 규제 대응
- **Q**: 이벤트 기반 데이터 캡처 인지? → 우대사항에서 봤다, 각 공정 이벤트로 취득 가능하겠다

### 2. 기술 검증 (개발팀장 팀장)
- **Q**: 왜 ECS? → 매니지드, 백엔드 1인 운영, k8s 비용, 규모 부담, Docker 이미 사용 → Fargate 서버리스, 인프라 AWS 위임
- **Q**: 스케일링 기준? → **스케일 아웃**(멀티 인스턴스), CPU 65→70%, 메모리 70%, "90%↑는 모니터링 거의 안 됨/뻗으면 중단", 비용 고려
- **Q**: IoT/AWS 셧다운(공장 화재, 리전 장애) DR 설계? → **비용 문제로 못 했다 솔직 인정**, 버지니아 사태 영향 없었음, 멀티클라우드(Azure/GCP) 제안 중, 최악 온프레미스
- **Q**: IoT 무슨 일? 기기 개발? → 아니다. HW 엔지니어 OEM 설계, MQTT 고려했으나 HW가 MVP에서 멈춤 → HTTP 통신, 서버에서 복호화, 로직
- **Q**: IoT 통신 단/양방향? → 단방향(HW push), 서버 명령 미구현
- **Q**: Java/Spring → NestJS 전환 계기? → 시솔지주 Express로 Node 친숙 → 이직하며 Nest, Spring과 유사, 각광
- **Q**: DB 트랜잭션 언제? → 쓰기 작업. IoT 재고 갱신 시 **레이스 컨디션** → 트랜잭션으로 처리
- **Q**: 릴레이션(FK) 거는 기준? (팀장은 안 검) → 정규화가 원칙이나 실무는 **도메인(엔티티), 요구사항 파악 우선**, 풀 정규화는 비현실적
- **Q**: 마이그레이션 자주? (규제로 컬럼 잦은 변경) → 기획 변경 시 함, 컬럼 추가→값 복사 방식, 대용량은 배치
- **Q**: 프론트 경험? → 사이드로 리액트
- **Q**: 다른 언어(파이썬 등)? → 직접 X, 소스는 봄, 파이썬 들여쓰기 숙지 필요
- **Q**: 좋은 소프트웨어란? → 예전엔 가독성, **지금은 AI가 잘 알아듣는 것** — 계층형 CLAUDE.md 컨텍스트 엔지니어링, 타입 언어, 워크플로우
- **Q**: 퀄리티 vs 속도? → **리스크 따라**. 환불 정책 미비 사례(퀄리티 사수) vs 검증 필요 시 GA 지표로 빠른 검증(속도)
- **Q**: 장점/단점 하나씩? → 단점: **대규모 트래픽 경험 없음**. 장점: 그래서 더 고민, 회사 맞춤 아키텍처 고민 자신

### 3. 컬처핏, 글로벌 (기획담당)
- **Q**: 서버 보안? → 보안학과. ISO/ISMS-P 인사이트? → 전문가 아님 솔직, 서버단: 파일 유출 방지, XSS, CSRF, Helmet, 시큐어 코딩, OWASP Top 10
- **Q**: 글로벌 워킹그룹(UNTP, EU 4sFast 파트너) 영어 가능? → FIDO/패스키 인증 경험으로 영어 메일, 미팅(번역기 병행), 듣기 OK, 스피킹/라이팅 보완 필요, 줌 채팅 참여 가능
- **Q**: AI 툴 클로드 외? → 코덱스(리뷰, 검증용), 클로드가 코덱스 부르는 워크플로우
- **Q**: AI 시대 개발방식 변화 거부감? → 일하는 방식 바뀜, 거시적 시야, 스펙 드리븐, 기획 참여, **판단은 인간 영역**
- **Q**: 이직 계기 + 어떤 회사와? → 제로투원(오류율 50%→0), SCM→VMI 자동발주, **B2B만 해서 B2C 갈증** + 윤회는 피드백 사이클 빠름(B2G/B2B) + 규제, 과거 경험 살림
- **Q**: ESG 관심? 약자 아나? → **정확히 모름 (환경 정도로 이해)** → 담당자가 E/S/G, LCA/sLCA, 패스트패션 설명
- **Q**: 패스트패션 브랜드? → 무신사, 스파오 (솔직)

### 4. 역질문 (동철 → 회사)
- 로드맵, 기여 시작점? → 프로젝트 다수(RND+클라이언트+코이카 정부과제), DPP는 하나의 산업, ESPR 진행, 위임법, 18개월 유예, 2029 의무화, UNTP
- 워크로드 괜찮은지 묻자 → 동철 "좋다"
- 내적 동기부여 유지? → 팀원 좋고 서로 도움 / 팀장 "양보다 난이도가 높다, 섬유 산업 기본부터 다 이해"
- 멤버 구성? → 개발 3, 디자인 3, 대표, 기획PM 2, 경영지원 2, 브랜드 3 + 박사 어드바이저, **총 14~16명** (놀람)
- MBTI? → 동철 ISTP / 담당자 ENFJ
- 자체 브랜드 LTS 옷 DPP 스캔 시연 (레벨4 폴리에스터)
- **태그 내구성 RND 역질문 → 담당자 "이런 질문하는 사람 뽑아야 돼"** (★ 강한 긍정 신호)
- DB 2개 이유? → 팀장: MySQL+Postgres, TypeORM. JSON 저장 많음(스키마 잦은 변경→릴레이션 매핑 시 마이그레이션 지옥), Postgres JSON 지원 우수 → 마이그레이션 중. 점유율, pgvector, 메시지큐까지 해결
- Redis 어디? → Hocuspocus 실시간 협업(노션식 커서), pub/sub
- Redis 데이터 유실 관리? → TTL 짧게(3분), 미사용 시 DB 세이브 후 삭제, IO 많지만 비용 저렴

## ✅ 잘 작동한 카드

1. **물류, QR 도메인 → DPP 트레이싱 연결** — 도메인 이해 첫 질문부터 자기 경험으로 자연스럽게 답
2. **모르는 건 솔직히 (GS1, ISMS-P, ESG 약자)** — 모름 인정 + 아는 범위는 정확히. 시그널 문서의 "모르면 모른다"대로
3. **DR 미설계도 비용 맥락으로 솔직 + 멀티클라우드 제안 중** — 변명 없이 인정 + 개선 방향 제시
4. **AI 협업 깊이 (클로드+코덱스 워크플로우, 컨텍스트 엔지니어링, 판단은 인간)** — 팀장이 "AI 1인 1프로젝트" 조직으로 전환 중이라 강하게 공명
5. **태그 내구성 RND 역질문** — "이런 질문하는 사람 뽑아야 돼" 직접 반응. 호기심, 제품 관점 어필 성공
6. **DB, Redis 깊은 역질문** — 팀장과 기술 동료 대화처럼 흘러감, 본인 지식도 자연 노출(pgvector, 메시지큐, 실시간성 챌린지)
7. **이직 사유 (B2B→B2C 갈증 + 규제 정합성 살림)** — 외부 귀책 없이 자기 성장 동기로 깔끔

## ⚠️ 다듬을 점

1. **ESG 약자를 못 댐** — ESG가 회사 정체성인데 "환경 정도"로만 답. 담당자가 직접 E/S/G 설명. **도메인 성의 부족으로 비칠 위험** (가장 큰 감점 가설). → [[Interview-Prep-Yunhoe-Domain]]에 ESG, LCA/sLCA, ESPR 위임법 흡수 필요
2. **GS1, GTIN 모름** — 글로벌 표준 식별 체계는 DPP 핵심. 도메인 선행학습으로 메울 수 있었던 부분
3. **스케일링 답변의 "90%↑ 모니터링 안 됨"** 표현이 다소 부정확 — CloudWatch 지표 수집은 동작함. "임계 초과 시 알람, 오토스케일 반응이 늦어 장애로 번진다"로 정정 권장
4. **대규모 트래픽 없음을 단점으로 먼저 제시** — 솔직하나, 윤회는 초고트래픽 서비스가 아니므로 굳이 약점으로 둘 필요 적었음. "스타트업 규모에 맞는 최적 설계 경험"으로 프레이밍 가능

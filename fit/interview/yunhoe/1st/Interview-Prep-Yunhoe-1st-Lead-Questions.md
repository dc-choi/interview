# 윤회 1차 본 미팅 — 백엔드 리드·컬처핏·역질문·체크리스트

## 1. 백엔드 리드 페르소나 질문 (대표가 던질 가능성 ★★★)

### Q1. "백엔드 리드로서 처음 3개월 / 6개월 / 1년에 뭘 하겠나?"

- **3개월**: 멀티테넌트 데이터 모델·이벤트 흐름 정합성 점검 → 도메인 이벤트 카탈로그·상태 머신 가시화. 관측·알림 표준화 (Grafana/Sentry 임계값·SLO 정의). 백엔드 2명 분담 룰·코드 리뷰·배포 루틴 정착
- **6개월**: 하네스·AI 협업 파이프라인 도입 (계층형 CLAUDE.md, Subagent, Stop Hook 자가 리뷰, 실 DB 통합테스트). 표준 데이터 모델 변화 대응할 마이그레이션 인프라(듀얼 라이트·검증 리포트). **★ EDA 도입 깊이 결정·로드맵** — 현재 어느 층까지 (RabbitMQ + SNS/SQS 위에) 들어가 있는지 진단 → 다음 층 (Outbox 신뢰성·Event Store 확장·Saga 도입 시점) 단계적 도입. DPP가 EDA 적합 도메인인 만큼 **층 7까지 가야 할지 사업 단계 보면서 결정**.
- **1년**: 본 계약 확장에 맞춘 멀티테넌트 격리 강화(필요 시 스키마 분리 옵션), 글로벌 리전(EU·중국) 운영 거버넌스. CTO 합류 시 도메인 깊이를 제공하는 리드로 역할 전환
- 핵심 메시지: **사업 단계마다 백엔드가 무엇을 해야 하는지** 그림을 가진 사람 + **EDA를 결정 프레임워크로 다룰 줄 아는 리드**

### Q2. "팀원과 의견이 다를 때 어떻게 결정하나?"

- 결정 단계를 분리: **사실 정렬(데이터·SLO·비용) → 옵션 비교(트레이드오프 명시) → 결정 + 되돌리는 비용**. 결정이 비가역이면 더 보수적으로
- 결정자가 본인이면 이유와 되돌리는 트리거를 글로 남김. 결정자가 팀원이면 가드레일만 두고 권한 위임
- **사례 (대안 — EventBridge 중복 회피)**: 트라이포드랩에서 조회 API P99 600ms 튀어 **Read Replica 도입 결정** — 정합성 vs 비용·복잡도 트레이드오프 정리. 쓰기 직후 강한 일관성은 Primary 분기, 대시보드는 Replica. **조회 40% 개선 + DB CPU 30% 감소**. Prisma `datasources`로 코드 변경 최소화.
- (백업 사례) EventBridge vs MSK는 이직 사유 시솔에 짧게 1회만. Tech Q2엔 별도 디테일.

### Q3. "혼자 책임지는 환경과 팀원과 같이 가는 환경의 차이를 본 적 있나?"

- **혼자**: 트라이포드랩에서 백엔드 사실상 단독 운영 — 0→1 구축·EventBridge+SQS 결정·관측 인프라 구축·대형 PoC 모두 직접. 의사결정 빠르지만 **단일 장애점·암묵지 의존의 한계** 체감 (본인이 5/15에 "백엔드 거진 다 했다"라고 직접 언급한 부분)
- 같이: 결정 코스트 증가, 회복 탄력성·암묵지 명시화. **카카오테크 캠퍼스 멘토링·하코 3000명 Prisma 발표**는 본인이 혼자만 알지 않게 의식적으로 만든 채널
- **단독 운영에서 본 한계** (트라이포드): 본인 휴가·아플 때 운영 위험, 도메인 지식 본인 머리에만, 신규 PoC 시 본인 응대 병목
- 윤회에서는 **B/E 2명 → 4명 전환기** — 1인 운영 모드를 빨리 끝내고 **명시지·표준·자동화**로 옮겨야 함

> ⚠️ **5/15 톤 일관성**: 사이드 보조 사례는 5/15에 안 꺼냈으니 본 미팅에서도 본인이 먼저 안 꺼냄. 면접관이 직접 "사이드/취미?" 물으면 그때만 답변 (마스터 #15 톤).

### Q4. "CTO 합류 전까지 백엔드 거버넌스는 어떻게 잡을 건가?"

- 기술 결정 RFC 템플릿 + 결정 이력 (ADR). 작은 결정도 글로 남겨 CTO 합류 시 인수인계 비용 최소화
- 채용 기준·온보딩 패스 정의. 본인이 채용 인터뷰에도 참여
- 외부 의존성 표준화: SaaS·라이브러리 도입 시 점수표(비용·운영·보안·락인). 향후 CTO와 같은 언어로 토론 가능

### Q5. "기존 코드/팀 문화에 들어가서 처음에 뭘 안 건드릴 건가?"

- 처음 4주는 **돌아가는 것 + 결정 맥락**을 읽는 시간. 리팩토링·구조 변경 금지
- 단, 관측·SLO·알림은 우선 손볼 영역 (사고 잡는 도구는 입사 직후 정비해야 안전)
- 입사 90일 후 결정 회고: 무엇을 처음 90일에 안 한 게 좋은 판단이었는지 정리

## 3. 역질문 (나 → 대표)

### A. 매출·런웨이 (가장 중요)

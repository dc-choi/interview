# 윤회 1차 치트시트 — 어필 카드, 보조 답변, 체크리스트

## 7. 워라밸 답변 (혹시 받으면)

> "**의미 있는 일이라면 시간 투자 괜찮음** (트라이포드 PoC 밤샘, 주말 사례). 다만 궁금한 건 **지속 가능성** — 12명 팀에서 이 강도를 받쳐주는 미션 공감, 팀워크, 성과 인정 구조가 어떻게?"

## 8. 어필 카드 5장 (5/15 검증 4 + 보강 1)

> ⚠️ **5/15 톤 일관성 가드 (가장 중요)**: 5/15 24분 통화에서 본인이 사이드(출석부) 한 마디도 안 꺼냄. 본 미팅에서 갑자기 99개 모임 멀티테넌트 SaaS 꺼내면 "왜 그때 안 말했나?" 의심 — 진정성 시그널 깨짐. **사이드는 본인이 먼저 X, 면접관이 직접 "취미, 사이드 있나?" 물으면만 답변**. 5년차 가드(본업 메인) + 5/15 일관성 가드 둘 다 적용.

| 카드 | 사용 시점 |

## 9. 보조 답변 (혹시 받으면)

### 개발 환경, 도구 (Claude Code 어떻게?)
> "터미널 중심. Claude Code를 IDE 수준으로 커스터마이징 — **계층형 CLAUDE.md / MCP / Subagent 3종 / PostToolUse Hook / Stop Hook 자가 리뷰**. dotfile zsh+tmux+neovim 통일. CARE ID 합류 시 **팀 dotfile, CLAUDE.md 표준 제안** 가능."
→ 자연스럽게 **3번 AI 도구 깊이 + 6개월 도입 4단계**로 흐름 연결

### 디버깅 프로세스 (단골 질문)
> "**재현 → 가설 → 분리 검증 → 해결 → 영향 점검 → 회고**. Prisma 1000ms 사례 — APM 로그 → EXPLAIN → 공식 문서 `relationLoadStrategy` → DB-level JOIN 90% 개선."

### Q5 멀티테넌트 받으면 (본업만)
> "트라이포드랩에서 대형 PoC(제약바이오, F&B) 멀티테넌트 데이터 모델링 했습니다. 단계는 논리적(tenant_id 컬럼) → 스키마 분리 → 물리적. 현 단계는 **논리적 + 강제 가드**가 합리 — Prisma middleware로 tenant_id 자동 주입, RLS로 실수 방지, 인덱스 첫 컬럼 tenant_id 필수. 전환 임계는 컴플라이언스, noisy neighbor, 대형 고객 수 누적."

### Q8 생애주기 상태머신 받으면 (본업만)
> "이벤트 소싱 + read model 패턴. `events(aggregate_id, sequence, event_type, from_state, to_state, actor, evidence)` append-only, read model은 `product_state(current_state, last_event_id)` 비동기 프로젝션. 잘못된 전이 방어는 코드 enum + DB 트리거 이중, 위반 시 보상 워크플로(Saga). 이력은 SCD Type 2 유사 snapshot 테이블로 시점별 상태 보존."

### 취미, 사이드 (대표 직접 면접 변형 가능)
> "**주일학교 출석부 — 99개 모임 멀티테넌트 SaaS** (NestJS+tRPC 모노레포). 활성 본당 78곳, 학생 2,800명, 출석 13,000건+, MAO 30일 활성 43곳. **Parish→Church→Organization 3단계 위계 테넌트 + 명시적 권한 가드 + Snapshot 패턴** 직접 설계, 운영. 본인 사용자 작은 트래픽이라도 **멀티테넌트 운영 사고 직접 봐온 영역**. 그 외 **하코 3000명 Prisma 발표, 카카오테크 캠퍼스 멘토링**으로 본인 이해 점검."

### 방어 멘트 (사이드 답변 후 받는 꼬리 대비)

- **"본업 영향?"** → "퇴근 후 1시간 정도 + 주말에 시간 내서 합니다. 본업 시간은 침해하지 않게 가드 두고 있습니다."
- **"본업이 부족해서 사이드?"** → "본업으로는 트라이포드 0→1 + EventBridge, Grafana, DB Lock, 슬로우 쿼리 99.3% 등 다 직접 했습니다. 사이드는 본업에서 만나기 어려운 도메인 패턴(멀티테넌트, 이력 추적, MAU 트래킹) 학습 채널입니다."
- **"본격 사업?"** → "거의 본격 사업처럼 운영하지만 본업과 별개. 사이드로 0→1 운영 사이클 학습하는 채널입니다."

**CARE ID 매핑 (강력)**: "윤회 멀티테넌트(브랜드사=테넌트) 패턴 = 본인 출석부 동형. Q5 답변에 본인 사이드 멀티테넌트 운영 사례 직접 인용 가능 — 이론이 아니라 99개 단체 실전 운영."

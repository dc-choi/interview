- [x] 장단점: 실행력 / 성급함 — [[My-FIT-Answers|FIT]] 그대로

### AI 도구 활용 깊이 ★★★ ([[My-FIT-Answers#10. AI 도구 활용 깊이 ★ (자주 받음)|마스터 10번]])

Claude Code 위에 **계층형 CLAUDE.md로 컨텍스트 분할**(토큰 폭주 차단), MCP(context7)로 라이브러리 최신 문서 실시간 조회, **Subagent 3종**(api-documenter, security-reviewer, performance-analyzer), **PostToolUse Hook**으로 편집 파일만 lint, prettier(전체 린트 회피로 루프 속도 확보), **Stop Hook 자가 리뷰 에이전트**로 작업 종료 시 버그, 품질 점검 자동화를 구성했습니다.

**원칙**: **설계와 의사결정은 직접, 반복 작업은 AI와 협업**.

**CARE ID 매핑 (6개월 도입 제안)**:
1. **계층형 CLAUDE.md**: DPP 도메인, 테넌트별, 인프라별 컨텍스트 분리
2. **Subagent**: DPP 표준 변경 시 영향도 매트릭스 자동 생성, 검증

**꼬리**:

- **"입사 후 얼마나 빨리?"** → "1개월 PoC, 3개월 표준화, 팀 확산, 6개월 본격 운영. 단 솔루션 안정성 우선 — 도구 도입이 운영 흔들면 무의미"

### 워라밸, 강도 (작은 회사, 강한 책임 자리)

> 윤회는 백엔드 리드 + 12명 회사 + 1년 9명 turnover 시그널. "이 강도 버틸 수 있나" 질문 가능성.
> prep/ 가이드: [[Sensitive-Question-Answers|#7 워라밸 치환표]] — "버티다, 견디다, 힘들다" 같은 부정 단어 본인이 먼저 쓰지 말 것. **"의미, 지속 가능성, 구조"로 프레임 재정의**.

**다듬은 본문**:
> 솔직히 말씀드리면 **의미 있는 일이라면 시간 투자는 괜찮습니다.** 트라이포드랩에서도 PoC 데모 전날 밤샘이나 50% 오류율 발견 후 주말 작업을 해봤습니다. 다만 궁금한 건 **지속 가능성**입니다. 12명 팀 + 본 계약 진행 단계에서 **이 강도를 받쳐주는 미션 공감, 팀워크, 성과 인정 구조가 어떻게 구성돼 있는지** 궁금합니다.

**핵심 기교** (Sensitive #7 룰):
- "괜찮습니다" = 절박 / "힘들겠다" = 부정 프레임 → **둘 다 회피**
- **부정 단어 본인이 먼저 쓰지 말 것** ("버틸 수 있나"를 받아 "버티겠습니다" 답변 X)
- **역질문으로 마이크 넘김** + 회사 단계 컨텍스트 매핑
- **자기 페이스 기준 제시** (트라이포드랩 PoC 사례 = 본인이 통제 가능한 강도 구체화)

### 개발 환경, 도구 ★★ ([[My-FIT-Answers#13. 개발 환경, 도구 ★|마스터 13번]])

### 사이드 프로젝트, 취미 ([[My-FIT-Answers-Extended#15. 사이드 프로젝트, 취미|마스터 Extended 15번]])

> 대표 직접 면접에서 "취미, 관심" 질문 가능성. **주일학교 출석부 = 본인이 만든 게 다른 사람 실서비스로 쓰이는 책임감** 어필 강함.

**다듬은 본문**:
> **주일학교 출석부**를 Express 5+tRPC 모노레포로 만들어 운영 중입니다. 2026-07-15 기준 **보고서상 모임 레코드 108개, 랜딩 누적 본당 85곳, 학생 3,669명, 출석 약 22,051건, MAO 34곳**입니다. 누적 본당과 모임 레코드는 활성 수치가 아닙니다. 성당 봉사에서 출석 관리가 종이, 엑셀로 흩어진 걸 보고 직접 풀었고, **Parish→Church→Organization 3단계 위계 테넌트, 명시적 권한 가드, Snapshot 패턴(이력 보존)까지 직접 설계, 운영**했습니다. 작은 트래픽이어도 **멀티테넌트 운영 문제(권한, noisy neighbor, tenant 자동화)를 직접 다뤄온 영역**입니다. 그 외 **하코 3000명 Prisma 발표, 카카오테크 캠퍼스 멘토링**으로 이해를 점검합니다.

**CARE ID 매핑** (★ 강력): 주일학교 멀티테넌트 SaaS 운영 = **윤회 브랜드사 멀티테넌트와 동형 패턴**. Q5 멀티테넌트 답변에 본인 실전 사례 직접 인용 가능 (Parish→Church→Organization 위계 = 윤회의 multi-brand 멀티테넌트 위계와 본질 같음). 단독 운영자 마인드 + 사용자 영향 우선 사고도 백엔드 리드 자리에서 그대로 작동.

**꼬리**: "본업 영향은?" → **"퇴근 후 1시간 정도 + 주말에 시간 내서 합니다. 본업 시간은 침해하지 않게 가드 두고 있습니다."**

# 개발자 면접 단골 질문 13개 · 답변 전략

기술 스택과 무관하게 전 세계 개발자 면접에서 반복적으로 등장하는 13개 질문. 각각은 **하나의 능력**을 확인하려는 목적이 있다. 답변 자체보다 **무엇을 보려 하는지 파악**하고 자기 경험으로 치환하는 것이 중요. STAR 형식(Situation-Task-Action-Result)을 기본 골격으로.

## 질문별 의도와 답변 방향

### 1. "가장 잘 아는 개발 방법론은 무엇인가요?"

- **의도**: 실제 개발 프로세스 경험·개념 이해도
- **답변 축**: 사용 경험 있는 방법론(Agile·Scrum·Kanban·XP) 중 **가장 깊이 있는 것 하나** 선택 → 해당 방법론의 이벤트·산출물·본인 역할을 예시로
- **함정**: 교과서 정의만 나열 → "실제로 써봤나?" 의심

### 2. "소프트웨어 개발 중 문제 발생 시 어떻게 해결하나요?"

- **의도**: 문제 해결 프로세스·체계성
- **답변 축**: 최근 사례 하나 선택 → **재현 → 원인 분석 → 가설 → 검증 → 해결 → 회고**
- **함정**: "바로 해결했다" 단순 영웅담 → 체계가 없다는 인상

### 3. "버그를 찾고 테스트하는 프로세스가 있나요?"

- **의도**: 디버깅·테스트 숙련도
- **답변 축**: 로그·스택 트레이스·바이너리 서치·프로파일러 등 **본인의 도구 세트**와 **단계별 접근법**
- 선호 도구: Chrome DevTools·IDE 디버거·APM·로깅·Git bisect 같은 이름을 자연스럽게

### 4. "코딩에서 가장 흥미로운 점은?"

- **의도**: 일에 대한 내적 동기·지속 가능성
- **답변 축**: **구체적인 경험**에서 느낀 몰입·성취. "문제 해결이 재밌다" 같은 일반론은 얕음
- 주니어는 학습 의지, 시니어는 **임팩트·레버리지** 관점

### 5. "잘했다고 생각하는 프로젝트를 소개해주세요"

- **의도**: 성과를 **객관화**할 수 있는가
- **답변 축**: 비즈니스 임팩트 + 기술적 기여 + **본인의 역할 비중**을 명확히
- **숫자 포함**: "응답 시간 40% 단축", "주간 사용자 2만→5만" 같은 계량 지표
- 함정: 팀 성과를 개인 것처럼 → "우리"와 "제가" 분리

### 6. "잘하지 못했던 프로젝트가 있나요?"

- **의도**: 자기 인식·실패로부터의 학습
- **답변 축**: 실패 사실 인정 + **원인 분석** + **바꾼 행동** + 재발 방지
- 함정 1: "없다" → 성찰 부족
- 함정 2: 남 탓 → 협업 부적합

### 7. "어떤 소스 컨트롤을 사용하나요?"

- **의도**: 협업 습관
- **답변 축**: Git 기반 워크플로(Git Flow/GitHub Flow/Trunk-based), 브랜치 전략, **PR 문화·코드 리뷰 관행**
- "git commit만 써봤어요"는 치명적

### 8. "이 회사 제품에서 가장 좋아하는 기능은?"

- **의도**: 사전 조사 깊이·진정성
- **답변 축**: 실제 써본 경험 + 개발자 관점의 분석(UX 세부, 성능, 아키텍처 추정)
- 함정: "좋은 것 같아요" 상투 → 관심 없음으로 해석

### 9. "비IT 동료와 어떻게 의사소통하나요?"

- **의도**: 커뮤니케이션 적응력
- **답변 축**: 기술 용어 번역 방식, 다이어그램·프로토타입·애널로지 활용, **구체적 에피소드**
- 기획자·디자이너·마케팅·고객지원과 협업한 **사례 1개** 준비

### 10. "개인적으로 진행하는 사이드 프로젝트가 있나요?"

- **의도**: 학습 의지·자발성·Work-Life 균형
- **답변 축**: 있다면 **무엇을 배우고 있는지**, 없다면 **업무 중 학습·지속 채널**(블로그·OSS 기여 등)로 대체
- 강박적으로 꾸며내지 말 것 — 거짓은 꼬리 질문에서 드러남

### 11. "왜 이 회사에서 일하고 싶은가요?"

- **의도**: 지원 진정성·맥락 적합도
- **답변 축**: **제품 → 사명 → 문화 → 본인 커리어 방향**의 연결선
- 구체 근거: 최근 제품 업데이트, CEO 인터뷰, 기술 블로그 포스트 언급
- 함정: "성장 가능성이 커서" (어느 회사든 적용되는 말)

### 12. "이상적인 개발 환경은 무엇인가요?"

- **의도**: 컬처 핏 + 본인의 작업 스타일 이해
- **답변 축**: **본인이 실제로 생산성 높은 조건**(코드 리뷰·페어 프로그래밍·딥워크 시간·원격 비율) + 회사 문화와의 교집합
- 솔직하되 유연성 표현

### 13. "마지막으로 질문 있으신가요?"

- **의도**: 능동성·준비도·관심도
- **답변 축**: **3~5개의 전략적 질문** 준비. 자세한 예시는 [[Global-IT-Interview#추천 질문 5가지|글로벌 IT 인터뷰 공격 질문]]
- "없습니다"는 사실상 **-0.5점**

## 답변 준비 공통 원칙

- **STAR 프레임**: 모든 경험 질문을 Situation·Task·Action·Result로 서술
- **숫자 3개 이상**: 성과를 구체 지표로
- **실패 사례 2~3개**: 작은 실패 + 중간 실패 + 회사에 아직 안 알려진 실패
- **회사 맞춤**: 동일 답변을 회사별로 80% 수정
- **영어 답변**: 키포인트만 암기, 연결은 즉흥. 전체 스크립트 암기는 어색

## 답변 길이 가이드

| 질문 유형 | 권장 길이 |
|---|---|
| 방법론·도구(1, 3, 7) | 60~90초 |
| 경험·프로젝트(2, 5, 6, 9) | 90~120초 (STAR) |
| 동기·문화(4, 11, 12) | 60초 |
| 관심·질문(8, 10, 13) | 60초 + 대화 유도 |

## 흔한 실수

- 모든 질문에 같은 프로젝트만 반복 인용
- 숫자 없는 성과 묘사("많이 빨라졌다")
- 약점 위장("완벽주의라 야근합니다")
- 실패를 남 탓으로
- 마지막 질문 없음/얕음

## 면접 체크포인트

- 13개 질문 각각에 **STAR로 90초 답변** 준비됐는가
- 최소 3가지 이상의 다른 프로젝트 사례가 있는가
- 실패 이야기에 **본인이 바꾼 행동**이 들어 있는가
- "마지막 질문" 리스트 5개 이상
- 지원 회사별로 맞춤화된 버전이 있는가

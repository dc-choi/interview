# 시스템 설계 연습 주제 카탈로그

라이브 코딩·시스템 설계 면접 대비 연습 풀. 각 주제는 [[Live-Coding-Process|5단계 프로세스]](명확화 → 제약 → 접근 → 구현 → 개선)를 동일하게 적용한다.

## Level 1 — 기본 (30~40분)

### 1. 추천 피드 시스템
> "초기 유저에게 맞춤 추천 피드를 제공하라"

핵심 — Cold Start Problem · 태그/카테고리 기반 추천 · 인기도 가중치 · 친구가 적은 유저 대응.

### 2. 알림 시스템
> "유저에게 실시간 알림을 보내는 시스템을 설계하라"

핵심 — Push/Email/SMS 멀티 채널 · 알림 우선순위 · 발송 실패 재시도 · 배치 vs 실시간 · 알림 피로도.

### 3. 좋아요·북마크
> "게시물에 좋아요·북마크 기능을 추가하라"

핵심 — 카운트 정합성(동시성) · 취소 처리 · 캐싱 · 100만 좋아요 핫키 대응.

### 4. 검색 기능
> "게시물 검색을 만들라"

핵심 — Full-text search vs LIKE · 자동완성 · 검색어 랭킹 · 오타 허용 · ElasticSearch 도입 시점.

## Level 2 — 중급 (40~60분)

### 5. 장바구니
> "이커머스 장바구니를 설계하라"

핵심 — 비로그인(쿠키/세션) vs 로그인(DB) 병합 · 재고 연동 · 옵션·수량 변경 · 품절 알림.

### 6. 선착순 쿠폰
> "선착순 100명 쿠폰 발급 시스템"

핵심 — 동시성(중복 발급 방지) · 대규모 트래픽(초당 1만+) · 공정성 · Redis 원자 연산 vs Queue · 100만 명 확장.

### 7. 댓글·대댓글
> "무한 depth 댓글 트리"

핵심 — Tree 구조(adjacency list vs nested set vs path enum) · 삭제 댓글 표기 · 페이지네이션 · 정렬 · 핫게시물 1만 댓글 로딩.

### 8. 이벤트 예약
> "콘서트 티켓 예매 시스템"

핵심 — 좌석 동시성 · 결제 대기 타임아웃 · 취소/환불 · 대기열 · 동시접속 10만/좌석 100.

## Level 3 — 고급 (60분+)

### 9. 실시간 채팅
> "1:1 실시간 채팅"

핵심 — WebSocket vs Polling vs SSE · 메시지 저장·조회 · 읽음 처리 · 오프라인 메시지 · 파일 전송 · 단톡방 확장.

### 10. 동영상 스트리밍
> "유튜브 같은 플랫폼"

핵심 — 대용량 업로드(chunked) · 해상도별 인코딩 · CDN · 시청 기록·이어보기 · 라이브 스트리밍 확장.

### 11. 배달 주문
> "배달의민족 같은 주문 시스템"

핵심 — 주문→결제→조리→배달 상태머신 · 라이더 매칭 · 실시간 위치 · 취소·환불 정책 · 동시 주문 처리.

### 12. 소셜 피드 (타임라인)
> "인스타그램 피드"

핵심 — 팔로우 그래프 · Fan-out on write vs read · 무한 스크롤 · 실시간 업데이트 · 셀럽(팔로워 100만) 처리.

## Level 4 — 물류 특화

### 13. 실시간 재고 관리
> "실시간 재고 관리"

핵심 — 입출고 · 재고 차감 동시성 · 안전 재고 · 이력 추적(이벤트 소싱) · 멀티 창고 동기화.

### 14. 주문 처리 (OMS)
> "주문→배송 전체 플로우"

핵심 — 주문 접수 · 결제 연동 · 재고 차감 · 송장 생성 · 배송 추적 · 상태 변경 이벤트 발행 · 주문 취소 가능 시점.

### 15. 배송 추적
> "택배사 연동 배송 추적"

핵심 — 다중 택배사 API 통합 · API 없는 곳 스크래핑 · 상태 정규화 · 실패·재시도 · 알림 · 1시간 API 장애 대응.

## 공통 설계 패턴

| 패턴 | 등장 주제 | 도구 |
|---|---|---|
| 동시성 제어 | 좋아요·쿠폰·재고·좌석 | Redis 분산락 · 낙관적 락 · DB 원자 연산 |
| 대규모 트래픽 | 쿠폰·예약·주문 | Queue · 캐싱 · Rate Limiting · 큐 워커 분산 |
| 상태 관리 | 주문·배송·예약 | State Machine · Event Sourcing · Outbox |
| 실시간 | 채팅·알림·위치 | WebSocket · SSE · Pub/Sub |
| 검색·추천 | 추천 피드·검색 · 타임라인 | Elastic · 벡터 검색 · Fan-out |

면접에서 등장한 주제를 위 패턴 매트릭스로 분해하면, 첫 5분 명확화 단계에서 어떤 축이 결정적인지 빠르게 잡힌다.

## 3주 연습 플랜

**1주차 (기본 5개)** — 추천 피드 · 알림 · 좋아요 · 검색 · 장바구니
**2주차 (중급+물류)** — 쿠폰 · 댓글 · 예약 · 재고 관리 · OMS
**3주차 (고급+물류)** — 배송 추적 · 채팅 · 동영상 · 배달 주문 · 소셜 피드

각 회: 60분 타이머 + 말하면서 코딩 + 종료 후 5분 자기 회고.

## 역분석: 실 서비스 매핑

- 네이버 쇼핑 → 장바구니·주문
- 쿠팡 → 재고·배송
- 인스타그램 → 피드·댓글
- 유튜브 → 동영상·추천
- 배민·요기요 → 배달 주문

"이걸 어떻게 만들었을까"를 역으로 풀면 면접에서도 사례 인용이 자연스러워진다.

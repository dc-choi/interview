## 2. 회사 맞춤 FIT 답변

> [[Common-Interview-Questions|FIT]] 기반으로 이 회사에 맞게 커스터마이징한 답변

### 1분 자기소개
> 구조: 이름/역할 → 도메인 경험 요약 → 핵심 성과 → 핵심 철학 → 왜 이 회사 → 마무리

(회사 도메인과 연결되는 경험 중심으로 재구성)

### 지원 동기
> 구조: 회사의 미션 이해 → 내 관련 경험 → 연결점 → 기여하고 싶은 것
> 회사가 하는 사업과 중요하게 생각하는 일을 내가 계속하고 싶은 이유에 연결한다. 비전, 인재상과 기여하고 싶은 바를 묶어 단기간에 퇴사할 사람이라는 우려를 줄인다.

(회사가 풀고 있는 문제 + 내 경험의 접점)

### 이직 사유
> 구조: 현 회사에서 충분히 성장한 점 → 한계를 느낀 지점(**자기 기준**으로) → 다음 도전 목표 → 지원 회사에서만 가능한 이유
> 먼저 [[My-Motivation-Reasons#0. 현재 사이클 변수 (이직할 때 이 블록만 교체)|현재 사이클 변수]]와 [[Sensitive-Question-Answers|현재 회사 퇴사 이유 작성 규칙]]을 확인한다. 다음 사이클 슬롯이 비어 있거나 현재 이직 사유가 확정되지 않았다면 사용자에게 먼저 묻고, 종료된 과거 사이클을 현재 답변으로 재사용하지 않는다. 현재 사이클이 확정된 뒤에만 B2C/B2B 분기를 지원 회사 맥락에 맞춰 사용한다.
> **외부 귀책 주의**: "그 회사가 X였다"만으로 끝내지 말 것. 장기근속 리스크 시그널. 자기 귀책 + 학습 + 현재 적용으로 재작성. [[My-Motivation-Reasons#5. 공통 원칙 (모든 이직 사유에 적용)|답변 원칙]] 참고

## 3. 예상 질문 (면접관 → 나)

> 질문마다 유형과 의도를 먼저 적고, 아래 항목 중 해당하는 답변 구조를 사용한다.
> 사실형은 핵심 답변, 흔한 오답, 꼬리 질문, 학습 정본이나 공식 출처를 포함한다.
> 설계형은 단일 정답 대신 답변 골격, 판단 기준, 대안과 트레이드오프, 꼬리 질문을 포함한다.
> 회사 문서에는 vault 링크만 남기지 말고 면접에서 말할 수 있는 핵심 내용을 본문에 흡수한다.

### 이력서 기반 기술 질문
> 이력서/포트폴리오 성과를 깊게 파는 질문. "어떻게?", "왜?", "다른 선택지는?" 패턴
> 포트폴리오 STAR 형식의 구체적 수치와 의사결정 과정을 활용

#### 질문 제목

- 질문 유형: 사실형/설계형/경험형
- 질문 의도
- 핵심 답변 또는 답변 골격
- 판단 근거와 구체적인 수치
- 사실형일 때: 흔한 오답, 학습 정본이나 공식 출처
- 설계형일 때: 판단 기준, 대안과 트레이드오프
- 꼬리 질문 대비

### JD 기반 기술 질문
> 자격요건/우대사항에서 파생되는 질문
> 각 질문에 실무 경험 연결 + 꼬리 질문 대비 포함

#### 질문 제목

- 질문 유형: 사실형/설계형/경험형
- 질문 의도
- 핵심 답변 또는 답변 골격
- 실무 경험 연결: 어떤 프로젝트에서 어떻게 적용했는지
- 사실형일 때: 흔한 오답, 학습 정본이나 공식 출처
- 설계형일 때: 판단 기준, 대안과 트레이드오프
- 꼬리 질문 대비

### 서비스 맥락 질문
> "우리 서비스에서는 이런 상황인데, 어떻게 하겠는가?" 패턴
> 내 실무 경험과 연결하여 답변 + 꼬리 질문 대비

#### 질문 제목

- 질문 의도
- 답변 골격과 설계 방향
- 판단 기준
- 대안과 트레이드오프
- 실무 연결 (유사 경험)
- 꼬리 질문 대비

### 컬처핏 / 소프트스킬 질문
> [[Common-Interview-Questions|FIT]], [[Self-Analysis|나찾기]] 기반으로 이 회사에 중요한 질문을 최소 5개 고른다.
> 일반 `prep/` 질문 풀에는 준비 포인트와 `common/` 링크만 두지만, 생성한 회사별 문서에는 선택한 마스터 답변 본문을 fork하고 회사 연결 끝줄을 붙여 자기완결적으로 만든다.

선택한 항목마다 다음 형식으로 확장한다.

#### 질문 제목

- 질문 의도
- 준비 포인트
- `common/` 마스터 fork 답변
- 회사 연결 끝줄
- 꼬리 질문 대비

- [ ] 장단점
- [ ] 동료 갈등
- [ ] 기획자 충돌
- [ ] 긴급 이슈 판단
- [ ] 번아웃 관리
- [ ] 성장 목표
- [ ] 조직문화
- [ ] (회사별 추가 항목)

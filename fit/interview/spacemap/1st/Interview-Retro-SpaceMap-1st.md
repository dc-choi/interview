## 진행 흐름 (실제 받은 질문)

### 인적, 자기소개
- 96년생, 성남 모란 거주, 판교 현 직장(재직 중), 통근 1시간
- 자기소개 → 4년차, VMI, 물류, 보안, 데이터 정합성, 이벤트 아키텍처, Docker/DB 성능 개선. 지원 이유 "방대한 우주 데이터로 데이터+백엔드 경험"

### 기술 검증
- **Q: "데이터 파이프라인 처리"가 뭔가?** → 솔직 정정: "흔한 데이터 파이프라인이라기보다 **데이터 정합성** 중심. IoT HTTP 요청 race condition → 비관적 락 직렬화"
- **Q: 카프카→EventBridge 변경?** → 카프카 도입 안 함. 발주 도메인 이벤트 디커플링, MSK 비용, 트래픽 작음, 프리티어 → EventBridge+SQS
- **Q: 서브쿼리 문제, 해결(Prisma)?** → app join 미지원 → N테이블 개별 쿼리 앱조인 레이턴시 2초 → relationLoadStrategy(MySQL JSON 파싱), 실행계획 cost 우수 확인
- **Q: NoSQL 경험?** → 시솔 몽고, "MySQL보다 전문성 떨어짐" 솔직
- **Q: SQL vs NoSQL 원칙?** → 스키마 자주 변경=NoSQL, 정형=RDBMS. 요즘은 **Postgres JSON으로 커버 후 필요시 정규화**
- **Q: 정합성, 타입 변화 처리?** → 정규화, 데이터 모델링, RDBMS. 배치는 **Airflow 경험 없음**, Spring Batch(실패 지점 재시작) 언급
- **Q: 그라파나/프로메테우스/로키 모니터링 구축 내용?** → ECS, ELB, RDS, 초기 단일서버 Loki → 서버 증가로 FireLens 도입
- **Q: 모니터링 도입 이점(비즈니스 결부)?** → 백엔드 가시성 소통 문제 해결, Grafana Alerting 선제 대응 / 비즈니스 = **자동 발주(VMI)**: IoT 무게센서 → 발주점 도달 시 자동 발주. 대형 제약사, 200억 FMCG PoC→정식 도입
- **Q: 시솔지주 업무?** → FIDO/패스키 PKI 서버 보안 인증 → 회사 피벗팅으로 물류 SI/SM, 배송 스크래핑, 통관 외부 API
- **Q: 시솔 이후 교육 의사결정?** → 시솔 **폐업→권고사직(경영상 해고)**. "재무, BM 확실한 회사, 사업 방향성도 보는 개발자" 학습. 부트캠프 온/오프 병행 → 스프링 리마인드(NestJS가 스프링 철학 닮아 도움)
- **Q: NestJS 장단점/아키텍처 프레임워크 장점?** → 아키텍처 잡힘, 소통비용↓, 비즈니스 집중. 단점: 스프링보다 생태계 적음. AI로 TS 풀스택 프로덕트 엔지니어 가능
- **Q: 헥사고날 구현?** → **이론만, 구현 X**. 외부 API 적어 헥사 분할 불필요, 클린 아키텍처 유스케이스만 적용
- **Q: 선호 프레임워크?** → 상황에 따라. 대용량 배치=Spring Batch가 정답이나 팀 상황 따라 NestJS+BullMQ/PGQ
- **Q: 코딩 좋아하나?** → 좋아함. 단 AI로 손코딩 안 한 지 오래, 거시적 방식 변화. "우주는 손으로 할 일 많을 듯"
- **Q: AI 툴?** → 클로드 코드 + **Conductor 병렬** + 코덱스 검증(클로드가 코덱스 호출). 스펙 드리븐. 클로드는 회사 지원, 코덱스 본인 부담($20→$100)
- **Q: CTO(영상 출신, 곧 환갑)와 세대차 충돌?** → 바텀업 주도, 백엔드 세부는 본인 권한, CTO 인프라/리서치 지원 → 시너지 좋음

### 확인사항

## ✅ 잘 작동한 카드

1. **솔직한 정정** — "데이터 파이프라인이라기보단 정합성 중심" / NoSQL, 헥사고날, Airflow 경험 부족 인정. 5년차 자리에서 아는 척 안 함 (시그널 부합)
2. **EventBridge+SQS 선택 근거** — MSK 비용, 트래픽, 프리티어 정량 비교, 발주 도메인 최종 일관성 OK
3. **Prisma relationLoadStrategy 깊이** — Raw query 대신 ORM 유지 + 실행계획 cost 검증
4. **AI 워크플로우 깊이** — Conductor 병렬, 코덱스 검증 호출, 스펙 드리븐. 면접관 "적극적으로 받아들인다" 반응
5. **CTO 세대차 질문 대응** — 부정 없이 "바텀업, 권한, 시너지"로 긍정 전환
6. **역질문 능동성** — 아키텍처를 깊게 파고들고, **모니터링 방치 지적 → 지표 설정, 에러 트레이싱, ElasticSearch 기여 제안**

## ⚠️ 다듬을 점

1. **데이터 파이프라인 경험 부족을 반복 자인** — JD 정중앙인데 자기소개 직후, 중간, 마지막 말까지 "흔한 파이프라인 아님/Airflow 경험 없음/데이터 처리 경험 부족" 3회. 솔직함은 좋으나 **핵심 요건에서 약점이 과노출**. "빠른 적응"으로만 방어. → 다음(경영진)엔 "정합성, 이벤트, 성능 = 데이터 엔지니어링의 토대"로 재프레이밍 필요
2. **STM 도메인 매핑 미발화** — 준비한 STM(conjunction/Pc/시계열) 카드를 자기소개에서 못 씀. "방대한 우주 데이터" 수준. 면접관이 도메인 안 물어 기회 적었으나, 2차에선 먼저 꺼낼 것
3. **시솔 이직 사유 외부 귀책 톤** — "회사 폐업→경영상 해고"를 그대로. 사실이나 마스터 옵션 B(자기 학습 격상)와 결이 다름. 2차 대비 톤 정리
4. **희망연봉 "5년차 테이블"** — 4년차가 5년차 요구. 직전 4천 중반 근거는 있으나, 1년 계약직 구조와 맞물려 협상 변수

## 결정 (현재 시점)

> ⚠️ **핵심 신호 — 질문 난이도 = 미들급, 역할 스코프 = 실행 핸즈.** 받은 질문(서브쿼리 N+1, SQL vs NoSQL 언제, NestJS 장점, 헥사고날 구현했나, "코딩 좋아하세요?")은 **3~4년차 스크리닝 수준**이지 5년차 질문이 아니었음. 시스템 디자인, 트레이드오프 의사결정, 장애 포스트모템, 팀 리딩 같은 시니어 깊이 질문 **0건**(아키텍처 깊이 대화는 본인이 역질문으로 끌어낸 것). 해석: **"5년+"는 라벨, 실제 역할은 "C++ 엔지니어 산출물 파이프라인 연결 + API + 모니터링 고도화"인 미들급 실행 핸즈**(면접관 본인이 그렇게 스코프). → 미들급 실행 역할 + 1년 계약직 + 본인 "5년차 테이블" 희망이 겹쳐 **역할 스코프, 처우 간극** 존재.

### 핵심 키워드 (면접에서 반복 등장할 것)

- **데이터 수집, 처리 파이프라인** — JD 주요 업무 첫 줄. 가장 중요한 축
- **데이터 엔진** — 우주 데이터를 가공, 연산하는 백엔드 핵심
- **우주 데이터** — 대용량 시계열(궤도, 추적 로그) 추정. IoT 시계열 데이터 경험과 매핑
- **API 서버** — 데이터 엔진 결과를 웹/외부에 노출
- **강한 몰입, 업무 강도** — 컬처핏에서 솔직하게 다룰 것

## 2. 회사 맞춤 FIT 답변

### 1분 자기소개
> 구조: 이름/역할 → 도메인 경험 → 핵심 성과 → 핵심 철학 → 왜 스페이스맵 → 마무리

안녕하세요, 4년차 백엔드 개발자 [이름]입니다.

트라이포드랩에서 NestJS 기반 VMI 서비스를 설계, 운영하면서, **데이터를 수집하고 처리해 API로 노출하는 파이프라인 전 구간**을 다뤘습니다. 수천 대 IoT 디바이스가 실시간으로 보내는 데이터를 DB Lock으로 정합성 있게 수집하고, EventBridge, SQS 기반 이벤트 아키텍처로 후속 처리를 분리했으며, 1억 건 규모 데이터에서 복합 인덱스 최적화로 슬로우 쿼리를 99.3% 개선했습니다. 운영 단계에서는 Grafana, Prometheus, Loki 관측 인프라를 직접 구축했습니다.

저는 **데이터 규모가 커져도 깨지지 않는 구조**를 만드는 데 강점이 있습니다. 스페이스맵이 다루는 우주 데이터가 대용량, 시계열 성격이라면, 제가 IoT 데이터 파이프라인에서 쌓은 수집, 정합성, 성능 경험이 그대로 연결된다고 봤습니다. 연차로는 5년에 1년 모자라지만, 다룬 문제의 깊이와 범위로 충분히 기여할 수 있다고 생각해 지원했습니다.

### 지원 동기
> 구조: 회사의 문제 → 내 경험 → 연결점 → 기여하고 싶은 것

- **회사의 문제**: 스페이스맵은 우주 데이터 — 위성 궤도, 추적 로그처럼 끊임없이 쌓이는 대용량 시계열 데이터 — 를 수집, 연산해 의미 있는 결과로 만드는 "데이터 엔진"이 핵심. 미 우주군, 우주청 과제를 수행하는 만큼 정합성과 안정성 요구가 높은 단계.
- **내 연결점**: 트라이포드랩에서 정확히 같은 문제를 풀었음 — 수천 대 디바이스의 실시간 데이터를 유실, 중복 없이 수집하고, 데이터가 1억 건으로 쌓여도 인덱스 1탐색으로 끝나는 구조를 설계했고, 수집, 가공, 알림을 이벤트로 분리했음.
- **기여하고 싶은 것**: 데이터 수집, 처리 파이프라인이 데이터 규모 증가에도 안정적으로 돌아가도록 만드는 일. JD의 "데이터 수집, 처리 파이프라인 및 API 서버"가 제 경험의 정중앙에 있음.

### 이직 사유
> 구조: 충분히 성장한 점 → 한계(자기 기준) → 다음 도전 → 스페이스맵에서만 가능한 이유
> **외부 귀책 금지** — 회사 탓 아닌 자기 성장 기준으로

- 트라이포드랩에서 0→1 구축, 대형 고객사 PoC 성공, 단일 서버→ECS 전환, 관측 인프라 구축, AI 협업 파이프라인 재설계까지 — 백엔드와 인프라 전반을 폭넓게 경험하며 충분히 성장.
- 다만 VMI/물류 도메인의 데이터는 비즈니스 트랜잭션 중심이라, 데이터 *자체의 복잡도*보다 *프로세스 복잡도*가 컸음. 다음 단계에서는 **데이터 자체가 어렵고 깊은 도메인** — 연산, 정합성, 규모가 본질인 영역에서 성장하고 싶음.
- 스페이스맵의 우주 데이터는 그 자체로 난이도가 높은 데이터(궤도 역학, 시계열, 대용량). 제가 쌓은 데이터 파이프라인, 성능 최적화 역량을 *더 어려운 데이터*에 적용하며 한 단계 깊어질 수 있는 환경.
- 꼬리 대비:
  - "우주 도메인 지식이 없는데?" → 도메인 지식은 학습 대상, 데이터 파이프라인 설계 역량은 전이 자산. 학습 의지는 새 언어, 새 도메인을 반복 흡수해온 이력(Java→Node, JSP→Spring, MongoDB→MySQL, AI 파이프라인)으로 증명.
  - "5년차 자리인데 4년차가 왜?" → 연차 숫자가 아니라 다룬 문제로 평가받고 싶음. (구체 근거는 Service 문서 #6)
  - "지금 회사 더 다니지?" → 성장 곡선의 다음 변곡점이 *데이터 난이도가 높은 도메인*이라 판단. 회사에 대한 불만이 아니라 내 커리어 설계상의 선택.

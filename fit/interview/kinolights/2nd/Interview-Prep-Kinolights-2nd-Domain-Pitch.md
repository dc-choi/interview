# 키노라이츠 2차 도메인 오너십 피치 (1차 녹취 기반)

> 출처: 1차 실무 인터뷰 녹취(클로바노트, 26.06.08, 116분) 26.06.10 확보. 회고([[Interview-Retro-Kinolights-1st]])에 압축되며 빠진 **후반 30분 — 리더가 직접 깔아준 도메인 메뉴**를 2차 열정 카드로 변환한 문서. 본편 [[Interview-Prep-Kinolights-2nd]] §2~3과 페어.
> 핵심: 전부 **1차에서 본인이 이미 꺼낸 카드와 붙는 도메인**이라 톤 일관성(본편 §1 원칙)을 깨지 않고 열정을 올릴 수 있다.

## 1. 리더가 1차에서 직접 깔아준 것 (녹취 요지)

| 리더 발언 | 함의 |
|---|---|
| 콘텐츠 서버가 되게 중요한 서버 — OTT 데이터 원천. 지난달 넷플릭스 오리지널 확보에 매진 | 회사 우선순위 1순위가 영화 데이터 파이프라인 |
| 스크래핑은 날 HTML이 아니라 API 엔드포인트를 찾으려 노력. 사이트마다 로그인, 프록시 특성 다름. **네이버 프록시 개편 필요, 로버스트하게 안 되는 사이트 한두 곳** | 본편 §2 개선 아이디어(가용성 최신성)가 **가설이 아니라 실재하는 난제로 확인됨** |
| 결제를 넣고 싶은데 아직 없다. 포인트 서버를 결제로 확장할 고민 | 결제 0→1 기회 |
| 소셜 로그인이 손 못 댄 레거시 — 나이스하게 정리하고 싶다 | 인증 도메인 정리 수요 |
| 노티피케이션 — 가입자 수가 꽤 돼서 푸시 개선 여지 | 대량 발송 개선 수요 |
| **각각의 도메인을 자기 오너십을 가지고 맡아줬으면. 책임감 갖고 주도적으로 이끌 사람이면 좋겠다. 선도해서 PoC도 보여줄 수 있으면** | 채용 기대치 = 도메인 오너 + PoC 선도 |
| 과거를 청산했다면 이제 미래로 나가야 되는 상태 (모노레포 통합, 패키지 최신화 완료) | 합류 시점 = 미래 단계의 시작 |

대표(1차 후반 합류한 조직 리드 — 2024년 영입된 공동대표 공동대표 B일 가능성 높음, [[Interview-Prep-Kinolights-2nd-Culture-Map|회사 레퍼런스]] §1)도 직접 말함: 생산성이 좋아진다면 AI는 끝도 없이 지원할 생각, 맥락 싱크와 투명한 정보 공유 강조. 그 사람 앞에서 1차 대화를 구체적으로 인용하면 경청과 진정성 신호가 두 배가 된다.

## 2. 도메인 메뉴 ↔ 내 카드 매핑 (전부 1차에서 꺼낸 카드)

| 키노 도메인 | 내 카드 (1차에서 말한 것) | 2차 한 줄 |
|---|---|---|
| **콘텐츠 서버 (스크래핑, 프록시)** ★ | 시솔지주 택배 조회 — 택배사 API가 아니라 스크래핑이 실체였고, 오류 나면 바로 감지해 바꿔줘야 했던 경험. 1차에서 **본인이 먼저 꺼낸 카드** | 1차에서 콘텐츠 서버와 프록시 개편 얘기를 듣고 제일 흥미로웠다. 키노에 떴는데 실제론 내려간 순간 검색 신뢰가 깨지니, 이 데이터 최신성이 제품 신뢰의 바닥이라고 본다 |
| **소셜 로그인, 인증 정리** ★ | FIDO 패스키 인증 서버 — CTO도 모르던 영역을 웹 표준 문서부터 읽고 일정까지 주도해 리드. 1차에서 리더십 사례로 말함 | 인증은 제가 표준 문서 레벨부터 주도해본 도메인이라 바로 오너십을 가질 수 있다 |
| **포인트 → 결제 확장** | 분산락 답변에서 결제 = 외부 API 트랜잭션 분리 사례로 언급 + Decimal.js 금액 정합성(금액 틀어지면 신뢰 즉사) | 발주 금액 정합성을 지켜본 경험이 결제 도메인의 감각과 같다 |
| **노티피케이션 개선** | 발주 알림 — EventBridge + SQS + DLQ 채널별 분리, 단일 실패 차단 | 대량 발송의 디커플링과 실패 격리는 제가 짜본 구조다 |

★ 두 개가 주력. 결제와 노티는 물으면 받치는 용도.

## 3. 2차에서 쓰는 법 (대표 상대 — 기술 디테일이 아니라 제품 언어로)

**합류하면 가장 하고 싶은 것 질문에 (30초 구어체)**:
> 1차 때 리더님께서 콘텐츠 서버 얘기를 해주셨는데, 그게 제일 기억에 남습니다. 결국 키노의 검색 신뢰는 키노에 뜬 정보와 실제 OTT 상태가 일치하느냐에 달려 있는데, OTT마다 사이트 특성이 다 달라서 이걸 로버스트하게 유지하는 게 어려운 문제라고 들었습니다. 제가 전 직장에서 택배 조회를 스크래핑으로 운영하면서 오류가 나면 바로 감지하고 바꿔줘야 하는 일을 해봐서, 이 문제는 제 경험이 바로 쓰이는 자리라고 느꼈습니다. 그리고 소셜 로그인 정리가 필요하다는 말씀도 있었는데, 인증은 제가 표준 문서부터 주도해본 도메인이라 오너십을 갖고 맡고 싶습니다.

(꼬리로 격상 — 대표가 그 데이터가 왜 중요한가로 받으면) 이 콘텐츠 데이터는 B2C 검색 신뢰의 바닥이면서 카카오, LG유플러스 같은 B2B 메타데이터 유통까지 받치는 자산이라, 정합성이 곧 매출이라고 봅니다.

포인트 세 개만 지키면 된다:
1. **1차 대화 인용으로 시작** — 들은 것을 정확히 기억하고 있다는 것 자체가 경청, 진정성, 그리고 이미 합류한 사람처럼 생각하고 있다는 신호.
2. **사용자 가치로 번역** — 대표 상대니까 프록시나 DLQ가 아니라 검색 신뢰, 사용자 이탈의 언어로. 기술 디테일은 물으면만 내려간다.
3. **오너십 어휘로 끝맺기** — 리더의 기대치(도메인 오너, PoC 선도)와 인재상(주도적 문제해결)에 정확히 꽂히는 단어다.

**주의**: 컬처 라운드에서 기술로 길게 빠지지 않는다. 위 30초로 던지고, 대표가 더 파면 그때 §2 표의 받침 카드로. 검색추천 관심(1차에서 말한 지원동기)과 충돌하지 않는다 — 검색추천이라는 목적지로 가는 길에 콘텐츠 데이터 최신성이 바닥이라는 구조로 잇는다.

## 4. 인원 데이터와 합류 의미 (녹취 1:38 부근)

| 확인된 것 | 함의 |
|---|---|
| 전체 25명, 제품조직 5명, 백엔드 2명(시니어 1 + 신입 1), 프론트 시니어 1, MAU 100만 | 개발자 3명이 100만 MAU 운영 — AI 레버리지를 전제한 조직. [[Interview-Prep-Kinolights-2nd-Drill\|드릴 §A]] 키노 숫자로 닫기의 근거 |
| 리더: (백엔드 시니어가) 혼자 다 하고 계십니다 | 실질 1인 + 신입 체제. 합류 = 사실상 **두 번째 시니어 축** — 사수 있음(깊이)과 즉시 영향력이 동시에 성립. 윤회 비교표([[Interview-Prep-Kinolights-2nd-Drill\|드릴 §B]])의 팀 합류 프레임이 약간 보정됨: 기다리는 깊이가 아니라 들어가자마자 큰 축 |
| 리더: 다른 데도 인력 채용을 계속하고 있다 | 채용 의지 실재 — AI 인력 논쟁은 채용 부정이 아니라 적정 판단 밀도 탐색. 같은 편 톤으로 받으면 됨 |

역질문 백업 추가: **제가 합류하면 어떤 순서로 도메인을 인수받게 되나요** (온보딩, 지식 전수 방식) — 시니어 1인에 운영 지식이 집중된 구조의 리스크를 검사 톤 없이 확인하는 질문. 리더가 말한 3개월, 1년 기대치(도메인 오너 + PoC 선도)와 자연스럽게 잇는다.

## 5. 같이 확인된 것 (본편, 드릴 갱신 근거)

- 본편 §2 제품 개선 아이디어(가용성 최신성)는 이제 **가설이 아니라 리더가 확인해준 실제 난제** — 2차에서 자신 있게.
- AI 도발 중 1차에서 실제 나온 새 패턴: **의사결정하는 날이 1년에 며칠이나 되나, 나머지 8시간은 뭘로 채우나** — [[Interview-Prep-Kinolights-2nd-Drill|드릴 §A]] 트리에 추가됨.
- B2C 야간 대응 수용 의사는 1차에서 이미 말함(당직 문화 인지, 받아들일 준비) — 2차에서 일관 유지.

## 6. 일하는 철학 매치 (가장 강한 컬처 카드 — 1차 최고 카드의 회사 맞춤 증거)

리더가 1차에서 직접 푼 본인의 의사결정 이력: 잘게 나뉜 마이크로서비스를 한두 개 빼고 **전부 모노레포로 통합**(한두 명이 운영하기엔 잘게 나눈 MSA가 오히려 불편, 유지가 안 됨), 멀티 데이터소스 일반화, 패키지 최신화, **DB와 파게이트 다운사이징 후 성능 문제없음**, 성능과 비용은 각자의 양심으로.

이게 본인이 1차 발표에서 말한 결정들과 정확히 동형이다: EKS 오버엔지니어링 회피하고 ECS, MSK 월 570달러 안 쓰고 프리티어 내 처리, 과도한 추상화 덜어내고 트랜잭션 스크립트, 인스턴스 라지에서 다운사이징(너무 당연해서 이력서에 안 적었다고 한 그것).

**2차 한 줄 (왜 우리 회사, 작은 회사 괜찮나, 팀과 맞을 것 같나 계열 질문에)**:
> 1차에서 리더님이 마이크로서비스를 통합하고 인프라를 다운사이징했다는 얘기를 들었을 때, 팀 규모와 운영 부담에 맞춰 기술을 고르는 철학이 저와 같다고 느꼈습니다. 저도 화려한 스택보다 지금 조직에 맞는 선택을 해왔던 사람이라, 이 팀이면 기술 논의에서 같은 언어를 쓰겠다는 확신이 들었습니다.

기술 열망 없다, 회사가 필요하면 쓴다는 1차 최고 카드를 **키노 실제 사례로 받친 버전**이라 톤 일관성이 완벽하다.

## 7. 보조 카드와 주의 (녹취 기타 캐치)

- **데이터로 설득 보조 사례**: k6 부하테스트 비용을 회사가 뭐라 안 했냐는 질문에, 관측 인프라를 먼저 깔아 조직이 데이터의 필요성을 이미 이해한 상태였다고 답함 — 설득과 협업 질문에서 환불 정책 사례 다음 보조로 쓸 수 있는 실증.
- **개인 AI 투자 카드**: 회사 Claude Max에 더해 본인 자비로 Codex 월 100달러(좋아하는 일이면 쓴다) — 대표의 AI는 복지가 아니라 생산성, 생산성이면 끝도 없이 지원 철학과 직결되는 내적 동기 증거. ⚠️ 단 1차에서 회사에 딱히 말씀 안 드렸다는 뉘앙스가 있었으니, 2차에선 그 부분 빼고 좋아하는 일이라 자비로도 투자한다는 톤으로만.
- **재택/대면 일관성**: 1차에서 소통과 회의는 대면 선호(비언어 소통), 재택은 혼자 집중 작업용이라고 말함 — 2차에서 자율 재택 복지를 반기는 톤으로 뒤집지 말 것. 현재 입장이 키노의 자율 운영 철학과 이미 자연스럽게 맞는다.
- **2차 압박 예측**: 1차의 대표(조직 리드)는 압박형이 아니라 맥락 설명형이었다. 압박이 재연될 가능성이 가장 큰 주제는 AI 인력 — [[Interview-Prep-Kinolights-2nd-Drill|드릴 §A]]로 커버 완료.

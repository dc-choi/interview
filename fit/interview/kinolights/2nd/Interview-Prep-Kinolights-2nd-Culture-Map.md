# 키노라이츠 인재상과 Work Rules 매핑 (2차 컬처)

> 출처: 공식 채용 페이지(Careers 인재상, Work Rules, Benefits, 백엔드 JD) 26.06.10 확보. 본편 [[Interview-Prep-Kinolights-2nd]]의 답변을 **회사가 직접 명시한 평가 축**에 못 박는 문서. 인재상은 CEO가 채용 기준으로 쓰겠다고 공언한 텍스트라, 2차(공동대표 2명 + 인사)는 사실상 이 축으로 평가된다고 가정한다.

## 1. 인재상 = 일이 되게 하는 사람

정의 두 줄: ① 조직의 목표 달성에 집중 ② 결정된 사항에 대해 상황에 따라 **리더십이나 팔로워십을 유연하게** 발휘.

| 축 | 회사 문구 핵심 | 내 카드 (1차에서 이미 쓴 것 위주) |
|---|---|---|
| 주도적 문제해결 | 문제가 무엇인지 먼저 제대로 파악 → 가장 잘 해결할 방법에 몰두 | 외주 레거시 정상화 — 문제 정의(키매핑 깨짐, 기능 미동작)부터 잡고 실DB 통합테스트로 규칙을 못박음. Prisma 성능 저하를 실행계획까지 파고든 검증 |
| 높은 책임감 | 결정되면 그 상황 안에서 최적 실행을 함께 고민. **근거 기반 비판은 OK, 감정적 냉소와 비아냥, 방관, 체념은 지양**. 작은 약속과 팀 규칙 준수 | 환불 정책 충돌 — 감정이 아니라 오차 비용 데이터로 비판해 결정으로 정리. 신입이 남긴 스택(Docker, GHA, Prisma)을 깎아내리지 않고 최소 변경으로 살림(전임자 비난 0) |
| 협업 | 팀보다 위대한 개인은 없다. 동료 존중과 배려로 최적 성과 | 비IT 동료와 기획 의도 먼저 묻기, 합의를 테스트 코드로 못박아 인식 차 제거(박람회 정상화) |

**리더십/팔로워십 유연 = 이직 사유 서사와 동형.** 단독 주도성(리더십)은 충분히 검증됐고, 이제 팀과 합(팔로워십)을 맞추며 깊이를 더하고 싶다 — 대표 앞에서 이 단어 쌍을 그대로 쓰면 인재상 어휘로 정확히 꽂힌다.

### 금지 축 (회사가 명시한 감점 단어)

감정적 냉소, 비아냥, 방관, 체념. 전 회사, 전 동료, AI 논쟁 어디서도 이 톤 금지 — 외부 귀책 0 원칙과 같은 축이다. 비판이 필요하면 반드시 근거(데이터, 비용)가 먼저 보이게.

## 2. Work Rules 공감 포인트 (자연스럽게 쓸 연결 한두 개만)

| 회사 제도 | 내 연결 |
|---|---|
| 투명한 정보 공유 — 첫날부터 전 정보 접근, 노션 회의록, 비담당자도 회의 참여와 의견 | [[Self-Analysis|ISTP]] 잘 맞는 환경(논리와 근거 중심 피드백)과 직결. 공감 멘트나 역질문 소재 |
| 일일 스크럼 — 혼자 고민하는 시간을 길게 두지 않는다 | 단독 운영의 한계(피드백 루프 부재)를 채우는 문화. 이직 사유의 팀과 합 서사를 회사 제도가 그대로 받쳐줌 |
| 타운홀 마지막에 상상마당 홍대 시네마에서 인생영화 관람 | 키노가 극장(상상마당 홍대)을 직접 운영 — 영화 투자배급과 콘텐츠 IP 확장의 실물 근거. 도메인 진정성 멘트와 §3 콘텐츠 사업 역질문의 구체 근거 |
| 1on1 — 월 1회 리더, 2주마다 동료, 자유 티타임 | 솔직한 회고 문화 카드(입사하면 팀에 주는 영향)와 연결 |

주의: 제도를 줄줄 외운 티 내지 말 것. 흐름에 맞는 한두 개만.

## 3. 백엔드 JD 공식 서술에서 얻은 새 근거

- **회사가 스스로 정의한 핵심 과제**: 빠른 제품 개발 속도와 안정적 운영의 양립. 새 기능을 빠르게 만드는 일과 오래 안정적으로 운영하며 점진 개선하는 일 — 둘 다에 흥미를 느끼는 사람을 찾는다고 명시. → 합류하면 하고 싶은 것 답변(검색추천 품질 + 운영 안정성)을 이 양립 문구의 결로 말하면 JD와 1:1.
- **서버와 인프라/DevOps를 분리하지 않고 한 팀이 책임**, 외부 의존이나 분업 경계 없이 기술 판단이 곧바로 서비스에 반영. 우대에 **중소 규모 조직에서 백엔드와 인프라를 함께 책임져 본 경험**까지 명시 — 본인 0→1 단독 설계와 운영 서사 그 자체. 메인 서사가 JD 요구와 동형임을 확인.
- **스택에 EventBridge, SQS, Batch 명시** — 본인 발주 자동화의 EventBridge + SQS + DLQ 조합과 동일 계열. 1차 E구간(폴링 vs 인터럽트)에서 흔들린 영역이 키노 실스택이므로, 기술 얘기가 스치면 역반영된 한 줄(인메모리 에미터는 인스턴스 간 전파 불가 → 외부 브로커 필연)로 깔끔하게. 그 외 DynamoDB, ElastiCache, OpenSearch, WAF, NCloud, serverless framework, release-please, Monorepo, Apollo Studio, Datadog, Amplitude, OpsNow360.
- **프로세스 공식 확인**: 2차 컬처(약 1~1.5h, CEO와 HR 총 3명) → 레퍼런스 체크 **최소 3명**(전 직장 동료, 상사, 후배 / 미동의 시 절차 제한 가능) → 처우협의. 수습 3개월, 수습 중 처우 동일.
- **보상 철학 Pay to Win** — 회사 성장 단계에 따라 개개인 최고 보상을 고민한다고 공식 명시. 평균연봉이 낮다는 외부 데이터와 별개로 회사가 보상 의지를 문서화해 둔 것이라, 처우협의 단계에서 시니어 별도 협의를 받칠 공식 근거.
- **AI 툴 구독 지원 공식 명시** — 1차에서 확인한 Claude Max 지원과 일치. AI 동의 톤(본편 §2)의 공식 근거.

## 4. 액션 (6/15 전)

- [ ] **레퍼런스 3명 추리고 사전 동의 받아두기** — 동료, 상사, 후배 조합. 협조 속도가 결과 시점(6/24 데드라인, 드릴 §B)을 당기는 레버다.
- [ ] 이직 사유 답변에 리더십/팔로워십 단어 쌍 한 번 넣는 연습.
- [ ] 팔로워십 사례 1개 확정 — 내 의견과 다르게 결정됐지만 결정 후 최적 실행에 집중한 일화(환불 정책 사례를 이 결말로 변형하거나 별도 사례). 인재상 책임감 축의 직격 대비.

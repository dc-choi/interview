## 2. 회사 맞춤 FIT 답변

> [[FIT|FIT]] 기반으로 이 회사에 맞게 커스터마이징한 답변

### 1분 자기소개
> 구조: 이름/역할 → 도메인 경험 요약 → 핵심 성과 → 핵심 철학 → 왜 이 회사 → 마무리

(회사 도메인과 연결되는 경험 중심으로 재구성)

### 지원 동기
> 구조: 회사의 미션 이해 → 내 관련 경험 → 연결점 → 기여하고 싶은 것
> 참고: [[Zerobase-Career-Seminar#지원동기|지원동기 작성법]]

(회사가 풀고 있는 문제 + 내 경험의 접점)

### 이직 사유
> 구조: 현 회사에서 충분히 성장한 점 → 한계를 느낀 지점 → 다음 도전 목표 → 지원 회사와의 연결
> B2C 지원 시 / B2B 지원 시 분기 — [[FIT#이직 사유 (현재 회사에서 왜 옮기려 하는가)|FIT 참고]]

## 3. 예상 질문 (면접관 → 나)

### 이력서 기반 기술 질문
> 이력서/포트폴리오 성과를 깊게 파는 질문. "어떻게?", "왜?", "다른 선택지는?" 패턴
> 포트폴리오 STAR 형식의 구체적 수치와 의사결정 과정을 활용

#### 질문 제목
> 관련: `[[문서링크]]`

- 준비 포인트 (불릿 리스트로 작성)
- 구체적 수치/상황 포함
- 꼬리 질문 대비

### JD 기반 기술 질문
> 자격요건/우대사항에서 파생되는 질문
> 각 질문에 실무 경험 연결 + 꼬리 질문 대비 포함

#### 질문 제목
> 관련: `[[문서링크]]`

- 개념 설명
- 실무 경험 연결 (어떤 프로젝트에서 어떻게 적용했는지)
- 꼬리 질문 대비

### 서비스 맥락 질문
> "우리 서비스에서는 이런 상황인데, 어떻게 하겠는가?" 패턴
> 내 실무 경험과 연결하여 답변 + 꼬리 질문 대비

#### 질문 제목

- 설계 방향
- 실무 연결 (유사 경험)
- 꼬리 질문 대비

### 컬처핏 / 소프트스킬 질문
> [[FIT|FIT]], [[Self-Analysis|나찾기]] 기반 — 이미 준비된 답변 중 이 회사에 특히 중요한 것 체크

- [ ] 장단점
- [ ] 동료 갈등
- [ ] 기획자 충돌
- [ ] 긴급 이슈 판단
- [ ] 번아웃 관리
- [ ] 성장 목표
- [ ] 조직문화
- [ ] (회사별 추가 항목)

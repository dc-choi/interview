# 연봉 협상 가이드 — 원칙과 시장 조사

> 출처: Career Repotting Project 연봉협상 가이드북 (정구철), 이직의 정석 뉴스레터, 개발자 취업과 이직 한방에 해결하기

협상 테이블에 오르기 위한 전제 조건, 핵심 원칙, 시장 연봉 조사 방법, 사전 준비 체크리스트를 다룬다.

---

## 0. 협상이 성립하기 위한 조건

협상 테이블에 오르기 위해서는 **레버리지**가 필요하다. 레버리지 없이 앉으면 협상이 아니라 구걸이 된다.

### 필수 전제 조건
- **최소 두 군데 이상에서 오퍼를 받은 상태** — 한 곳만 붙은 상태에서는 협상력이 거의 없다
- **면접에서 좋은 피드백**을 받았거나 **근거가 충분**해야 함 (JD 매칭도, 기술면접 통과, 레퍼런스 체크 통과 등)
- **무턱대고 "희망액만큼 안 주면 안 간다"**는 태도는 협상 자체가 결렬될 가능성이 높음 — 회사는 "이 사람은 다른 곳을 택할 핑계를 찾고 있다"고 판단

### 추가 레버리지가 되는 것
- **현 직장의 복지/성과급/교육비/기술 커리어 관리**를 지원사와 비교해서 **남는 부분(손해 보는 부분)**이 있으면 그 자체가 좋은 협상 카드
  - 예: "현 직장에서는 연말 성과급이 기본급의 20%가 있었는데, 이번 오퍼에는 성과급 항목이 명시되지 않아서…"
- 단, **거짓말은 금지** — 국민연금/레퍼런스로 확인될 수 있고, 들키면 협상 자체가 끝남

### 협상 없이 수락해야 하는 상황
- 한 곳만 붙었고 그 회사가 **내 마지노선 이상**을 제시했을 때
- 회사 사정이 명확히 어렵고 **향후 재협의 약속**이 신뢰할 만할 때
- 처우보다 **배울 수 있는 환경/커리어 도약**이 훨씬 큰 경우

## 1. 핵심 원칙

### 연봉협상 = 등가교환

- 연봉을 낮춰서 갈 지원자도, 특별한 이유 없이 높은 연봉을 줄 회사도 없다
- **누구도 손해보는 결정을 하지 않는다** — 이것이 연봉협상이 등가교환인 이유
- 협상의 목적은 **관철이 아닌 타협점을 찾는 것**

### 명분이 전부다

- 명분이 없으면 때를 쓰는 것밖에 되지 않는다
- 합리적인 근거가 있으면 회사는 움직인다
- 반대로 명분 없이 "그냥 더 주세요"는 통하지 않는다
- **명분에 따라 숫자가 조정되거나 고정된다**

### 회사도 채용을 원한다

- 경력직 TO는 보통 1명, 채용에는 최소 2달 소요
- 서류 검토, 면접, 협상... 회사도 엄청난 시간과 에너지를 쏟은 것
- 면접 통과 = "이 사람과 같이 일하고 싶다"는 확신을 준 것
- **너무 위축되지 말 것** — 연봉협상은 일방적이지 않다

### 연봉협상은 철저한 논리 싸움

- 본인의 입장에서 합리적, 논리적으로 협상하면 회사에도 좋은 인상을 줌
- 반면 이전 면접 때 이미지를 상쇄시킬 정도로 집착하면, 회사 상황에 따라 채용이 진행되더라도 신규 조직에서 높은 연봉에 대한 기대치를 등에 업고 성과를 내기 어려움

## 2. 시장 연봉 조사 (협상의 선제 조건)

**연봉 협상은 면접장에서 시작하는 게 아니라 지원 전에 시작한다.** 시장 데이터를 모르고 들어가면 감으로 숫자를 부르게 되고, 알고 들어가면 근거로 숫자를 부를 수 있다.

### 반드시 확인할 4가지

| 항목 | 확인 방법 | 왜 필요한가 |
|------|----------|-----------|
| **회사 평균 연봉** | 잡플래닛 국민연금 데이터 (가장 신뢰도 높음), 리멤버 | 내 희망 연봉이 회사 수준 대비 어디쯤인지 파악 |
| **시장 연봉 (직무/연차)** | Glassdoor, 원티드, Paylab, 블라인드 | 동일 직무·연차의 시장 중위값 확인 |
| **동종 업계 초봉** | Blind IT업계 테크직군 초봉 시트, codingmonster | 초봉 x1.3~1.5로 경력 연봉 추정 가능 |
| **회사 규모/투자 단계** | 크런치베이스, 뉴스, 기업 정보 사이트 | 연봉 지불 여력 판단 (Seed vs Series B vs 대기업) |

### 사이트별 특성

| 사이트 | 데이터 출처 | 신뢰도 | 비고 |
|--------|----------|--------|------|
| **잡플래닛 (국민연금)** | 회사가 실제 신고한 금액 | 높음 | 전 직군/전 연차 평균이라 직무별 구분 없음 |
| **잡플래닛 (사용자 입력)** | 개인이 직접 입력 | 보통 | 표본 수가 적으면 왜곡 큼, 로그인/멤버십 필요 |
| **리멤버** | 출처 불명 | 보통 | 회사 개요와 함께 빠르게 확인 가능 |
| **Glassdoor** | 익명 제출 | 보통 | 외국계/글로벌 기업에 강함, 한국 스타트업은 표본 부족할 수 있음 |
| **원티드** | 채용 플랫폼 자체 데이터 | 보통 | 직군별 연차별 필터 가능, 백엔드 특화 데이터 확인 |
| **Paylab** | 글로벌 급여 조사 | 낮음 | 글로벌 데이터 혼합, 한국 시장과 괴리 가능 |
| **Blind 초봉 시트** | 현직자 크라우드소싱 | 높음 | 신입 초봉 기준, 경력은 직접 환산 필요 |
| **codingmonster** | 업계 정리 | 보통 | 회사 규모별 초봉 범위 참고용 |

### 경력 연봉 추정법 (초봉 데이터 활용)

동종 업계 초봉 데이터만 있을 때, 경력 연봉을 추정하는 방법:

| 경력 | 초봉 대비 배수 | 예시 (초봉 4,000만 기준) |
|------|-------------|----------------------|
| 3년차 | x1.1~1.2 | 4,400~4,800만 |
| 5년차 | x1.3~1.5 | 5,200~6,000만 |
| 7년차 | x1.5~1.8 | 6,000~7,200만 |

### 데이터 교차 검증

- **하나의 출처만 믿지 말 것** — 출처마다 편차가 크다
- 최소 3개 이상의 출처를 교차 검증하여 **레인지**를 잡는다
- 여러 출처가 가리키는 공통 구간이 실제 시장 가치에 가장 가깝다

## 3. 협상 전 준비할 것

### 사전 조사 체크리스트

| 항목 | 확인 여부 |
|------|----------|
| 회사 평균 연봉 (국민연금 기준) 확인 | |
| 시장 연봉 (직무/연차) 최소 3개 출처 교차 검증 | |
| 회사 규모/투자 단계 확인 | |
| 채용 절차에서 처우협의가 별도 단계인지 확인 | |
| 마지노선 설정 완료 | |
| 명분 3가지 이상 정리 완료 | |

### 마지노선 설정

- **협상 전에 반드시 마지노선을 정할 것** — 협상 중에 정하면 흔들림
- 마지노선은 절대 상대에게 공개하지 않음
- 마지노선 이하의 제안이 오면 거절할 각오가 되어야 함

### 명분 정리

명분으로 쓸 수 있는 것들:
- 시장 데이터 (국민연금, Glassdoor 등 객관적 수치)
- JD 자격요건과의 매칭도
- 기술면접 통과 사실 (역량 검증 완료)
- 포지션의 역할과 책임 범위
- 동종 업계 유사 포지션 처우 수준

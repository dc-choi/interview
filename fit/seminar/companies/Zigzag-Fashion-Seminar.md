# 패션도메인특강(지그재그)

## 패션산업구조

### 브랜드유형
| 유형 | 특징 | 예시 |
|------|------|------|
| 보세 | 공장에서 대량 생산, 디자인 자유도 낮음 | 동대문, 동묘 |
| SPA | 하나의 법인이 기획~판매 전과정 처리, 온라인 쇼핑몰 진출 추세 | 유니클로, 자라, 스파오, H&M |
| 디자이너브랜드 | 백화점 입점, 원자재 대비 높은 가격대 | 패션계/연예인 브랜드 |
| 명품 | 역사와 전통이 핵심 가치, 한국 인지도 상승 중 | |

### 생산~유통프로세스
기획/디자인 → 원단 수급 → 시제품 제작 → 재단/봉제(하청) → 납품 → 도매 → 소매 → 소비자

- 옷 하나가 나오기까지 오래 걸리지만, 패션 유행은 빠르게 변함
- 제작 완료 후 유통 시점에 이미 유행이 바뀌면 손해
- **개발자가 기여하는 영역**: 도매~소비자 구간 (유통/판매 플랫폼)

## 커머스개발플로우

표면적으로는 `상품전시 → 상품상세 → 주문 → 결제`로 보이지만, 실제로는:

```
상품등록 → 상품전시 → 장바구니 → 주문 → 결제 → 배송 → 정산
  + 회원, 쿠폰, 멤버십, 검색, 리뷰, 교환, 반품, 취소 ...
```

### 구간별핵심기술
| 구간 | 핵심주제 | 면접포인트 |
|------|---------|-----------|
| 앞단(전시/검색) | 트래픽이 집중되는 영역 | 캐시, Elasticsearch |
| 중단(주문/결제) | 데이터 정합성이 중요 | 동시성, 큐, 무결성 (백엔드 단골 질문) |
| 뒷단(정산/배송) | 외부 시스템 연동 | PG, DB, Transaction |

## 기술스택

| 분류 | 스택 |

| 언어/프레임워크 | Java, Kotlin, TypeScript / Spring, Node.js |

- 풀스택은 가볍게 공부하게 되는 위험이 있음

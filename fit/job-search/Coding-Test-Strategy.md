# 코딩테스트 준비 전략

## 기본 원칙

- 지원하는 회사의 언어로 시험을 치루는 것이 유리 (가이드라인 확인)
- 주요 평가: 구현 능력, 문제 해결 능력, 기본적인 CS/알고리즘 평가
- 기술 면접을 위한 첫 관문 — 채용의 허들로 제일 많은 지원자들이 탈락하는 단계
- 1차 필터링을 하겠다는 의도가 강하게 작용하는 관문

## 준비 방법

- 짧게 풀 수 있는 퀴즈 문제부터 점차 복잡한 문제까지 단계별 접근
- 빈출이 높은 것을 정복하되, 각 단계별 하-중-상까지 순차적으로 해결
- 문제를 이해하는 것이 첫 관문
- 시간 복잡도와 테스트 패스까지 고려해야 함
- 반드시 회고를 거쳐 더 좋은 방향은 없었는지 리뷰

### 무작정 공부하기 어렵다면
- 기본적인 프로그래밍 언어를 숙달한 후 아래 사항 집중
- CS에 대한 공고한 이해, 자료구조, 알고리즘
- 여러 유형의 퀴즈 풀이, 동영상, 관련 서적 학습

### 유의할 점
- 3개월 정도는 집중적으로 공부해야 함
- 주객이 전도되지 않게 너무 많은 시간 투자는 부담
- 자료구조의 이해는 필수
- 간단한 퀴즈성 문제로 머리를 말랑말랑하게 한 뒤 본격적인 준비
- CS 지식을 공고히 하고 언어 구사 능력을 키우는 것에 초점
- 자료를 정리하고 타인의 인사이트를 얻을 수 있는 **팀 스터디 형태** 추천

## 단계별 접근 전략

### 1단계: 자신감을 심는 단계
- 쉬운 문제 위주
- 간단한 알고리즘 퀴즈나 계산
- 프로젝트 오일러

### 2단계: 핵심 자료구조와 알고리즘 숙지
- 정렬, 자료구조, 유명 알고리즘 단계별 정복
- 문제의 유형이나 풀이 패턴 학습

### 3단계: 본격적인 훈련
- 서적이나 동영상 참고
- 중고급 난이도 풀이가 중요
- **중상급 문제에 도달해야 합격권**

## 코딩테스트 복기

코딩테스트를 패스했다면 면접 자리에서 리뷰하는 시간을 가질 수 있다.

- 왜 이 방식으로 풀었는가? → 면접관의 질문에 대비
- 문제의 해결과정, 복잡도 설명 → 본인의 풀이 복기
- 최선의 자료구조는 무엇인가? 다른 풀이 방식은 없는가? → 더 나은 해법 제시

## 분류별 필수 지식

### 정렬 알고리즘
병합정렬, 버블정렬, 퀵정렬, 힙정렬, 삽입정렬

### 자료구조
이진트리, B트리, 스택, 큐, 우선순위큐, 연결리스트, 연결해쉬맵, 해쉬테이블

### 검색 알고리즘
선형검색, 이진검색

### 그래프 탐색 알고리즘
DFS, BFS

### 문제해결 전략
백트래킹, 완전탐색, 투포인터, 분할정복, DP, 그리디

### 캐시 교체 알고리즘
LRU, LFU

### 최단경로 알고리즘
다익스트라, 벨만-포드

## 핵심 습관

- 문제를 이해하는 단계가 가장 어렵다
- 매일매일 꾸준함이 중요
- 단계별로 리뷰와 회고
- 둘 이상 모여 공유
- 알고리즘 전문 사이트를 통한 훈련의 반복과 스터디
- 복잡도를 고려하고 테스트 검증 필수
- 인터넷에 공유되는 유명한 문제들도 도전

## 해외 취업 목표

- 기간을 정해 계획을 가지고 단계별 실력 향상
- 기출 문제를 빠르게 풀어낼 수 있어야 함
- 반복 훈련과 손 코딩
- 완전한 코드가 아니라도 의사 코드를 써서 설명하는 훈련

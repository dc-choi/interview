# IT 기업 유형별 특성

개발자 진로를 고민할 때 "어느 회사로 가야 하는가"는 "어느 **유형**의 회사인가"를 먼저 생각해야 한다. 같은 "개발자"라도 **소속 회사 유형**이 커리어 방향·학습 속도·연봉 궤적을 크게 좌우한다. 이름보다 **구조**로 본다. [[Company-Selection-Criteria|회사 선택 4축]]이 회사 내부 평가 기준이라면, 이 문서는 **회사 유형 자체**의 분류.

## 11가지 유형

### 1. 대기업 '갑' 회사 (발주처·SI 원청)

- 영어·성적·학벌 기준 **높음**
- 신입에게 **코딩 능력 덜 중시** — 문서·기획·예산 관리 비중 큼
- 예산 오너십·주도권 있음
- **장점**: 높은 연봉, 주도적 의사결정
- **단점**: 개발보다 제안·기획·회의 중심. 리스크 관리 부담 큼
- 적합: 비즈니스·PM 지향이 강한 개발자

### 2. 대기업 '을' 회사 (SI·SM)

- 갑 지시 따라 다양한 산업·프로젝트 경험 가능
- **폭넓은 교육 인프라** — 신입 훈련 체계적
- **장점**: 다양한 경험·교육, 브랜드 네임
- **단점**: 을의 포지션, 낮은 연봉, 야근 多, 기술보다 납기 우선
- 적합: 초기 경력에 다양한 산업·기술 스택을 빠르게 경험하고 싶은 경우

### 3. 아웃소싱 '병' 회사

- 학벌·영어 허들 **낮음**
- 실무 기술은 필요
- **장점**: 진입장벽 낮음, 실무 경험 축적, 프리랜서 전환 시 고액 가능
- **단점**: 낮은 급여, 악조건, 과다 야근, 경력 관리 어려움
- 적합: 학력·네트워크가 약하지만 실력으로 증명하려는 경우의 **초기 발판**

### 4. 대기업 서비스 회사 (네이버·카카오 등)

- "**스펙이 아닌 실력**"이 기준
- 수평적 조직, 개발자 처우 좋음, 학습 문화
- **장점**: 고연봉, 좋은 동료, 기술 투자, 수평 문화
- **단점**: 기술 경쟁 치열, 고객 민원 대응, 제품 라이프사이클 짧음
- 적합: 자기주도적이고 성장 지향적인 개발자

### 5. 스타트업

- 학벌·성적 무관, **실력·포트폴리오**만 중요
- 스톡·RSU로 업사이드 확보 가능
- **장점**: 지분·경영 경험, 빠른 성장, A-Z 개발 경험
- **단점**: IPO·인수 확률 낮음, 높은 업무 강도, 실패 리스크
- 적합: 리스크 감수 + 큰 학습 곡선을 원하는 경우
- [[Startup-Is-Not-Kindergarten|스타트업의 직설 문화]] 참고

### 6. 해외 기업의 한국 지사

- 영어 + 높은 기술력 필수
- LinkedIn·헤드헌터 채널
- **장점**: 고연봉(TC 기준), 세계 수준 엔지니어, 해외 경험 발판
- **단점**: 유대감 약함, 초보자 부적합, 본사 정책 의존
- 적합: 영어 가능 + 중급 이상 경력
- [[Global-IT-Interview|글로벌 IT 인터뷰 준비]]

### 7. 원격근무 해외 기업

- 재택근무 기반, 신흥 채용 유형
- 시차·커뮤니케이션 적응 필요
- **장점**: 거주지 자유, 글로벌 스탠다드 경험
- **단점**: 오프라인 네트워킹 부족, 온보딩·관계 형성 어려움
- 적합: 비동기 커뮤니케이션 능숙 + 자기 주도성 강한 경우

### 8. 게임 회사

- 자유로운 분위기, 고급 기술 활용
- 크런치 문화는 조직마다 편차 큼
- **장점**: 창의적 프로젝트, 고난도 기술(그래픽·실시간·AI)
- **단점**: 프로젝트 실패 리스크, 장시간 근무 가능성
- 적합: 게임 개발에 강한 동기 있는 경우

### 9. 중견 SW 전문 회사 (자체 솔루션)

- 자체 솔루션 개발·패키징·딜리버리
- **장점**: 제품 전체 주기 경험, 도메인 전문성
- **단점**: 낮은 연봉, 제한된 기술 스택, 영업·고객 대응 비중
- 적합: 특정 도메인(금융·보안·산업) 전문가 지향

### 10. 공기업·공공기관

- **정년 보장**·안정성 중심
- 개발 속도·기술 최신성은 제한적
- **장점**: 워라밸, 고용 안정
- **단점**: 기술 정체, 관료적 문화, 낮은 학습 속도
- 적합: 안정·워라밸 우선, 기술 최전선에 대한 욕심 없음

### 11. 금융기관 개발팀

- 초봉 매우 높음
- 개발 자율도·의미는 제한적
- **장점**: 고연봉, 안정성, 금융 도메인 전문성
- **단점**: 레거시 기술, 보수적 문화, 외주 의존도
- 적합: 고연봉 + 안정 우선, 도메인 특화 관심

## 선택 축 요약

| 축 | 우선시 | 추천 유형 |
|---|---|---|
| 기술 성장 | 속도·학습 | 대기업 서비스 / 스타트업 |
| 연봉 | 현재 수입 | 해외 지사 / 금융 / 대기업 서비스 |
| 안정성 | 장기 고용 | 공기업 / 대기업 갑·금융 |
| 다양한 경험 | 산업 폭 | 대기업 을 / 아웃소싱 |
| 자기주도성 | 실력만 | 스타트업 / 서비스 / 해외 |
| 워라밸 | 시간 | 공기업 / 해외 원격 |

**하나의 기준으로만** 고르면 2~3년 후 후회. 2~3개 축 복합 고려가 현실적.

## 경력 경로 패턴

- **SI/SM(경험 축적) → 서비스(기술 고도화) → 스타트업 CTO 또는 해외**
- **공기업(안정) → 이직 시장 진입 어려움** — 이동 의도 있으면 조기에
- **중견 SW(도메인 전문) → 관련 대기업·스타트업 도메인 리더**
- **금융 개발 → 핀테크 스타트업·외국계 핀테크**
- **스타트업 → 서비스 대기업으로 전향**은 비교적 흔함
- **대기업 서비스 → 해외 원격/지사**가 최근 증가

## 유형 간 이직 난이도

- 서비스·스타트업·해외는 **실력 증명**이 관건
- 공기업·금융은 **외부 이직이 어려움** — 내부 체류가 길어지면 갇힘
- SI·아웃소싱에서 서비스·해외로 **점프업**은 포트폴리오·개인 학습의 누적 필요
- 같은 유형 내 이직이 **가장 수월**

## 흔한 오해

- "대기업 = 무조건 좋다" — 갑/을/서비스 구분 없이 묶으면 판단 실패
- "스타트업은 위험하다" — 시리즈·재무·리더십 확인하면 선택 가능. **시리즈 B 이상 + 런웨이 18개월**이 최소 안전선
- "공기업은 평생 안정" — 기술 정체로 이직 시장에서 빠르게 가치 하락
- "해외 지사 = 해외 본사" — 조건·문화·승진 경로가 매우 다름
- "게임 회사는 다 크런치" — 조직별 편차 큼. 사전 검증 필요

## 면접 체크포인트 (본인 커리어 계획용)

- 본인의 **우선순위 축 3개**를 명시할 수 있는가
- 현 직장 유형의 **장단점**을 객관적으로 말할 수 있는가
- 다음 목표 유형의 **진입 조건**을 파악했는가
- 유형 간 이직의 **난이도 지도**를 감각으로 가지고 있는가
- 회사 브랜드가 아닌 **유형 특성**으로 평가하는가

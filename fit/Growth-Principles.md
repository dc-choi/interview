# 성장의7가지원칙

## 1.반복(Sequence)
- 한번의 시도로 변화를 만드는 것은 어렵다
- 성장이든 설득이든 여러 번 반복하며 전략을 조정해야 한다
- 예: 팀장을 설득하려면 3개월간 여러 번 시도하며 접근법을 설계하는 것

## 2.맥락속실전(Situated&Authentic)
- 구체적인 상황 속에서 직접 경험해야 진짜 배움이 된다
- 이론만으로는 실전 능력이 생기지 않음 (예: 협상의 3요소를 알아도 협상 경험이 없으면 소용없음)
- 코드를 짤 때도 "누가 쓸 것인가?"라는 맥락 없이 짜면 성장이 없음
- 목적 없고 쓸모 없는 연습은 쓸모가 없다

## 3.회고(Reflection)
- 경험 후 되돌아보는 과정이 반드시 필요
- 타인의 피드백이 자기 인식의 사각지대를 보완해줌
- "내가 그랬었어?" 라는 깨달음은 혼자서는 얻기 어려움

## 4.새로운실험(Experiment)
- 회고에서 끝나면 안 됨. 반드시 새로운 시도로 이어져야 한다
- 깨달음 자체는 아무 의미가 없음. 행동 → 피드백 → 조정의 루프가 핵심
- "오늘 당장 뭘 할 건가?"로 구체화하지 않으면 깨달음만 반복하게 됨

## 5.자기효능감(Self-Efficacy)
- "마음만 먹으면 할 수 있겠다"는 감각이 가장 중요
- 초기에 이 감각을 빠르게 올려야 함
- 처음부터 너무 큰 목표를 잡아 실패가 반복되면 포기하게 됨
- 작은 성공을 빠르게 경험하는 설계가 필요

## 6.습관화(Habituation)
- 배운 것을 체화하지 않으면 원래 관성으로 돌아감
- 의식적 노력 → 반복 → 무의식적 습관의 단계를 거쳐야 진짜 변화

## 7.소셜화(Socialization)
- 혼자 아는 것으로는 시스템이 바뀌지 않음
- 새로운 개념을 배웠다면 팀원과 상사를 설득해서 함께 적용해야 함
- 아이디어가 좋다는 것만으로는 설득이 안 됨. 반복적 설득과 전략이 필요 (1번과 연결)

## 면접에서활용하기
- **"어떻게 성장하시나요?"** → 이 7단계 프레임으로 답변 구성 가능
- **"팀에 새로운 기술을 도입한 경험"** → 반복 설득 + 소셜화 사례로 활용
- **"실패 경험과 극복"** → 회고 → 실험 → 자기효능감 회복 스토리로 구성

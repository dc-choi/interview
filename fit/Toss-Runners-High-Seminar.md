# 토스러너스하이가이드

## 채용철학

### 토스가확인하고싶은것
1. 앞으로도 잘할 거라고 확신할 수 있는가? → 과거 이력 + 새로운 문제 풀이
2. 스스로 끊임없이 다양한 방향으로 성장할 것인가? → 과거 이력
3. 위 두 가지를 충족하면 **같이 일하고 싶다**고 판단

### 채용프로세스
- 넥스트 개발자 챌린지: 1차(알고리즘, 서술형) → 2차(면접)
- 링크드인 기반 커리어 상담 → 러너스하이 실습(1개월) → 직무 인터뷰
- 온보딩 후 바로 결과를 만들어낼 수 있는 사람을 찾음

### 채용여정
서류접수 → 직무 인터뷰 → 문화 적합성 인터뷰 → 레퍼런스 체크 → 처우협의 → 최종합격

## 내부평가지표

> 상세 내용은 [[Toss-Evaluation-Criteria|토스 내부 평가 기준]] 참고

### 기술코드/설계
- 라이브러리 내부 구현을 분석하여 문제 해결
- 모범사례와 비교하여 더 나은 설계 방향 제시
- 더 높은 시스템 성능으로 요구사항 충족

### 기술운영
- 장애를 빠르게 감지하고 신속히 대응
- 장애물이나 예기치 못한 상황에서 충분히 회복
- 동일 장애 재발 방지를 위한 Root Cause 분석
- 로깅과 메트릭으로 문제 파악 용이한 상태
- 가설을 데이터로 검증
- 배포 임팩트를 측정하고 보여주기
- 성능 최적화/운영 자동화 지속적 노력

### 제품
- 비즈니스가 더 성장하도록 만들고 있는가
- 담당 제품의 규모와 영향력
- 내가 있을 때와 없을 때의 차이를 팀이 느끼는가

### 커뮤니케이션
- 데이터를 기반으로 의견 조율 및 협의
- 기록을 통해 문맥을 명확히 전달
- 피드백/설명 시간 = ROI로 생각

### 문화
- 정확한 피드백 능력 (아는 만큼 보인다)
- 지속적으로 결과를 낼 수 있는 방법을 제시

## 커리어조언

### 결과로말한다
- 비즈니스의 성공은 다양한 기회를 가져옴
- 원래 되는 것보다 안 되는 것이 많음. 찬성보다 반대가 많음
- 하지만 결과로 말한다는 생각으로 일함
- **살아남는 자가 강한 자**

### 주니어에게기대하는것
- 주어진 것을 잘 만들어내는 역량
- 단순 기능 구현에서도 최적화 고민 필요

### 경력기술서작성팁
- 간결한 문장
- **양보다 임팩트**

- 언어는 딱히 안 탐

# 금융도메인특강(핀테크)

## 금융도메인구조

| 도메인 | 업무 | 예시 |
|--------|------|------|
| 여신 | 대출 관련 | 신용대출, 주택담보대출 |
| 수신 | 예금/통장 관련 | 모임통장, 입출금통장 |
| 이체 | 송금 관련 | 간편이체, 계좌이체 |

### 시스템계층(참고)
- **채널계**: 고객 접점(앱, 웹)
- **계정계**: 핵심 금융 거래 처리
- **정보계**: 데이터 분석/리포팅

## 핀테크의특수성

### 트래픽처리방식
- 핀테크의 트래픽은 타 도메인과 다름
- 대규모 트래픽을 한번에 받지 않고 **대기표 발급** 방식으로 처리
- 이유: 연동된 외부 금융기관이 대규모 트래픽을 감당할 수 없기 때문

### 망분리
- 금융권은 법적으로 망분리 환경 구축이 의무
- CI/CD도 eCams 같은 전용 도구를 사용 (Jenkins를 쓰는 곳도 있음)

### 전망
- 이윤을 발생시키기 가장 좋은 대상은 돈
- 핀테크는 전망이 좋은 도메인

## 핀테크주니어에게요구하는역량

| 역량 | 이유 |
|------|------|
| 트랜잭션이해 | 돈과 직결되므로 트랜잭션 처리가 가장 중요 |
| 꼼꼼함 | 요청의 일관성+정합성을 맞추는 작업이 많음. 예외처리가 핵심 |
| 끈기 | 현업 프로젝트는 거대하고, 그 안에서 문제를 찾아야 함 |
| 기술스택이해도 | 깊이 있는 이해를 보여줄 수 있어야 함 |
| 질문하는용기 | 모르는 것을 물어볼 수 있어야 함 |

## 오해와진실

| 오해 | 진실 |
|------|------|
| 핀테크 관련 프로젝트 없으면 지원 못한다 | 빠르게 배우는 능력, 기술을 깊이 파고드는 자세면 충분 |
| 핀테크 트래픽은 타 도메인과 비슷하다 | 외부 금융기관 연동 제약으로 대기표 방식 등 독자적 처리 |

## 기술스택

- Spring MVC, Boot, Security, Batch
- JPA, MySQL
- RabbitMQ, Kafka
- Jenkins, SonarQube
- 채용 공고 우대사항의 기술을 갖추면 눈에 띔
- 3년차까지 지원 가능하니 겁먹지 말 것

## 업무프로세스&협업

### 개발프로세스
기획 → 요구분석 → 설계/구현 → 테스트 → 유지보수

### 실무팁
- **커뮤니케이션 = 문서화.** 문서화하면 더 나은 소통이 됨
- Jira로 Task 분배, Gitflow 전략 적용
- 코드 리뷰에서 자유롭게 질문할 것 (개념 질문, 구현 방식 논의 등)
- 페어 프로그래밍이 어려우면, 개발 전에 서로 생각을 나누는 것만으로도 노하우를 얻을 수 있음
- 신입일 때 회사가 어떻게 돌아가는지 모르면 무조건 물어볼 것
- 블로그를 꾸준히 할 것

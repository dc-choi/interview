# 애자일 회고 기법

> 출처: 개발자 취업과 이직 한방에 해결하기

회고는 성장의 핵심이다. ([[Growth-Principles#3.회고(Reflection)|성장의 7가지 원칙 — 회고]] 참조)
**"오늘 당장 뭘 할 건가?"로 구체화되지 않으면 회고는 깨달음만 반복하는 자기만족이 된다.**

상황과 목적에 따라 다음 기법 중 하나를 선택해 사용한다.

---

## CSS (Continue / Stop / Start)

**목적**: 팀의 개선에 중점을 둔 회고. 가장 단순하고 실행에 즉각 옮기기 쉬운 프레임.

| 항목 | 질문 |
|------|------|
| **Continue** | 지속해야 할 것은 무엇인가? |
| **Stop** | 그만해야 할 것은 무엇인가? |
| **Start** | 새로 시작해야 할 것은 무엇인가? |

**언제 쓰나**: 스프린트 종료, 릴리즈 직후 팀 프로세스 점검. 액션 아이템을 즉시 도출하고 싶을 때.

---

## 4L (Liked / Learned / Lacked / Longed For)

**목적**: 조직 안에서 잠시 재점검을 위한 회고. **개인의 의견 반영에 중점**을 두어 감정/경험/욕구를 골고루 수집한다.

| 항목 | 질문 |
|------|------|
| **Liked** | 좋았던 것은 무엇인가? |
| **Learned** | 배운 것은 무엇인가? |
| **Lacked** | 부족했던 것은 무엇인가? |
| **Longed For** | 원했던 것은 무엇인가? (마지막에 "해야 할 것"을 붙여 확장 가능) |

**언제 쓰나**: 프로젝트 중간 점검. 구성원 간 감정 상태와 니즈를 파악하고 싶을 때. CSS보다 더 넓은 시야가 필요할 때.

---

## KPT (Keep / Problem / Try)

**목적**: 개인과 팀 모두에 적용 가능한 균형 잡힌 회고 프레임.

| 항목 | 질문 |
|------|------|
| **Keep** | 잘하고 있어서 유지하면 좋은 점은? |
| **Problem** | 개선해야 하고 해결해야 할 일은? |
| **Try** | 앞으로 시도해야 할 일은? |

**CSS와 차이**: KPT는 "Problem"이라는 **문제 정의 단계**가 명시되어 있다. CSS의 "Stop"이 행동 중심이라면 KPT의 "Problem"은 원인 분석 중심이다.

**언제 쓰나**: 주간 회고, 개인 회고. 문제를 명확히 정의한 뒤 실험적 시도로 연결하고 싶을 때.

---

## TIL (Today I Learned)

**목적**: "오늘 무언가를 배웠는가?"라는 질문을 축으로 하는 일일 학습 회고.

| 항목 | 질문 |
|------|------|
| **Event** | 오늘 있었던 일 |
| **Emotion** | 그 때 느꼈던 감정/느낌 |
| **Learning** | 이로 인해 배운 점 |
| **Action** | 해야 할 일 / 하지 말아야 할 일 |

**언제 쓰나**: 매일 가볍게. GitHub TIL 레포나 블로그에 쌓아두면 면접에서 **"꾸준히 학습하는 사람"의 증거**가 된다.

---

## WIW (What I Want)

**목적**: 세 가지 질문에 초점을 맞추어 **개인의 목표를 설정하고, 구체적 행동 계획을 수립**하는 기법. 목표 달성을 위해 적극적으로 움직일 수 있도록 방향을 설정하는 용도.

| 질문 | 내용 |
|------|------|
| **내가 뭘 했는가?** | 내 경험, 내가 수행한 프로젝트 등 |
| **배운 것은 무엇인가?** | 경험으로부터 얻은 지식, 어려움 등을 고려하여 정리 |
| **내가 해야 할 것은 무엇인가?** | 미래 목표와 계획 수립 |

**언제 쓰나**: 분기/반기/연간 회고. 커리어 방향 재설정. 1년 회고 작성. **면접 직전 자기 객관화**에도 효과적.

| 상황 | 추천 기법 |
|------|---------|
| 매일의 학습 기록 | TIL |
| 주간 개인/팀 회고 | KPT |
| 스프린트 종료 후 액션 도출 | CSS |
| 프로젝트 중간 점검, 감정까지 포함 | 4L |
| 분기·연간 커리어 재설정 | WIW |

## 회고의 공통 원칙

1. **구체적 행동으로 끝낼 것** — 깨달음만 반복하면 성장 없음. 반드시 "오늘/이번 주에 뭘 할지" 결정
2. **정직할 것** — 부풀리기 금지. 내가 한 일과 하지 않은 일을 구분
3. **기록으로 남길 것** — 말로만 하면 다음 회고 때 같은 이야기가 반복됨
4. **타인의 피드백을 포함할 것** — 자기 인식의 사각지대를 보완 (타인 → "내가 그랬었어?")
5. **행동 → 피드백 → 조정의 루프** — 회고에서 끝나면 안 되고 새로운 실험으로 이어져야 함

## 면접에서 활용하기

**Q. 어떻게 성장하고 계신가요?**
- 어떤 회고 프레임을 쓰는지 (예: 주간 KPT, 분기 WIW)
- 실제로 최근 회고에서 도출한 액션과 그 결과
- [[Growth-Principles|성장의 7가지 원칙]]과 연결해서 답변 구성

**Q. 실패 경험과 극복 과정은?**
- 회고(4L의 Lacked/Longed For) → 실험(Try) → 자기효능감 회복의 스토리로 구성

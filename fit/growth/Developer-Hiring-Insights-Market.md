# 개발자 채용 인사이트 — 시장·CS·역량

## 기업이원하는개발자

### 채용시장의현실
- 신입에게도 경력직 수준의 스펙과 우대사항을 요구하는 추세
- 채용 혹한기로 공급은 증가하고 수요는 감소하지만, 채용 눈높이는 낮아지지 않음
- 결과적으로 소수의 경력직 채용 위주로 시장이 움직임

### 신입채용의우대사항으로올라온것들
- CS지식(운영체제, 네트워크, 자료구조)
- 협업도구(Slack, Jira, Git 브랜치 전략)
- 운영경험(배포, 모니터링, 장애 대응)
- 이런 역량은 현장 경험이 필요하지만, 학교에서는 얻기 어려움

### 차별화포인트
- 똑같은 포트폴리오로는 안 됨
- 기획 → 설계 → 구현 → 배포 → 운영까지 전과정을 경험한 프로젝트가 필요
- 많은 트래픽, 보안, 안정성에 대한 고민이 담긴 경험이 중요

## CS기초의중요성

- 기초 지식이 없으면 금방 바닥이 드러남
- 장기적인 연봉에 영향을 줌
- 소통을 위한 최소한의 학습이 반드시 필요
- 모든 CS를 다 알 수는 없지만:
  - **몇몇 핵심 주제**는 심도 있게 이해하고, 코드와 이론을 매핑할 수 있어야 함
  - **나머지 주제**는 적어도 개론 수준의 이해가 필요

## 문제해결역량

- 성장 가능성 = 문제를 잘 정의하고 해결할 수 있는 능력
- 문제 해결의 과정:
  1. 팀 안에서 문제의 관점을 논의
  2. 해결을 위한 기술과 협업 방법을 논의
  3. 필요한 지식을 나열하고, 부족한 부분은 학습
  4. 점진적으로 디버깅, 코드 리뷰, 테스트, 릴리즈, 배포
  5. 회고를 통해 지식 습득과 협업을 점검
- 해결 방안이 현장에서 잘 동작하는지 확인하는 **운영 경험**이 핵심

## 협업에대한인식

### 기존방식의문제
- 철저한 분업으로 다른 팀원이 문제를 어떻게 해결했는지 모름
- 다른 영역에 대한 기술 이해가 없음
- 그걸 알 수 있는 협업 도구나 활동이 부재

### 좋은협업이란
- 지식을 공유하고 다른 사람에게 질문하는 것
- 개선할 점을 받아들이고 실제로 개선하는 것
- 모든 구성원이 개발 전 과정을 함께 경험하는 것
- 기술적/방법적/태도적 측면에서 회고를 하는 것

## 동기유지와성장

- 개발의 재미는 결과를 확인할 때, 더 높은 기술적 목표에 대한 열정이 생길 때 느낌
- 동기를 지속적으로 유지하려면 **성취의 단계를 더 잘게 나누어야** 함
- 깊은 고민을 먼저 해본 뒤 질문해야 의미 있는 답을 얻을 수 있음

## 개발자역량

결과적으로 일이 되게 하는 능력. 일이 되게 만드는 것을 고민하는 것이 전문가적 역량.
예측 가능한 퍼포먼스를 보여주는 것이 프로의 역량. 설득하고 이해시키는 커뮤니케이션 역량.
이것이 모여 성과로 연결된다.

> 경험과 의지가 일정 수준의 숙달을 만들고, 그 숙달이 전문적인 기술을 만들어냄.

### 기술적역량

**1. 프로그래밍 언어와 인프라의 공고한 이해**
- CS, 알고리즘과 코딩 테스트
- 개발 스택 (모바일, 프론트, 백)
- 데이터베이스와 SQL
- 기본적인 서버 인프라
- 개발 스택에 필수적인 프레임워크의 이해

**2. 소프트웨어 엔지니어링**
- 시스템 디자인
- 코드 품질과 테스트
- 디자인 패턴, OOP
- 데이터베이스 설계와 NoSQL, Redis
- ER 다이어그램, 엔티티 디자인
- Lock, 캐싱 전략
- 프레임워크와 에코시스템

### 업무역량
- 업무 분석 능력, 데이터의 흐름 파악, 호출 구조 파악
- 비즈니스 요구사항 분석 (정책 관련 코드 분석, 요구사항 정의)
- 도메인의 이해
- 큰 그림을 그리는 능력과 그 안에서 디테일을 챙기는 꼼꼼함
- 요구사항을 프로그래밍으로 구현하는 능력
- 주어진 미션을 되게 만드는 노력

## 문제해결능력(심화)

종합적인 논리, 창의적 사고, 경험치가 필요한 영역.
해결까지 필요한 전문적 기술 지식 필요. 문서화, 전파, 공유, 재발 방지.

### 취업준비자가 갖춰야 할 것
- CS 기본 지식과 자료구조, 알고리즘을 바탕으로
- 언어 스펙 공부
- RDBMS, ANSI SQL
- 라이브러리 뜯어보기, 프레임워크 뜯어보기
- 프레임워크를 활용한 프로젝트
- 무엇인가 만드는 것을 좋아하는 사람이라면 반드시 결과물로 어필 (덕업일치)

### 이직준비자가 갖춰야 할 것
- 연차에 비례하는 경험과 실력
- 시스템 디자인, 엔티티 설계
- OOP와 대표적인 디자인 패턴
- RDBMS와 NoSQL, Redis
- 서비스를 운영하기 위한 네트워크와 인프라
- 클라우드 서비스의 이해
- 애플리케이션의 장애 트래킹과 문제 해결 방법
- 성능 테스트, 성능 개선 (애플리케이션 튜닝, SQL 튜닝)
- 비동기 아키텍처에 대한 이해
- RabbitMQ, Kafka
- 컨테이너 플랫폼 환경의 이해 (Docker, K8s)
- MSA

# 2026 하반기 수학 기초 로드맵

수학 기초가 거의 없다는 현재 출발점을 그대로 인정하고 기초 연산부터 시작한다. 2026년 하반기의 개인 학습 주축은 수학 하나다. 검색과 추천은 현업 문제를 해결하는 데 필요한 만큼 즉시 학습하되, 체계적인 검색과 추천 실습은 수학 gate 뒤 2027년 별도 로드맵에서 시작한다. 코딩 테스트와 DevOps 실행 계획은 이 문서에서 분리한다.

## 결정 요약

| 시기 | 주축 | 병행 범위 | 하지 않는 것 |
|---|---|---|---|
| 2026 하반기 | 기초 연산, 대수, 함수와 그래프 | 업무에 필요한 검색/추천 개념과 작은 계산 | 검색/추천 로드맵 병행, DevOps 커리큘럼, 자격증, 코딩 테스트 |
| 2027 수학 2단계 gate 이후 | 검색 엔진 baseline과 품질 평가, 이후 추천 baseline | 부족한 수학을 필요한 지점에서 보충 | 검색과 추천 동시 착수, 고급 모델부터 시작하기 |

검색과 추천의 전환 조건은 [[2027-Search-Recommendation-Roadmap|2027 검색 엔진 우선, 추천 시스템 전환 로드맵]]에서 관리한다. DevOps는 [[2027-DevOps-Practical-Roadmap|2027 DevOps 실전 로드맵]]에서 별도로 관리하며 검색/추천의 완료 여부를 시작 조건으로 삼지 않는다.

## 전체 흐름

```mermaid
flowchart LR
  A[수와 기초 연산] --> B[대수, 함수와 그래프]
  B --> C[벡터, 행렬, 확률과 통계]
  C --> D[미분, 회귀와 순위 평가 수학]
  D --> E[2027 검색 엔진 baseline]
  E --> F[2027 추천 baseline]
```

수학을 모두 끝낸 뒤에만 업무를 할 수 있다는 뜻은 아니다. 업무에서는 기존 [[OpenSearch|OpenSearch 학습 지도]]와 [[Recommendation-System-Architecture|추천 시스템 지식 지도]]를 참고해 필요한 개념을 그때그때 연결하되, 개인 학습 진도는 기초를 건너뛰지 않는다.

## 학습 원칙

- 시작점을 증명하기 위한 진단시험이나 별도 퀴즈를 요구하지 않는다. 모르는 단원은 처음부터 배우고, 익숙한 단원은 예제를 풀면서 자연스럽게 통과한다.
- 고정 종료일보다 gate를 우선한다. 2026년 안에 모든 수학을 끝내지 못해도 실패가 아니다.
- 초반에는 손계산과 풀이 설명이 중심이다. Python 검산은 식을 읽을 수 있게 된 뒤 붙인다.
- 한 번에 한 개념만 다룬다. 막힌 상태에서 진도를 밀기보다 필요한 이전 개념으로 돌아간다.
- 교재 완독, 문서 수와 공부 시간 자체를 성과로 계산하지 않는다.

## 0단계: 수와 기초 연산, 기간 미정

### 학습 범위

1. 자연수, 정수, 음수와 수직선
2. 사칙연산 순서, 괄호와 부호
3. 분수와 소수의 변환 및 사칙연산
4. 비율, 비례식과 백분율
5. 거듭제곱, 제곱근과 단위 변환
6. 식에 숫자를 대입하고 결과를 검산하는 방법

### 통과 gate

- [ ] 분수, 소수와 백분율을 서로 바꾸고 같은 크기인지 설명한다.
- [ ] 필요한 약분과 통분을 한 뒤 분수와 소수의 사칙연산을 풀이한다.
- [ ] 수직선에서 정수의 순서와 부호를 설명하고 음수와 괄호가 포함된 사칙연산을 풀이 순서와 함께 적는다.
- [ ] 비례식으로 CTR, 전환율과 증가율의 작은 예제를 계산한다.
- [ ] 1 이상의 작은 정수의 거듭제곱과 0 이상 완전제곱수의 제곱근을 계산하고 두 연산의 관계를 설명한다.
- [ ] 공식에 값을 대입하고 길이, 시간과 데이터 크기 단위를 변환한 뒤 예상 범위로 결과를 검산한다.

## 1단계: 대수, 함수와 그래프, 기간 미정

### 학습 범위

1. 변수, 상수, 항과 계수
2. 일차방정식, 부등식과 식 변형
3. 좌표, 기울기와 일차함수 그래프
4. 함수의 입력, 출력과 합성의 직관
5. 지수 법칙, 제곱근과 로그의 의미
6. 합 기호, 평균과 간단한 수열 표기

### 통과 gate

- [ ] 식에서 변수, 상수, 항과 계수를 구분하고 일차방정식과 간단한 부등식을 풀며 부등호가 바뀌는 조건을 설명한다.
- [ ] 좌표, 기울기, 표, 식과 그래프가 같은 일차함수 관계를 표현하는 예를 만든다.
- [ ] 함수의 입력과 출력을 표시하고 두 간단한 함수의 합성 결과를 계산한다.
- [ ] 양수의 지수 법칙과 제곱근을 작은 숫자로 검산하고 밑이 2나 10일 때 지수와 로그의 역관계를 확인한다.
- [ ] 간단한 수열을 항으로 나열하고 시그마 표기를 실제 덧셈으로 풀어 평균을 계산한다.

## 2단계: 검색과 추천에 필요한 수학, 기간 미정

### 학습 순서

1. 집합과 함수로 candidate universe, filter와 mapping을 표현한다.
2. 비율, 빈도와 로그를 이용해 TF, IDF와 BM25 score 구성 요소의 방향을 읽는다.
3. 벡터, 행렬, 내적과 cosine으로 작품 특징과 유사도를 계산한다.
4. 확률, 조건부확률, Bayes 정리, 평균과 분산으로 노출과 클릭 데이터를 읽는다.
5. 미분을 변화율과 기울기로 이해한 뒤 loss와 gradient에 연결한다.
6. 선형 회귀와 로지스틱 회귀를 예측 baseline으로 이해한다.
7. 정확도와 MRR, Recall@K, NDCG@K가 측정하는 대상의 차이를 확인한다.

### 통과 gate

- [ ] 작은 작품 집합을 candidate universe로 두고 조건에 따른 filter와 함수 mapping을 표기한다.
- [ ] 주어진 식과 작은 TF, 문서 빈도 및 문서 수로 IDF를 계산하고 BM25 구성 요소가 score를 올리거나 포화시키는 방향을 설명한다.
- [ ] 작품 세 개의 작은 벡터로 내적과 cosine을 손으로 계산한다.
- [ ] 행렬의 shape를 읽고 가능한 곱셈과 불가능한 곱셈을 구분한다.
- [ ] 작은 표본의 평균, 분산과 조건부확률을 계산하고 Bayes 정리로 사후확률을 갱신하며 각 분모를 설명한다.
- [ ] 한 변수 loss의 값을 바꿔 기울기와 gradient가 parameter를 어느 방향으로 움직이는지 설명한다.
- [ ] 작은 예제 두 개로 선형 회귀와 로지스틱 회귀의 예측 대상, 출력과 loss 차이를 설명한다.
- [ ] 분류 정확도와 검색/추천 Top K의 MRR, Recall@K 및 NDCG@K가 왜 다른지 작은 예로 보인다.

## 0단계 고정 자료와 두 교재의 사용법

0단계의 주자료는 EBSMath 한 곳으로 고정한다. 각 경로에서 개념 콘텐츠 하나와 문제 풀이 콘텐츠 또는 문제지 하나만 고르고, 같은 단원의 대표 문제 세 개를 풀이 과정과 함께 남긴다. 막히면 다른 강의를 추가하지 않고 바로 앞 경로로 돌아간다.

| 순서 | EBSMath 경로 | 남길 결과 |
|---|---|---|
| 1 | 초5 1학기, 자연수의 혼합계산 | 괄호와 사칙연산 순서를 설명한 풀이 |
| 2 | 초3 1학기 분수와 소수 → 초4 2학기 분수/소수 덧셈과 뺄셈 → 초5 1학기 약분과 통분, 분수의 덧셈과 뺄셈 → 초5 2학기 분수/소수 곱셈 → 초6 분수/소수 나눗셈 | 분수와 소수의 사칙연산, 약분/통분과 상호 변환 |
| 3 | 초6 1학기 비와 비율 → 2학기 비례식과 비례배분 | 비율, 백분율과 단위 변환 풀이 |
| 4 | 중1 정수와 유리수 | 수직선, 부호와 사칙연산 풀이 |
| 5 | 중1 소인수 분해의 거듭제곱 표기 → 중3 제곱근과 실수의 제곱근 | 거듭제곱과 제곱근을 작은 수로 검산한 풀이 |
| 6 | 중1 문자와 식의 문자의 사용과 식의 계산 | 식에 값을 대입하고 예상 범위로 검산한 풀이 |

1번부터 어렵다면 EBSMath 안에서 초3과 초4의 덧셈, 뺄셈, 곱셈과 나눗셈으로 내려가 시작한다. 단위 변환은 3번의 비율 문제에 길이, 시간과 데이터 크기 단위를 넣어 연습한다. 한 경로를 끝내기 전에 시리즈나 게임을 추가 수집하지 않는다.

공개 목차에서 구입한 두 책 모두 기초 개념을 일부 다루지만, 분수, 소수와 비율을 순서대로 복원하는 주교재인지는 확인되지 않는다. 두 책은 EBSMath에서 해당 선행 개념을 익힌 뒤 참고하며 책 순서를 그대로 따라가지 않는다.

| 교재 | 시작 시점 | 먼저 볼 범위 | 보류 범위 |
|---|---|---|---|
| 개발자를 위한 필수 수학 | 0단계와 1단계 | 1장의 정수, 연산 순서, 변수, 함수, 거듭제곱과 로그 중 이해되는 부분 | 선형대수는 2단계, 회귀, 분류와 신경망은 그 이후 |
| 컴퓨팅 사고력을 키우는 이산수학 3판 | 1단계 | 집합과 함수, 1장의 수 표현은 필요할 때 참조 | 행렬과 확률은 2단계, 관계, 그래프, 트리와 증명은 그 이후 |

책에서 막히면 이해력이 부족한 것으로 결론 내리지 않는다. 책이 생략한 선행 개념을 찾아 0단계나 1단계로 돌아가는 신호로 본다.

## 한 주 운영법

최근 4주 실제 개인 학습 시간의 중앙값을 주간 총상한으로 먼저 정한다. 총상한이 허용하면 수학에 주 5시간에서 10시간을 배정하되, 중앙값이 5시간 미만이면 그 상한 안에서 시작하고 학습 시간을 억지로 늘리지 않는다. 2027년에 검색/추천과 DevOps가 병렬로 시작되어도 시간을 별도로 더하지 않고 같은 총상한 안에서 재배분한다.

최근 4주 기록이 없다면 주축인 수학은 기존 생활 패턴의 총학습 시간을 늘리지 않은 채 4주를 기록하고, 중앙값이 생긴 뒤 시간 배정을 다시 정한다. 총상한이 생기기 전에는 병렬 트랙을 추가하지 않는다.

- 평일: 개념 하나를 읽고 예제 두세 개를 손으로 푼다.
- 주말: 틀린 이유를 다시 적고 같은 유형을 숫자만 바꿔 풀어본다.
- 비율: 손계산 60%, 읽기와 말로 설명하기 25%, 코드 검산 15%를 기본으로 한다.
- 산출물: 매 단원마다 대표 풀이, 틀린 이유와 다시 푼 결과를 한 묶음으로 남긴다.
- 휴식: 같은 문제에 40분 이상 막히면 표시하고 선행 개념으로 돌아간다.

## 수학 이후의 2027 전환

2단계 gate를 통과한 뒤 [[2027-Search-Recommendation-Roadmap|2027 검색 엔진 우선, 추천 시스템 전환 로드맵]]을 시작한다.

1. 통합 discovery의 ID, eligibility, query/event와 평가 계약을 먼저 고정한다.
2. 검색 엔진 baseline과 품질 평가를 먼저 통과한다.
3. 같은 계약을 재사용해 추천 baseline, taxonomy 후보와 서빙 검증으로 넘어간다.
4. 1차 성공선 뒤에는 [[Search-Recommendation-Discovery-Learning-Path|검색과 추천 디스커버리 학습 경로]]의 남은 단계를 이어간다.

검색을 먼저 두는 이유는 추천의 수학적 선행조건이어서가 아니다. 현재 업무 우선순위가 검색에 있고, 작품 정본, eligibility, 요청/노출 로그와 평가 계약을 먼저 검증하면 추천 실습에서도 재사용할 수 있기 때문이다. 검색과 추천을 동시에 시작하지 않는다.

DevOps는 검색/추천 완료에 종속시키지 않는다. 2027년 6월 실행 점검과 주간 총상한을 통과한 경우에만 [[2027-DevOps-Practical-Roadmap|2027 DevOps 실전 로드맵]]에 따라 제한적으로 시작한다.

## 월별 회고 질문

- 무엇을 외웠는가보다 어떤 식을 혼자 읽고 계산할 수 있게 됐는가
- 막힌 원인이 현재 단원인가, 생략된 선행 개념인가
- 공부 시간을 늘려야 하는가, 단위를 더 작게 나눠야 하는가
- 업무상 필요한 검색/추천 지식과 개인 수학 진도를 섞어 압박하고 있지 않은가
- 다음 달에도 현재 단계가 주축이어야 하는가

# 2026 하반기 수학 기초 로드맵

수학 기초가 거의 없다는 현재 출발점을 인정하고 이미 구입한 두 교재를 장기 기반 과정으로 끝까지 학습한다. 범위는 《개발자를 위한 필수 수학》 1장부터 6장과 《컴퓨팅 사고력을 키우는 이산수학 3판》 1장부터 12장이다. 신경망과 경력 조언은 수학 gate에서 제외하고, 교재 예제에서 확인된 선행 개념의 빈칸만 별도 자료로 보충한다. 2026년 하반기의 개인 학습 주축은 수학 하나이며 체계적인 검색과 추천 실습은 두 교재의 지정 범위와 수학 gate를 닫은 뒤에 시작한다.

## 결정 요약

| 시기 | 주축 | 허용하는 현업 보충 | 하지 않는 것 |
|---|---|---|---|
| 2026 하반기 | 필수 수학 1-6장과 이산수학 1-12장 | 교재에서 실제로 막힌 선행 개념과 업무에 필요한 검색/추천 개념 | 신경망, 외부 기초 과정 선완주, 체계적 검색/추천 진도, DevOps, 자격증, 코딩 테스트 |
| 수학 2단계 gate 이후 | 검색 엔진 baseline과 품질 평가, 추천 baseline, taxonomy 후보와 logging/serving 검증 | 부족한 수학을 필요한 지점에서 보충 | 검색과 추천 동시 착수, 고급 모델부터 시작하기 |

검색과 추천의 전환 조건은 [[2027-Search-Recommendation-Roadmap|2027 검색 엔진 우선, 추천 시스템 전환 로드맵]]에서 관리한다. DevOps는 추천과 taxonomy 1차 성공선 뒤 [[2027-DevOps-Practical-Roadmap|2027 DevOps 실전 로드맵]]으로 전환한다.

수학을 모두 끝낸 뒤에만 업무를 할 수 있다는 뜻은 아니다. 업무에서는 기존 [[OpenSearch|OpenSearch 학습 지도]]와 [[Recommendation-System-Architecture|추천 시스템 지식 지도]]를 참고해 필요한 개념을 그때그때 연결하되, 개인 학습에서는 교재를 중심으로 한 현재 단계만 진행한다.

## 학습 원칙

- 구입한 두 교재를 먼저 펼친다. 외부 강의나 초중등 과정 전체를 선행조건으로 두지 않는다.
- 시작점을 증명하기 위한 진단시험이나 별도 퀴즈를 요구하지 않는다. 교재 예제와 워크북을 풀면서 이해 여부와 막힌 선행 개념을 확인한다.
- 지정한 모든 장을 읽되 모든 연습문제 정답을 완료 조건으로 삼지 않는다. 장별 핵심 개념을 설명하고 대표 문제를 풀이하면 다음 장으로 진행한다.
- 고정 종료일보다 gate를 우선한다. 2026년 안에 모든 수학을 끝내지 못해도 실패가 아니다.
- 초반에는 손계산과 풀이 설명이 중심이다. Python 검산은 식을 읽을 수 있게 된 뒤 붙인다.
- 한 번에 한 절과 한 개념만 다룬다. 막힌 원인이 확인되면 그 개념만 보충한 뒤 같은 교재 위치로 돌아온다.
- 교재 완독, 문서 수와 공부 시간 자체를 성과로 계산하지 않는다.

## 0단계: 필수 수학 1장, 기간 미정

### 학습 범위

1. 1.1 정수론부터 1.7 로그까지
2. 1.8 오일러 수와 자연로그
3. 1.9 극한과 1.10 미분
4. 1.11 적분
5. 교재 예제에 등장하는 정수, 음수, 분수, 소수, 비율과 백분율 계산

### 통과 gate

- [ ] 교재 예제의 괄호, 부호와 사칙연산 순서를 풀이 과정으로 설명한다.
- [ ] 변수와 상수를 구분하고 식에 값을 대입해 예상 범위로 결과를 검산한다.
- [ ] 함수의 입력과 출력을 표시하고 간단한 예를 직접 만든다.
- [ ] 시그마 표기를 실제 덧셈으로 풀고 작은 평균을 계산한다.
- [ ] 작은 수로 거듭제곱을 계산하고 지수, 로그, 오일러 수와 자연로그의 관계를 확인한다.
- [ ] 극한, 미분과 적분을 각각 접근값, 변화율과 누적량의 작은 예로 설명한다.
- [ ] 교재 진행을 막은 분수, 소수, 비율 또는 백분율이 있었다면 해당 계산을 보충하고 같은 예제로 돌아와 풀이한다.

## 1단계: 필수 수학 2장부터 6장, 기간 미정

### 학습 범위

1. 2장 확률
2. 3장 기술 통계와 추론 통계
3. 4장 선형대수학
4. 5장 선형 회귀
5. 6장 로지스틱 회귀와 분류

### 통과 gate

- [ ] 결합확률, 조건부확률과 Bayes 정리를 계산하고 이항분포와 베타분포의 쓰임을 작은 예로 설명한다.
- [ ] 평균과 분산을 계산하고 모집단, 표본, 편향, 추론과 t 분포의 차이를 작은 데이터 예로 설명한다.
- [ ] 벡터의 내적과 cosine 유사도, 선형 변환 및 행렬 곱을 계산하고 행렬식, 역행렬과 고유 벡터의 의미를 작은 행렬로 설명한다.
- [ ] 선형 회귀의 잔차와 제곱 오차, gradient, 과대적합, 결정 계수와 train/test 분리를 하나의 작은 예로 연결한다.
- [ ] 로지스틱 회귀의 확률 출력과 로그 오즈를 계산하고 오차 행렬, Bayes 분류, ROC/AUC 및 클래스 불균형을 설명한다.

## 2단계: 이산수학 전체와 검색 연결, 기간 미정

### 학습 순서

1. 《컴퓨팅 사고력을 키우는 이산수학 3판》 1장부터 4장까지 수의 표현, 논리, 증명과 집합
2. 5장부터 7장까지 행렬, 관계와 함수
3. 8장부터 10장까지 그래프, 트리와 부울대수
4. 11장 순열, 조합과 확률 및 12장 알고리즘
5. 두 교재의 범위 밖인 TF, IDF, BM25와 MRR, Recall@K 및 NDCG@K는 연결 문서에서 보충

### 통과 gate

- [ ] 작은 수를 두 진법으로 변환하고 해당 진법의 사칙연산과 컴퓨터 내부 표현 예를 풀이한다.
- [ ] 명제와 논리 연산을 진리표로 확인하고 직접, 간접과 귀납 증명 예를 각각 하나씩 설명한다.
- [ ] 집합, 합성관계, 동치관계, 부분순서와 함수로 candidate universe, filter 및 mapping을 표현한다.
- [ ] 작은 행렬의 연산, 행렬식, 역행렬과 연립일차방정식 예를 풀이한다.
- [ ] 같은 대상을 그래프와 트리로 표현하고 오일러 경로, 해밀턴 경로 및 이진트리의 적용 차이를 설명한다.
- [ ] 부울함수, 순열, 조합, 확률과 확률분포를 작은 예제로 계산한다.
- [ ] 같은 문제를 두 알고리즘으로 표현하고 입력 크기에 따른 효율성 차이를 설명한다.
- [ ] IDF와 BM25 구성 요소를 계산하고 MRR, Recall@K 및 NDCG@K의 차이를 작은 예로 보인다.

## 두 교재의 사용 순서와 보충 규칙

| 순서 | 중심 자료 | 범위 |
|---|---|---|
| 1 | 개발자를 위한 필수 수학 | 1장 기초 수학과 미적분 |
| 2 | 개발자를 위한 필수 수학 | 2장 확률과 3장 통계 |
| 3 | 개발자를 위한 필수 수학 | 4장 선형대수학 |
| 4 | 개발자를 위한 필수 수학 | 5장과 6장 회귀 및 분류 |
| 5 | 이산수학 3판 | 1장부터 4장까지 |
| 6 | 이산수학 3판 | 5장부터 7장까지 |
| 7 | 이산수학 3판 | 8장부터 12장까지 |
| 8 | 검색과 추천 연결 문서 | BM25 구성 요소와 Top K 순위 평가 수학 |

필수 수학 7장 신경망은 제외하고 8장 경력 조언은 선택 독서로 둔다. 둘 다 수학 gate와 다음 단계 진입 조건에 포함하지 않는다. 이산수학은 1장부터 12장까지 모두 현재 수학 gate에 포함한다. 두 책을 동시에 진행하지 않고 표의 현재 행 안에서도 한 절씩 진행한다.

보충 자료는 고정 선행 과정이 아니다. 교재에서 막히면 페이지와 예제, 막힌 계산을 먼저 적고 그 개념 하나만 EBSMath 같은 자료에서 확인한다. 대표 문제 세 개를 풀이한 뒤 원래 교재의 같은 예제로 돌아오며, 막힘이 없다면 EBSMath를 열지 않는다. 한 개념을 해결하기 전에 다른 강의나 교재를 추가 수집하지 않는다.

## 한 주 운영법

수학은 현재 유일한 체계적 개인 학습 트랙이다. 주 5시간에서 10시간은 참고 범위로만 두고 생활에서 실제로 확보되는 시간 안에서 진행한다. 시간이 부족한 주에 진도를 압축하거나 다음 트랙을 병행하지 않는다.

기간 미정인 gate를 억지로 당기지 않고, 한 주 안에 검토 가능한 풀이 묶음으로 진도를 직접 확인한다.

- 평일: 현재 교재의 한 절을 읽고 예제나 워크북 문제 두세 개를 손으로 푼다.
- 주말: 틀린 이유를 다시 적고 같은 유형을 숫자만 바꿔 풀어본다.
- 비율: 손계산 60%, 읽기와 말로 설명하기 25%, 코드 검산 15%를 기본으로 한다.
- 산출물: 교재명, 절과 페이지, 대표 풀이, 틀린 이유와 다시 푼 결과를 한 묶음으로 남긴다.
- 휴식: 같은 문제에 40분 이상 막히면 표시하고 선행 개념으로 돌아간다.

## 수학 이후의 단일 전환 순서

2단계 gate를 통과한 뒤 [[2027-Search-Recommendation-Roadmap|2027 검색 엔진 우선, 추천 시스템 전환 로드맵]]을 시작한다.

1. 통합 discovery의 ID, eligibility, query/event와 평가 계약을 먼저 고정한다.
2. 검색 엔진 baseline과 품질 평가를 먼저 통과한다.
3. 같은 계약을 재사용해 추천 baseline, taxonomy 후보와 서빙 검증으로 넘어간다.
4. 추천과 taxonomy 1차 성공선 뒤 DevOps 실전으로 전환한다.
5. DevOps 핵심 4 통과 뒤 [[Search-Recommendation-Discovery-Learning-Path|검색과 추천 디스커버리 학습 경로]]의 남은 고급 단계를 재개한다.

검색을 먼저 두는 이유는 추천의 수학적 선행조건이어서가 아니다. 현재 업무 우선순위가 검색에 있고, 작품 정본, eligibility, 요청/노출 로그와 평가 계약을 먼저 검증하면 추천 실습에서도 재사용할 수 있기 때문이다. 검색과 추천을 동시에 시작하지 않는다.

DevOps는 추천과 taxonomy 1차 성공선 뒤에만 시작한다. 앞 단계가 늦어지면 DevOps 시작 목표도 압축하지 않고 뒤로 미룬다.

## 월별 회고 질문

- 무엇을 외웠는가보다 어떤 식을 혼자 읽고 계산할 수 있게 됐는가
- 막힌 원인이 현재 단원인가, 생략된 선행 개념인가
- 공부 시간을 늘려야 하는가, 단위를 더 작게 나눠야 하는가
- 업무상 필요한 검색/추천 지식과 개인 수학 진도를 섞어 압박하고 있지 않은가
- 다음 달에도 현재 단계가 주축이어야 하는가

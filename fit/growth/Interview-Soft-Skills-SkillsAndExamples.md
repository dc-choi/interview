# 인성면접 가이드 — 예시 질문·탈락 요인·체크리스트

## 인성면접 예시 질문과 해석

### 질문 1: 팀 프로젝트에서 기여 역량과 동료에게 바라는 역량
> 업무를 할 때 어떤 요소를 중요한 가치로 생각하는지, **자기 객관화** 파악

### 질문 2: 쇼핑몰 장바구니→주문/결제 프로세스 설명
> 회원, 장바구니, 쿠폰, 상품 관리, 주문, 결제, 배송, 반품, 환불, CS, 정산에 대한 **업무 파악 능력**

### 질문 3: 좋은 코드, 견고한 소프트웨어가 무엇인가?
> 꾸준한 관심을 가지고 **개발자로서 성장을 지속하는지** 확인. 스스로 고민을 많이 해봐야 함

### 질문 4: 동료 코드가 배포마다 버그/장애를 유발하면?
> 실수를 방지하기 위한 **제도적인 장치**에 대한 고민 (코드 리뷰, 테스트, CI/CD 게이트 등)

### 질문 5: 새 기술 적용 vs 일정 부족, 본인의 입장과 설득 방법
> 기술·품질·일정 간의 **트레이드오프** 조정과 합리적인 안을 찾아가는 방법

### 질문 6: 어려웠던 문제 해결, 동료에 긍정적 영향을 미친 사례
> 조직에 **좋은 영향력**을 미치는 사람인가, 임팩트 있는 업무를 어떻게 해결했는가

## 소프트웨어 품질관리와 기술부채

### 품질과 시간의 트레이드오프
- 모든 업무에는 데드라인이 존재하며, 일정 지연은 비용
- 개인의 입장과 회사의 입장 중 무엇이 우선인가를 항상 고려

### 기술부채 관리 영역
- 소스포비아 (코드 공포증) 해소
- 업무 속도 저하 원인 분석
- 결함/버그 추적 관리
- 히스토리 관리
- 테스트 및 자동화
- 아키텍처 개선
- 코딩 가이드, 코드 리뷰 가이드 배포

## 기술면접 탈락 요인과 사례

### 자주 나오는 탈락 사유
1. **기술 스택은 일치하지만 구사 수준/이해도가 기준 미달**
   - 이력서에 적혀 있는 건 많은데 정작 꼬리 질문에서 막힘
2. **이력서에 적은 내용을 증명하지 못하는 경우**
   - "주도했다" / "최적화했다" / "설계했다"의 실제 근거가 나오지 않음
3. **주장은 있으나 실전에서 증명 불가**
   - "클린 아키텍처를 중시합니다" → 왜 그게 필요했는지, 실제로 어떻게 적용했는지 설명 불가
4. **연차 대비 아키텍처/인프라 지식 부족**
   - 3년차가 DB 인덱스 / 트랜잭션 격리 수준을 모름
   - 5년차가 로드밸런서 / 캐시 계층 설계를 못 함
5. **기술적 답변이 모호하고 장황해서 커뮤니케이션이 어려운 경우**
   - 질문의 핵심을 파악 못 하고 주변만 맴돔
6. **분명히 잘 아는 것 같은데 대답을 잘 못하는 경우**
   - 공부는 되어 있는데 **설명 훈련**이 안 됨 → "아는 것"과 "설명할 수 있는 것"은 다름

### 왜 이런 사례들이 나오는가
- 기본적으로 **업무를 통해 성장하는 것이 가장 중요**
- "이걸 왜 쓰게 되었는가?"에 대한 **근본적인 호기심**이 있어야 함
- 사용하는 기술셋을 **내 것으로 만들려는 노력**이 있어야 함
- 업무에서 그런 기회가 없다면 **스터디, 개인/팀 프로젝트**로 개발·운영 이슈를 직접 고민하고 해결해본 경험을 쌓아야 함

### 기초 체력
- **CS, 알고리즘, 네트워크, 자료구조, 주력 언어, 프레임워크**
- 이걸 **설명할 수 있는 소프트 스킬**을 갖추어야 함 — 아는 것만으로는 부족
- 면접관은 지원자의 설명을 통해서만 판단할 수 있음

### 탈락했을 때 점검 체크리스트
- [ ] 어떤 질문에서 막혔는가? (기술? 경험? 태도?)
- [ ] 이력서에 쓰인 내용을 꼬리 질문 3단계까지 답할 수 있는가?
- [ ] 같은 질문이 다시 나오면 답할 수 있도록 노트 정리했는가?
- [ ] 연차 대비 부족한 영역이 무엇인지 객관적으로 확인했는가?
- [ ] 다음 면접 전까지 보강할 1~2개 핵심 주제를 정했는가?

## 인성면접 주의사항 체크리스트

- [ ] 이론적 내용이나 전체 과정을 머릿속에서 정리
- [ ] 리더십, 그라운드룰에 대한 이해
- [ ] 회사의 문화, 인재상을 미리 찾아서 정리
- [ ] 실수를 방지하기 위한 제도적인 장치 고민
- [ ] 일정과 품질의 트레이드오프 고려
- [ ] 내 직무의 역량이나 열정 정리
- [ ] 조직의 컬처핏, 협업이나 커뮤니케이션 사례
- [ ] 동료들에게 긍정적인 영향을 미친 사례 준비
- [ ] **외워서 말하지 않기** — 동작 방식, 운용 방식, 문제 해결 경험을 정리해두고 **말로 설명하는 훈련**

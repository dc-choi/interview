# 개발자 성장 단계

"얼마나 많이 짰는가"보다 **"어떤 추상화까지 다루고 있는가"** 가 연차를 가른다. 성장은 선형이 아니라 **계단형** — 각 단계마다 벽이 있고, 그 벽을 넘기까지 긴 정체기가 있다. 코드 → 데이터 → 프레임워크 → 알고리즘/패턴 → 데이터 심화 순으로 추상 수준이 올라가며, 한 계단마다 **새로운 사고방식** 이 필요하다.

## 성장의 5단계

### 1단계: Computational Thinking

**습득 과제**: 일상 사고를 **논리적·절차적 분해**로 전환.

- 어려움 1: "밥을 먹다"라는 한 문장을 행동 단위로 쪼개기
- 어려움 2: 변수·조건문·함수 같은 언어 문법 습득
- 극복: 쉬운 문제를 **많이** 풀며 체득. 이론보다 몸

### 2단계: 데이터 이해

**습득 과제**: 추상 개념을 **데이터 구조**로 표현.

- 어려움 1: "학생" → `{name, id, grade}`로 분해
- 어려움 2: 자료구조·포인터·객체·관계 같은 **표현 방식**
- 비전공자 목표: 2단계의 첫 어려움까지 극복하면 개발자와 소통 가능

### 3단계: 프레임워크 이해

**습득 과제**: "자동으로 되던 것들"이 실은 **누군가 만든 추상**임을 깨닫고, 그 내부 동작을 이해.

- 어려움 1: OS·라이브러리·IDE·네트워크 스택이 추상임을 인지
- 어려움 2: 프레임워크 내부 메커니즘·설계 의도 파악 ("왜 이렇게 만들 수밖에 없었나")
- 실천: 코드 리뷰·공식 문서·내부 구현 읽기

### 4단계: 알고리즘과 패턴

**습득 과제**: 지속적 이론 학습 + **경험으로만 체득되는** 패턴 흡수.

- 어려움 1: 시간복잡도·자료구조·알고리즘에 대한 장기 학습 의지
- 어려움 2: 동시성·분산·실패 복구 같은 **경험적 이해 필수** 영역
- 이 단계를 넘으면 "만렙 개발자" — 평균 8~10년차 이상

### 5단계: 데이터 심화

**습득 과제**: 소프트웨어의 본질이 **데이터에서 나오는 통찰**임을 이해.

- 빅데이터·머신러닝·분석 이론을 실무에 결합
- **외부 지식 필요** — 통계·수학·경제·도메인
- 소프트웨어 기술만으로는 닿지 못하는 영역

## 성장의 원리

### 계단형

성장은 매일 균등히 쌓이지 않는다. **한 단계 안에서는 정체 → 다음 단계로 도약 → 또 정체**. 벽 앞에서 "내가 발전 없다"고 느끼는 시기가 반복되는 건 정상.

### 신체 학습

한 번 체득한 단계는 **자전거 타기처럼** 쉽게 잊히지 않는다. 기술 스택·언어·프레임워크는 바뀌어도 **하위 단계의 사고 패턴은 재사용된다**.

### 좋은 교육/환경

각 단계에서 **충분히 머물게** 해주는 것이 좋은 환경. 건너뛰게 하면 위층에서 무너진다. "좋은 사수·팀"이 중요한 이유.

## 주니어 → 미드레벨 전환 7단계

연차 1~3년에 **구체적으로 연습해야 할 행동 체크리스트**. 성장 단계 모델과 교차 적용.

### 1. 시니어 피드백 경청

- 처음엔 방어적 감정이 들 수 있음 — **받아들이는 근육**을 기르는 게 우선
- 리뷰어의 기준을 모방하다 어느 순간 내 기준으로 내재화
- **아프고 빠른 피드백** 이 친절하고 모호한 피드백보다 낫다

### 2. 코딩-테스트-리팩터-테스트 이중 검증

- 작동하는 코드에서 끝내지 말고 **개선 루프** 1회 더
- 작동 단계가 끝난 후 테스트는 경계 케이스까지

### 3. 스스로 결정 내리기

- 모호한 상황에서 본인 판단으로 작은 결정 누적 → **결정 근육**
- 완벽을 기다리다 결정을 미루면 생산성 손실
- 작은 결정을 틀리며 **판단 능력을 복리**로 키움

### 4. 진행·기대치·문제 소통

- 현재 상태, 남은 리스크, 필요한 도움을 **선제적으로** 전달
- "사고는 감추지 말고 먼저 알린다" — 신뢰의 근간
- 실시간 공유가 사후 보고보다 항상 싸다

### 5. Git 숙련

- 단순 commit/push가 아니라 **rebase·cherry-pick·bisect·reflog**까지
- 오픈소스 PR·대형 레포 커밋 히스토리 읽기로 학습

### 6. 실수에서 배우기

- 실수를 **학습 소재로** 다룸 — 포스트모템 습관
- "다시 안 하겠다"가 아니라 **구조적 방지책** 도출
- 실수 로그를 개인 위키에 쌓기

### 7. 360도 관점

- 단순 구현자를 넘어 **비즈니스·사용자·운영** 관점까지
- 기대 이상의 산출물을 만드는 개발자로 전환
- 이 7번째 단계가 미드레벨 이상을 가른다

## 실제 성장 궤적 사례 (3~4년차)

### 1년차: "레알 고난과 역경"

- 매일 자괴감, 체계 없이 공부
- 첫 프로덕션 배포·실사용자 피드백이 원동력
- 인지 과부하 상태가 정상

### 2~3년차: "나는 얼마나 아는가?"

- "1인분은 한다"는 인정 + **성장 체감 저하**
- 새 기술 시도·토이 프로젝트·기술 글쓰기
- 2년 반 전후에 이직 결심이 흔함 — 안전지대 탈출로 급성장

### 3~4년차: "신세계, 위기는 기회"

- 팀 이동·기술 전환(Django → FastAPI·NestJS 등)
- **동료 퇴사 인수인계** 같은 사건이 급성장 트리거
- 인프라~앱 A-Z 독립 구축 경험이 시니어 자격의 디딤돌

## 성장을 가속하는 환경 조건

- **도전적 과제 비중 70%** — 해본 일로만 채워지면 정체
- **피드백 루프 짧음** — 코드 리뷰·장애 리뷰·모니터링
- **자율성 + 책임** — 결정권과 결과 소유
- **동료 밀도** — 같이 일하는 3~4명 중 배울 사람이 있는가
- **실패를 학습으로** 다루는 문화

## 정체기 탈출 트리거

실전에서 성장을 한 단계 끌어올린 계기들.

- **이직·팀 이동** — 새로운 스택·도메인 강제 노출
- **동료 퇴사 인수인계** — 기존에 안 보던 영역 책임
- **실서버 장애 대응** — 이론으로만 알던 것이 실감
- **A-Z 독립 구축** — 인프라·CI/CD·DB까지 전체 책임
- **공개 글쓰기·발표** — 모호한 이해를 언어로 정리

## 흔한 함정

- 1년차에 "난 안 되는 것 같다" → 인지 부하가 정상 상태. 6개월만 버티면 지도가 생김
- 2~3년차 정체기에 **이직만이 답**이라고 판단 → 내부에서 해결 가능한지 먼저 검토
- "완벽히 이해한 뒤 이동" → 한 단계의 완벽은 불가능. **80% 체감 후 다음 단계**
- 피드백을 감정으로 받음 → 기술·사람 분리 훈련
- Git을 `commit/push`만 쓰며 몇 년 → 근본 도구 숙련이 낮으면 리더 경로에 걸림돌

## 면접 체크포인트

- 본인이 5단계 중 **어느 단계를 돌파 중**이라고 말할 수 있는가
- 주니어→미드 7단계 중 본인의 **약점 2가지**를 자각했는가
- 성장이 **계단형**이라는 사실을 체득한 증거(정체기 이야기)
- 정체기 탈출 트리거 2~3개를 경험했는가
- 실패·피드백을 다루는 **개인 습관**이 있는가

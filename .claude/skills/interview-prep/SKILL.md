# 면접 준비 문서 생성 스킬

채용공고 URL과 이력서/포트폴리오를 분석하여 `fit/interview/{회사명}/{차수}/Interview-Prep-{회사명}-{차수}.md` 문서를 생성한다. **차수별 서브폴더(`1st/`, `2nd/`, `coffeechat/` 등)는 필수** — 차수 자료를 회사 폴더 루트에 두지 않는다 (CLAUDE.md 회사별 폴더 규칙). 전체 워크플로우(마스터 fork, prep 정렬, vault 흡수)는 CLAUDE.md의 "신규 회사 면접 준비 워크플로우"를 우선 적용한다.

## 답변 배치

- 개인 경험, FIT, 개인 기술 경험의 재사용 답변 본문은 `fit/interview/common/`에서만 관리한다.
- 예상 질문을 준비 내용 없이 질문만 저장하지 않는다.
- `fit/interview/prep/`의 사실형 기술 질문에는 질문 의도, 간결한 핵심 답변, 흔한 오답, 꼬리질문, 학습 정본이나 출처를 반드시 포함한다.
- 설계형 기술 질문에는 하나의 정답을 만들지 말고 답변 골격, 판단 기준, 대안과 트레이드오프, 꼬리질문을 반드시 포함한다.
- 개인 경험과 FIT 질문에는 준비 포인트와 `common/` 마스터 링크만 두고 답변 본문은 복제하지 않는다.
- 상세 기술 설명은 `tech/` 정본을 반복하지 말고 링크한다.
- 회사별 면접 문서는 선택한 마스터 답변과 관련 기술 내용을 흡수해 자기완결적으로 작성한다.

## 입력

- **필수**: 채용공고 URL
- **필수**: 이력서 경로
- **선택**: 포트폴리오 경로 (프로젝트 상세, 기술 스택, 성과 등 추가 정보)
- 필수 인자가 부족하면 사용자에게 질문

## 처리 순서

### 1단계: 입력 파싱
- 첫 번째 인자: 채용공고 URL
- 두 번째 인자: 이력서 경로
- 세 번째 인자 (있으면): 포트폴리오 경로
- 필수 인자가 부족하면 사용자에게 질문

### 2단계: 데이터 수집 (병렬 처리)
다음을 선별하지 말고 한 번에 수행:
1. **채용공고 수집**: WebFetch로 URL 내용을 상세히 추출 (회사 소개, 포지션명, 주요 업무, 자격 요건, 우대 사항, 근무 조건, 채용 절차 등)
2. **이력서 읽기**: Read로 이력서 PDF 내용 확인
3. **포트폴리오 전체 읽기** (경로 있으면): 프로젝트 상세, 기술 스택, 성과를 예상 질문과 어필 포인트에 활용
4. **common 전체 읽기**: `My-*.md` 9개, `Common-Interview-Questions.md`, `Interview-Retro-Template.md`
5. **prep 전체 읽기**: 인덱스와 템플릿을 제외한 가이드 14개 전체. 면접 형식별 문서는 JD를 보고 적용 여부를 판단하되 읽기는 생략하지 않는다.
6. **최신 지표 검증**: `~/myown/school-manage/docs/business/STATUS.md`에서 수치와 기준일을 다시 확인한다. 활성, 누적, 레코드 수를 서로 바꾸어 쓰지 않는다.
7. **tech vault 흡수**: 마스터 기술 카드와 회사 특이 도메인의 정본을 찾아 정량, 비교, 심화 꼬리를 회사 문서 본문에 흡수한다. 회사 문서에 vault 링크만 남기지 않는다.
8. **템플릿과 예시 읽기**: `fit/interview/prep/Interview-Prep-Template.md`, `fit/interview/actionpower/1st/Interview-Prep-ActionPower.md`

### 3단계: 문서 생성
`fit/interview/{회사명}/{차수}/Interview-Prep-{회사명}-{차수}.md` 파일을 생성한다. 차수 폴더가 없으면 새로 만든다.

#### 프론트매터
```yaml
---
tags: [fit, interview, {회사영문명소문자}]
status: active
category: "Interview - Fit"
aliases: ["{회사영문명} Interview Prep", "{회사한글명} 면접 준비"]
---
```

#### 문서 구조 (템플릿 기반)

**헤더**:
- 면접일: TBD (알 수 있으면 기재)
- 채용 공고 URL
- 템플릿 링크

**1. JD 분석**:
- 회사 개요 테이블 (회사명, 서비스, 단계, 규모, 도메인, BM 추정)
- 포지션 정보 테이블
- 핵심 키워드 (면접에서 반복 등장할 것들)
- 자격요건 vs 내 경험 매칭 (강/중/약)
- 우대사항 vs 내 경험 매칭 (강/중/약)
- 기술 스택 비교 (동일/유사/없음)

**2. 회사 맞춤 FIT 답변**:
- 1분 자기소개: 이름/역할 → 도메인 경험 → 핵심 성과 → 핵심 철학 → 왜 이 회사 → 마무리
- 지원 동기: 회사의 미션 → 내 경험 → 연결점 → 기여하고 싶은 것
- 이직 사유: 성장한 점 → 한계 → 다음 도전 → 꼬리 질문 대비

**3. 예상 질문 (면접관 → 나)**:
- 이력서 기반 기술 질문 (테이블: 질문 | 준비 포인트 | 관련 지식)
- JD 기반 기술 질문 (테이블: 질문 | 준비 포인트 | 관련 지식)
- 서비스 맥락 질문 (테이블: 질문 | 준비 포인트)
- 컬처핏/소프트스킬 질문 (체크리스트)

**4. 역질문 (나 → 면접관)**:
- 기술/아키텍처 (3~4개)
- 팀/조직 (2~3개)
- 성장/기대치 (2~3개)

**5. 면접 준비 체크리스트**:
- 보강이 필요한 기술 영역 (테이블: 영역 | 관련 문서 | 복습 완료)
- 과제 대비 (채용 절차에 과제가 있으면)
- 강하게 어필할 포인트 (매칭도 "강"인 항목)
- 주의사항

### 4단계: Job Search Tracker 업데이트
- `fit/job-search/Job-Search-Tracker.md`의 해당 기업 행에 채용공고 링크 추가 (없으면)

### 5단계: 결과 보고
- 생성된 문서 경로
- 강점/갭 요약
- 우선 준비 사항 안내

## 작성 규칙

- 관련 지식 컬럼에는 레포 내 지식 문서를 Obsidian 위키링크로 연결: `[[파일명|표시명]]`
  - 지식 문서가 있는지 Glob/Grep으로 확인 후 링크
  - 없으면 `(외부 자료)` 또는 빈칸
- 매칭도 판단 기준:
  - **강**: 이력서에 직접적 경험과 성과가 있음
  - **중**: 관련 경험은 있으나 직접적이지 않거나 깊이 부족
  - **약**: 경험 없음 또는 매우 부족
- 예상 질문은 최소 24개 이상 (이력서 기반 7+, JD 기반 7+, 서비스 맥락 5+, 컬처핏 5+)
- 액션파워 예시 문서의 품질과 구체성을 기준으로 작성
- FIT 답변은 `[[FIT-Framework|FIT 프레임워크]]`, `[[Self-Analysis|나찾기]]` 문서를 참조하도록 링크
